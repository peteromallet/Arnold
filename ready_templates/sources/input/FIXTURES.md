# Smoke fixtures

Small input assets committed for the corpus matrix smoke runs. Kept tiny on
purpose -- these are smoke fixtures, not realistic content. The matrix bootstrap
copies these into the run's `input/` dir via `python -m vibecomfy.fixtures copy
--target input` (see `vibecomfy/fixtures.py`).

If a workflow needs a more specific fixture (talking-head with a real face,
music, multi-channel audio, etc.), add a new dedicated fixture and a
per-workflow input contract -- do not extend these smoke assets to cover
modalities they were not built for.

## Audio

| File              | Source                                 | License           | Format                       | Intended use                                                              |
|-------------------|----------------------------------------|-------------------|------------------------------|---------------------------------------------------------------------------|
| `speech_smoke.wav`| Self-generated via macOS `say -v Samantha` (Apple system TTS) | Self-generated; usable as project-internal smoke fixture | mono, 16kHz, 16-bit PCM, ~2.94s | Audio-extraction smoke; non-silent so `NormalizeAudio` and downstream audio nodes get realistic, non-DC input. |

The text spoken is `"This is a smoke test audio sample for vibe comfy."`. The
clip is intentionally short and project-internal -- it is not a substitute for
real speech datasets in any production / benchmarking context.

## Guide videos

All four videos are 256x256, 5 frames at 8fps (~0.625s of video) with the same
visual content (a synthetic test pattern with a sliding rectangle, used as a
deterministic placeholder). Each one has the `speech_smoke.wav` clip muxed in
as an AAC mono 16kHz audio stream so workflows that extract audio from the
video produce a non-empty waveform.

| File                       | Visual content                          | Audio                              | Intended use                                                                                              |
|----------------------------|-----------------------------------------|------------------------------------|-----------------------------------------------------------------------------------------------------------|
| `ltx_smoke_guide.mp4`      | synthetic 256x256 test pattern, 5f@8fps | `speech_smoke.wav` muxed (AAC)     | LTX guide-video smoke -- e.g. `ltx2_3_runexx_*` paths that run a guide video through audio extraction.    |
| `wolf_interpolated.mp4`    | synthetic 256x256 test pattern, 5f@8fps | `speech_smoke.wav` muxed (AAC)     | Generic `VHS_LoadVideo` smoke under WanVideoWrapper / RunExX paths whose templates name `wolf_interpolated.mp4`. |
| `bubble.mp4`               | synthetic 256x256 test pattern, 5f@8fps | `speech_smoke.wav` muxed (AAC)     | LTX RunExX templates that name `bubble.mp4`.                                                              |
| `10.mp4`                   | synthetic 256x256 test pattern, 5f@8fps | `speech_smoke.wav` muxed (AAC)     | LTX community templates that name `10.mp4`.                                                               |

The video bytes are deterministic given the generator (`vibecomfy.fixtures._synthesize_guide_video`).

## Regeneration

```bash
# Primary path: copy committed fixtures into a target input dir.
python -m vibecomfy.fixtures copy --target input

# Fallback path: synthesize fresh. Used when committed assets are missing or
# when VIBECOMFY_FIXTURES_REGENERATE=1 is set. The fallback substitutes a
# 220Hz sine tone for `speech_smoke.wav` (silent-content; satisfies shape
# contracts but gives no useful audio signal).
python -m vibecomfy.fixtures regenerate --target input
```
