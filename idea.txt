# Hideable "Preview query" picker for the VibeComfy chat panel

## Outcome
Add a dev/demo-only, **hideable "Preview query" picker** to the VibeComfy ComfyUI chat panel. A user picks one of ~6 curated saved "agentic scenarios" (real prior DeepSeek runs, fully validated) and the panel **plays it as if a live agent turn happened**: loads the original workflow graph onto the canvas, populates the chat thread (user query → agent reply), and presents the candidate edit (Apply/Reject) — so the user can screen-record a polished before→after **demo video** deterministically, with no live model calls. Gated so it never appears (and its routes never answer) in normal use.

## Why
We have ~100 saved live agentic runs on disk (`out/agentic/agentic-100-20260630-021138/`), 69 fully validated. Each stores the original graph, the agent's reply, and the candidate edited graph. Re-running live costs tokens and varies; the batch-REPL `fixture_provider.py` is the wrong protocol. This picker deterministically replays a chosen run for recording.

## Scope (IN)
- Two backend HTTP routes (aiohttp) to list + serve scenarios.
- One new frontend ES module: a hideable picker toolbar wired into the panel.
- Minimal demo branches in the existing Apply/Reject path.
- A web-asset rebuild.

## Scope (OUT / anti-scope)
- Do NOT touch production agent-edit, submit, or live-apply behavior except the single, well-marked `__demoMode` early-return branches specified below.
- Do NOT build a new bundler/build pipeline — reuse `scripts/build_web_cache_bust.sh`.
- Do NOT render media (no GPU). This shows graph diffs + reasoning + validation gates, not pixels.
- Do NOT change the scenario data on disk; routes read it read-only.

## Locked decisions (verified against source — do not relitigate)
All file:line below confirmed by reading the actual source (a Codex high-reasoning sense-check verified these).

**Frontend (`vibecomfy/comfy_nodes/web/`):**
- `applyGraphCandidateInPlace(app, candidate, options)` at `comfy_adapter.js:112` is **synchronous** — does `graph.clear()` + `graph.configure(candidate)`. `app` is imported in `vibecomfy_roundtrip.js:57` from `"../../scripts/app.js"`.
- Chat thread state = `panel.state.transcriptMessages`; re-render via `scheduleRenderAgentPanel("thread", panel)` (imported from `./panel_scheduler.js`). Message roles are literally `"user"` and `"agent"` (NOT "assistant").
- Candidate bubble renders only when the **panel-level** state is set (message fields alone are NOT enough): `panel.state.turnId`, `panel.state.candidateGraph`, `panel.state.applyEligibility`, `panel.state.applyAllowed`/`canvasApplyAllowed`, `panel.state.candidateGraphHash`, and a `panel.state.sessionId`. See `candidateActionState` at `agent_candidate_actions.js:150`, `applyEligibility` at `:88-99`, and the normal post-submit state in `agent_edit_lifecycle.js:1363-1423` — **mirror that state exactly** on replay.
- `applyAgentCandidate(panel)` at `vibecomfy_roundtrip.js:9846` requires `sessionId`+`turnId` and POSTs to `/vibecomfy/agent-edit/accept` with scope/idempotency guards — **it fails in demo**. Add the demo early-return **at the very top, before** the session/turn guard (`:9851`) and `assertApplyScopeConsistency` (`:9873`), and mimic the `APPLY_SUCCESS` transition (`agent_edit_lifecycle.js:2237-2263`) — not just a graph swap.
- `rejectAgentCandidate` at `vibecomfy_roundtrip.js:~10495` silently no-ops without session/turn (`candidateActionState` doesn't gate Reject on session, `:206-207`). Add a parallel `__demoMode` early-return there.
- Panel shell: `createAgentPanelShell()` at `vibecomfy_roundtrip.js:3114`; header `headerRight` (~`:3231`) holds status/settings-gear/close — add a "▦ Demo" toggle there (mirror the settings-popover pattern). Global panel ref via `currentAgentPanel()` from `./panel_runtime.js`.
- Candidate controls render inside a **collapsed `details` row** (`panel_thread.js:1065-1071, 1195-1275`) — after pushing the demo assistant message, **auto-expand that turn's details** so changes + Apply are visible on screen.
- Frontend JS = ES modules loaded **directly** by ComfyUI (no bundler). A new `web/preview_picker.js` is `import`-ed from `vibecomfy_roundtrip.js`.

**Backend (`vibecomfy/comfy_nodes/`):**
- Routes: aiohttp `@app.routes.get(...)` async handlers in `agent/routes.py`; registered via `register_agent_edit_routes(app)` (~`:1598`); `__init__.py:108-130` wires `PromptServer`. Mirror existing `/vibecomfy/agent-edit/chat` (~`:1419`): `request.query.get`, `asyncio.to_thread`, `_web.json_response`, `_to_serializable`.
- Resolve repo root via `Path(__file__).resolve().parents[3]` from `routes.py` (reaches the repo regardless of ComfyUI cwd — verified).
- **Candidate-path resolution:** only ~39/100 `response.json` carry `artifacts.candidate_ui`. Resolve in order: (a) `response.json.candidate_graph`/`candidate.graph`; (b) `artifacts.candidate_ui`; (c) fallback `Path(artifacts.original_ui).with_name("candidate.ui.json")`. All 6 curated picks have a candidate graph.

**Build (was wrong in the original plan — use this):**
- `__init__.py:51-79` serves **only** `web_dist/<content-hash>/`. `build_web_cache_bust.sh --force` defaults to the **git SHA** (wrong); use the content-hash mode that `scripts/run_local_agent_comfy.sh:47-52` already uses: **`bash scripts/build_web_cache_bust.sh --hash --force`**.

## Open questions
- For a clean demo video, some candidate graphs reference custom-node packs the local ComfyUI may lack (e.g. `QwenEmotionNode`, VACE) → nodes render "missing/red". Not blocking (structure/links still draw), but note which of the curated 6 need packs installed for a clean recording.

## Constraints
- **Hideable + inert by default.** Frontend gate: `localStorage["vibecomfy_demo_picker_enabled"] === "1"`. Backend gate: routes 404 unless `os.getenv("VIBECOMFY_DEMO_PICKER") == "1"`.
- **Path-traversal-safe backend:** validate `id` by exact-match allowlist (curated manifest ids or the summary's `scenario_id`s); reject path separators; resolve under the fixed run dir.
- Non-demo behavior unchanged (all demo code guarded by `panel.state.__demoMode` / env flag).
- One repo (vibecomfy). No GPU. No live model calls.

## Done criteria
1. `bash scripts/build_web_cache_bust.sh --hash --force` produces a `web_dist/<12-char>/` matching the new content hash.
2. With `VIBECOMFY_DEMO_PICKER=1` + `localStorage` flag set, the panel shows a "▦ Demo" toggle → dropdown of the 6 curated scenarios.
3. Load & Play a scenario: canvas loads the original graph → user bubble → working beat → agent bubble + candidate appear with details auto-expanded; **the Apply button is enabled** (proves the panel-state wiring).
4. Apply (bubble or a picker-bar button) swaps the canvas to the candidate graph; deterministic on reload.
5. Without the flags, the picker is gone AND `/vibecomfy/demo/*` returns 404.
6. Existing agent-edit tests still pass; the `__demoMode` branches don't affect non-demo paths.
