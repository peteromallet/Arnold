# Node Resolution Epic

Make vibecomfy able to **understand** a ComfyUI workflow (know every node's real
fields/outputs) and **run** it (install the exact custom nodes it needs) —
reliably, lightly, and without the silent-miscompile / stale-snapshot failures
we hit porting an Ideogram-4 workflow.

## Status

**DESIGNED + REVIEWED — not yet launched.**

- Full design, the verified root cause, and a decision log: **[STRATEGY.md](./STRATEGY.md)**.
- Acceptance contract — the failing workflow, what must happen at each gate, and the runnable gate suite: **[testing/testing.md](./testing/testing.md)** (`pytest tests/acceptance/node_resolution`).

## Shape — 3 gated sprints, each independently shippable

| Sprint | Theme | Ships |
|---|---|---|
| **A** | Correctness spine | Wrong/skewed schema can't be consumed silently (fail-closed at the emitter's real arity site); identity-keyed cache generated by *executed* introspection; merge-on-refresh `build_cache`; real schema-hash plumbing. **Closes the Ideogram crash class.** |
| **B** | Environment realization | Hardened install (state sentinel + cross-pack pip preflight); `ensure-env(workflow)` orchestrator; provenance parsing for *which* packs. **"Drop in a workflow → it installs what it needs and runs."** |
| **C** | Faithful pinning + snapshot demotion | Version-pinned provenance (git-checkout-SHA, local-first git, `aux_id` path); demote/auto-generate the snapshot (core from pinned pip-comfy, per-pack). **Exact authored versions; the hand-captured monolith is gone.** |

Stop after A → correctness fixed. After B → the full user-facing capability.
C → durability.

## Spine (the one invariant)

Schema authority is a resolved **`(pack, commit)` identity** — not "live vs
cache," not a frozen artifact. *Executed* introspection (`/object_info`) is the
authoritative generator; the cache is an identity-keyed serving layer; AST
source is a low-confidence stopgop (blind to `io.Schema` nodes). Correctness
comes from **identity + fail-closed**, not from "go live every time."

## Explicit non-goals (anti-over-engineering)

No universal node-schema registry; no perfect cross-pack dependency solver
(preflight + fail-closed, not a SAT solver); no three-source arity triangulation
in step 1 (UI-vs-cache directionality is enough first); no speculative
"any-backend" abstraction.
