# vibecomfy — Node Resolution, Install & Schema Strategy (v2, post-review)

**Status:** hardened after a 12-reviewer pass (Codex + Opus on architecture; 10 DeepSeek agents on details) and direct code verification. **Scope:** how vibecomfy fetches node schemas, installs custom nodes, and stays current — for both *understanding/porting* a workflow and *running* it, as lightly as possible.

> v2 changes the **spine** and the **first move** from v1. v1 said "flip the porter to live-first." Verification proved that's aimed at the wrong code and wouldn't fix the bug. See §9 for what changed and why.

---

## 1. The two jobs

1. **Understand / visualize ("what's in it").** Import a workflow, know every node's real fields/outputs, port to clean Python — **without booting the full runtime**, but **always correct**.
2. **Run it.** Ensure the workflow's custom nodes are installed at the right versions, then execute — light, provenance-scoped.

Both reduce to: **authoritative, identity-keyed, per-node schema, plus the ability to materialize the environment that provides those nodes.**

---

## 2. The spine (corrected)

**Schema authority is a resolved `(pack, commit)` identity — not "live vs cache," and not a frozen artifact.**

- The crash (`ComfyMathExpression`: snapshot said 2 outputs, reality 3) was **not** a sourcing-order bug. A live boot of the *wrong ComfyUI version* would miscompile identically. The defect is **(a) no identity binding the schema to the node, and (b) fail-open consumption.**
- **Executed introspection (`/object_info` / `node_info()`) is the authoritative *generator*** of identity-keyed schema evidence — it is the only source correct for *all* nodes (incl. the modern `io.Schema`/`define_schema` API). [`nodes_math.py:64`]
- **The cache is the *serving* layer**, keyed to `(pack, commit)`, generated by that introspection. Porting serves from it cheaply.
- **AST source-parsing is a low-confidence stopgap only** — it is blind to `io.Schema` nodes (returns `None`). [`provider.py:988`]
- **Correctness comes from identity + fail-closed**, not from "going live every time."

## 4. What's actually broken/missing (verified)

1. **The emitter's arity path bypasses the provider chain.** Arity comes from `consume.py` (`emitter.py:1947`), not `ConversionSchemaProvider`. Reordering the provider chain does **not** fix the crash. *(This corrects v1.)*
2. **Fail-open on a *known* node with disagreeing arity.** `class_is_known()` returns True for `ComfyMathExpression`; nothing compares the cache's output count against the workflow's UI-declared count → silent miscompile. The existing `UnknownNodeSchemaError` only covers *unknown* nodes. [`consume.py:283`]
3. **No identity plumbing.** `compute_schema_hash` is dead code; `drift.py` compares it against a *different* (file-bytes) hash → never matches. `version="unknown"` for every lockfile entry. So "(pack,commit) identity / schema-hash cache key" must be **built**, not wired. [`node_packs_lockfile.py:286`, `runtime/drift.py:164,188`]
4. **Authoritative source not used for the cache.** The snapshot is hand-captured (`@runpod-snapshot`, ComfyUI 0.18.2), monolithic, replace-on-refresh. No path regenerates it from executed introspection at a pinned version. [`serialize.py:96`, `comfy_metadata.json:7`]
5. **Provenance unused + registry can't pin.** `cnr_id`/`aux_id`/`ver` are carried through but never parsed; `resolve_pack` returns *latest* with no version param; ~8% of real workflows have no provenance at all. [`pack_resolver.py:220`]
6. **Install is unfit for batch/orchestration.** Clone-succeeds/pip-fails leaves a dir that, on retry, hits `_refresh_existing` and **skips pip** while reporting "refreshed"; per-pack bare unpinned pip installs guarantee conflicts at scale; batch loop aborts on first failure with a polluted env. [`node_packs_install.py:115,142,163`]

## 5. Target architecture

1. **Fail-closed, identity-aware schema consumption.** At the emitter's real arity site, compare the workflow's declared arity against the served schema; **fail closed on disagreement** (cache outputs < UI outputs ⇒ stale-snapshot error; > ⇒ warn, may be unused). Generalize to disagreement across the (up to three) arity sources.
2. **Executed introspection as the authoritative generator.** Per-pack schemas generated from `/object_info`/`node_info()` (handles `io.Schema`) at install/ensure time, written to an **identity-keyed** cache (`<pack>@<commit>.json`). AST source demoted to low-confidence stopgap.
3. **Real identity plumbing.** Wire `compute_schema_hash` into install; fix the `drift.py` algorithm mismatch; use **`(pack_slug, git_commit)`** as the cache lookup key, `class_schema_sha256` as a *verification* hash.
4. **Merge-on-refresh cache.** `build_cache` preserves packs not in the source; per-pack version, not a global tag. Core regenerable from a pinned pip-installable ComfyUI (CPU-bound, **not** "light"); custom packs captured per-pack.
5. **Provenance-driven, version-pinned resolution.** Parse `cnr_id`(slug)/`aux_id`(owner/repo)/`ver`(commit SHA); add version-pin to `resolve_pack`/install (git-checkout SHA); **local-first git resolution** (read installed `.git` origin+HEAD) for offline/private packs; warn (never silent-latest) on the provenance-less tail.
6. **Hardened install + `ensure-env(workflow)` orchestrator.** Fix the clone-ok/pip-fail sentinel + a cross-pack `pip install --dry-run --report` preflight; then compose: parse provenance → resolve `(pack, commit)` set → idempotent install/verify → executed-introspect → identity-keyed cache → ready to port/run. Per-pack failure collection, not fail-fast.

---

## 6. Light & efficient (reconciled with reality)

- **Cheap porting = read the identity-keyed cache.** Do **not** boot ComfyUI or run `import_all_nodes_in_workspace` for porting — that's 5–30 s, ~a full boot. Correctness comes from identity + fail-closed, not from going live per-port.
- **Live boot is reserved for two things:** generating cache entries at ensure-time, and actual execution.
- **AST source is the only true zero-boot read**, but low-confidence (io.Schema-blind) — usable for triage with explicit confidence marking, never authoritative.
- **Provenance-scoped install** — only the packs a given workflow needs.

---

## 7. Known hard parts / risks

- **Cross-pack dependency conflicts** (bare unpinned pip, last-wins) — the genuinely hard part; needs a joint preflight, still not fully solvable.
- **Registry can't serve a pinned version** — needs git-checkout-SHA or a GitHub-tags fallback; offline only works for previously-cached URLs (no TTL).
- **Executed introspection runs third-party node code** — a real trust/security surface; must be gated.
- **io.Schema coverage** — AST can't see it; only executed introspection can.
- **Provenance gaps** (~8% of workflows) and `aux_id`-only nodes need a distinct (GitHub) resolution path.

---

## 8. Sequencing (smallest-first, corrected)

1. **Fail-closed arity-disagreement at the emitter's real site** (`consume.py` `require_class_output_count` + `emitter.py:1947/4102`), directionality rule. ~30 lines; kills the silent miscompile; no install/boot/reorder. **This is the first move** (v1's "flip provider" is dropped).
2. **Identity-keyed, executed-introspection cache** + merge-capable `build_cache` + real schema-hash plumbing. Makes the cache correct and self-validating.
3. **Harden install** (state sentinel + cross-pack pip preflight).
4. **Provenance → version-pinned resolution** (parse + version-pin `resolve_pack`/install + local-first git + provenance-less fallback).
5. **`ensure-env(workflow)` orchestrator** composing the hardened pieces, idempotent, per-pack failure collection.
6. **Demote/auto-generate the snapshot** (core from pinned pip-comfy; custom per-pack).

## 9. What changed from v1 and why (decision log)

- **Spine: "live vs cache" → "(pack,commit) identity + fail-closed."** Verified: a live boot of the wrong version miscompiles identically; the bug is missing identity + fail-open, not sourcing order. (Codex/Opus, confirmed in code.)
- **Dropped "flip the porter to live-first" as step 1.** Verified the emitter takes arity from `consume.py`, bypassing `ConversionSchemaProvider` [`emitter.py:1947`]; AST source is io.Schema-blind [`provider.py:988`]; in-process enum isn't light; `RuntimeSchemaProvider` is cache-shadowed [`provider.py:694`]. Reordering the chain wouldn't fix the crash.
- **First move is now fail-closed arity-disagreement detection** at the verified-correct code path.
- **"memoize on class_schema_sha256" → "(pack_slug, git_commit) key; schema-hash is verification."** Verified the schema-hash is dead code with a conflicting drift algorithm.
- **Added:** install state-machine fix + cross-pack pip preflight; executed-introspection as authoritative generator; merge-on-refresh `build_cache`; local-first git resolution; introspection-trust/security note.
