# Adversarial architecture review — Claude Opus

> Reviewer: Claude Opus (independent, could not see the Codex review). Verified
> claims against the code. Verdict: **change it (don't ship as written).**

I verified the doc against the code. Most of its factual claims hold; two are wrong in ways that matter.

**1. Is "schema is a derived view of installed packages; live is truth, cache is fallback" the right spine? — Yes, but the doc under-specifies the spine that actually matters.** The real invariant isn't "live vs cache," it's **"schema must be keyed to a resolved `(pack, version/sha)` identity, and every consumer must agree on that identity."** Live-vs-cache is a *sourcing* detail beneath that. The doc gets the slogan right but buries the load-bearing part (provenance-pinned identity) as gap #2 and sequencing step #3. The crash wasn't caused by "cache instead of live" — it was caused by *no version identity binding the schema to the node*. A live boot of the wrong ComfyUI version would have produced the same miscompile. Make identity the spine; live/cache is the cache policy under it.

**2. Is "live-first for BOTH porting and runtime" correct? — No, and this is the doc's biggest conceptual error.** Section 1 + 6 insist porting stays cheap and "may target nodes the user hasn't installed." Live-first contradicts that: live introspection *requires the pack installed* (in-process enum or `/object_info`). For porting, live-first means **install-to-port**, which violates the stated cost budget for the majority case (porting a workflow you don't run locally). For porting, the right spine is **provenance-pinned cache-first** (resolve `(pack, ver)` from `cnr_id`/`ver`, fetch/generate that pack's schema), with live as the escalation when provenance is missing. Live-first belongs to *runtime only*. The doc inverts this.

Worse: the doc's own gap-#1 premise is **factually wrong**. `ConversionSchemaProvider` is NOT live-cache-last-by-default — runtime is gated off entirely (`enable_runtime=False`, `provider.py:450,562`; `port.py:1501,1511`). The order is `node_index → object_info_cache → object_info_index → source → widget`, runtime never consulted unless `--runtime-object-info`. So sequencing step 1 ("flip the porter to live-first; reorder") does not even describe the real code path. And note: even `RuntimeSchemaProvider.object_info()` reads the on-disk cache *first* (`provider.py:694-701`) — so "flip to live" wouldn't go live anyway while a cache file exists.

**3. Single biggest flaw:** the architecture treats the bug as a *sourcing-order* problem when it is a *missing-identity + fail-open* problem. Reordering to live-first papers over it for installed nodes and breaks the lightweight-port case, while leaving the actual defect — no `(pack,ver)` binding, silent arity fallthrough — only addressed in later, optional steps.

**4. Is the sequencing right? — No. Step 1 is the wrong first move.** It's neither smallest nor highest-leverage, and it's mis-described. The genuine highest-leverage smallest change is **sequencing step 2: fail-closed + arity-disagreement detection.** That directly converts the silent miscompile into a loud, actionable error (the real harm), is ~localized to `get_schema`/the emitter, requires no install path, no live boot, no reorder, and is correct regardless of how 3–5 land. Do step 2 first; demote "flip to live-first" and scope it to *runtime only*.

**5. Structurally missing:** (a) **no cross-source reconciliation** — the doc says "fail-closed on disagreement" but the providers return first-hit and discard the others (`CompositeSchemaProvider.get_schema`, `provider.py:352-357`); you can't detect arity disagreement without *querying multiple sources and comparing*, which no path does. (b) **No schema-contract versioning / migration story** for ports already emitted against stale schemas. (c) **Trust/security of live introspection** — booting arbitrary custom-node Python to read `INPUT_TYPES` executes third-party code; the existing `requesting_provenance`/gate machinery isn't wired to the introspection path.

**Verdict: change it (don't ship as written).** The direction is salvageable but the spine and sequencing are mis-pointed. **Single most important change:** make **provenance-pinned `(pack,version)` identity the spine**, lead with **fail-closed multi-source reconciliation** (step 2 first), and **scope live-first to runtime only** — keep porting on pinned-cache-first so the lightweight, install-free port path survives.
