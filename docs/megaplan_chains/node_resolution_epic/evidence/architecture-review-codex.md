The spine is right, but the wording is too blunt. The better spine is: **schema authority is the realized Comfy environment identity, not any standalone artifact.** Live introspection is the best observation of that identity; generated caches are acceptable only when keyed to the exact environment or declared pack version. Today `ConversionSchemaProvider` does the opposite: `node_index` first, cache next, installed source later, runtime last and off by default ([provider.py](../../../../vibecomfy/schema/provider.py), [port/_shared.py](../../../../vibecomfy/commands/port/_shared.py)). That is architecturally wrong for correctness.

“Live-first for both porting and runtime” is only half-correct. Runtime must be live-authoritative because it queues against the environment it just started; `_build_schema_provider()` returns `RuntimeSchemaProvider` for run validation ([session.py](../../../../vibecomfy/runtime/session.py)). But porting has two modes: **faithful porting** and **cheap triage**. Cheap triage cannot require installing arbitrary custom nodes, so its boundary should be: use installed/live schemas when the relevant pack is already installed and matches provenance; otherwise use version-addressed generated cache and mark confidence/debt explicitly. Do not silently fall back to latest local cache. Also, current `RuntimeSchemaProvider` is not truly live-first: it reads `out/cache` before calling `/object_info` ([provider.py](../../../../vibecomfy/schema/provider.py)). That must be fixed or the architecture repeats the bug under a “runtime” label.

The biggest flaw is **undisciplined environment identity**. The proposal talks about live, cache, lockfile, provenance, and `class_schema_sha256`, but does not define a single `SchemaEnvironment` contract that every provider returns and every compiler gate compares. Without that, “live-first” becomes provider ordering, not correctness. You need identity: Comfy commit/version, pack slug, pack version/commit/path, schema hash, provider kind, and workflow provenance match status.

Sequencing should change. Step 1 should not be “flip porter to live-first.” That is small but too easy to make expensive, nondeterministic, and still stale because runtime caches are currently pre-consulted. The first move should be **fail-closed schema evidence and arity disagreement detection with provenance in diagnostics**. Make wrong schema impossible to consume silently. Then flip provider precedence behind that gate. Existing pin checking already warns/errors on structured custom node refs versus lock entries ([custom_node_refs.py](../../../../vibecomfy/custom_node_refs.py)); extend that same discipline to schema evidence before changing defaults.

Structurally missing: a typed `SchemaEvidence`/`SchemaEnvironment` model, a mode split (`triage`, `faithful-port`, `run`), cache invalidation keyed by environment identity instead of just paths, and an explicit policy for partial schemas. Source parsing is useful but not authoritative for dynamic `INPUT_TYPES()`; it must be low-confidence unless executed in the package environment.

Verdict: **change it, then ship the direction**. Single most important change: define and enforce schema environment identity/conflict checks first; then make porting prefer matching live/installed evidence over frozen caches.
