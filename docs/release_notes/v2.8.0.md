# v2.8.0 — Seams, IR Purity, and Contract Foundation

**Released:** 2026-05-28
**Semver:** MINOR (controlled breaking — see below)
**Tag:** `v2.8.0`

---

## Overview

v2.8.0 is a **controlled breaking minor** under the 2.x line. This release
introduces structural changes to internal IR purity, validation contracts, helper
classification, and public input metadata that may require template authors to
update their workflows. Every breaking change is documented below with its
migration path.

The release lays the foundation for Sprint 3 goals: clean separation of
IR-neutral modules, compile-backend parity, helper-edge resolution, and a
verifiable public API contract.

---

## Controlled Breaking Minor Classification

This release is deliberately tagged as a MINOR version bump despite containing
backward-incompatible changes. Under the project's versioning policy:

- **Major versions (3.x)** would signal a complete public API redesign with no
  migration path from 2.x. That is not the case here — the core abstractions
  (VibeWorkflow, nodes, edges, compile, ready templates) remain intact.
- **Patch versions (2.7.x)** would be inappropriate because new validation
  strictness, removed internal API surfaces, and contract expectations may
  surface as failures in existing code.
- **Controlled breaking minor (2.8.0)** is the correct fit: the changes are
  bounded, documented, and come with explicit migration notes. Template authors
  who follow the migration checklist below should experience no silent breakage.

## Public IR Contract

The code-facing anchor for these guarantees is `vibecomfy.contracts.ir`. It
publishes stable codes that tests, diagnostics, and release notes can refer to
without importing `VibeWorkflow` or ComfyUI-specific modules.

Exact public guarantees for the 2.8.x line:

- `set_input(name, value)` only mutates registered public inputs. Unknown names
  are contract violations (`ir.public_input.unregistered`).
- `set_input(name, value)` rejects stale public inputs whose target node or
  target field no longer exists (`ir.public_input.stale_target`).
- `ValidationReport` exposes success through `report.ok`
  (`ir.validation.report_ok_field`). There is intentionally no compatibility
  alias for this field.
- `validate().ok` means `compile("api")` succeeds
  (`ir.validation.ok_implies_compile_api`).
- Every compiled edge endpoint resolves to a node present in the compiled
  prompt (`ir.compile.edge_endpoint_resolved`).
- Helper/UI-stripped edges are either rewired or reported; they are never
  silently dropped (`ir.compile.helper_edge_rewired_or_reported`).

### Migration Examples

Register public inputs before calling `set_input()`:

```python
wf = load_workflow_any("image/z_image")
wf.register_input("prompt", "3", "text", "old prompt")
wf.set_input("prompt", "a glass teapot on basalt")
```

`set_prompt()` is a convenience wrapper around `set_input("prompt", value)`, so
it has the same registration requirement:

```python
wf = load_workflow_any("image/z_image")
wf.register_input("prompt", "3", "text", "old prompt")
wf.set_prompt("a glass teapot on basalt")
```

Template authors should prefer `InputSpec` when defining ready-template public
inputs:

```python
from vibecomfy.templates import InputSpec, ReadyMetadata, new_workflow, node

READY_METADATA = ReadyMetadata.build(
    capability="t2i",
    template_id="image/example",
    inputs={
        "prompt": InputSpec("1", "text", "a glass teapot", "STRING"),
    },
)

with new_workflow(READY_METADATA, source_path=__file__) as wf:
    prompt = node("CLIPTextEncode", text="a glass teapot")

wf.finalize_metadata()
```

For validation checks, use `report.ok`:

```python
report = wf.validate()
if not report.ok:
    raise RuntimeError(report.issues)
```

## Breaking Changes

| Change | Affected Users | Migration |
|---|---|---|
| `validate()` now calls `compile("api")` unconditionally and treats compile failures as validation errors | Anyone calling `wf.validate()` programmatically | Ensure workflows compile before validation; check `report.ok` instead of assuming silent passes |
| `set_input()` raises `ValueError` for unregistered or stale input targets | Template authors using `wf.set_input()` or convenience methods (`set_prompt`, `set_seed`, `set_steps`, `set_model`) | Register inputs via `bind_input()` before calling `set_input()`; re-register after node replacement |
| Public input metadata (`bind_input` / `finalize_metadata`) is now required before `set_input` | Template authors who previously set inputs before registration | Move `bind_input()` calls before `set_input()` calls in template `build()` methods |
| `_next_node_id()` uses lowest-unused-gap instead of max+1 | Snapshot-based test authors | Auto-generated node IDs may change when gaps exist; regenerate snapshots or accept baseline deltas |
| Helper classification and helper-resolution extraction to IR-neutral modules | Direct importers of internal helper APIs | Import from the new canonical modules (`vibecomfy._helper_classify`, `vibecomfy._helper_resolve`) instead of internal locations |
| Widget-alias dependency extraction to `vibecomfy._widget_aliases` | Direct importers of widget alias utilities | Update imports to the new canonical module |

## New Features

### 1. IR-Neutral Module Extraction

Three new IR-neutral modules have been extracted to enforce clean separation
between ComfyUI-specific logic and the generic workflow IR:

- **`vibecomfy._helper_classify`** — helper node classification
- **`vibecomfy._helper_resolve`** — helper-edge resolution and tracing
- **`vibecomfy._widget_aliases`** — widget alias mapping

Import-linter configuration guards these modules to prevent accidental
ComfyUI-specific imports from creeping in.

### 2. Shared Edge-Input Assembly

Both `compile("api")` and internal `_compile_graphbuilder()` now use a
single shared helper for resolved edge-input construction, eliminating
duplicate logic and ensuring compile-backend parity.

### 3. Public Input Validation

`set_input()` and its convenience wrappers (`set_prompt`, `set_seed`,
`set_steps`, `set_model`) now validate:

- The named input is registered (via `bind_input()`)
- The target node still exists in the graph
- The target field still exists on the target node

Stale inputs from replaced nodes are caught eagerly instead of silently ignored.

### 4. Metadata Provenance

`finalize_metadata()` now preserves public input metadata provenance through
snapshotting, ensuring that metadata survives graph mutations without losing
traceability to the original input declarations.

## Migration Checklist

```bash
# 1. Check that all set_input() calls come after bind_input() calls
rg "set_input\|set_prompt\|set_seed\|set_steps\|set_model" ready_templates/

# 2. Verify workflows validate after compile changes
vibecomfy port check <ready-id> --json

# 3. Update any direct imports of extracted modules
#    Old: from vibecomfy.workflow import _classify_helper
#    New: from vibecomfy._helper_classify import classify_helper

# 4. Regenerate snapshots if node ID changes affect tests
uv run pytest tests/test_snapshot_*.py --update-snapshots
```

## Known Limitations

- **Reroute and PrimitiveNode remain conversion-only for this release.**
  Compile-time stripping is deferred to a future sprint unless focused failing
  tests prove it is required.
- **Import-linter protected list uses actual extracted module names** as
  determined during Steps 4-6. Placeholder names from planning should not be
  used in the linter config.
- **Snapshot regenerator (`--check` gate)** requires runtime ComfyUI
  availability for full fidelity; off-machine CI will produce degraded results.
