# VibeComfy v2.7.0 Release Notes

## Documentation Reconciliation

This release records the M7 documentation pass. The goal is to make the public
docs describe the API surface settled by M6, not older audit findings or stale
agent instructions.

- `AGENTS.md` is now a thin bootstrap that points to `CLAUDE.md`.
- `CLAUDE.md` is the canonical long-form agent guide and cites
  `artifacts/m6-public-api.md` for public import claims.
- `README.md` now tells one current v2.7 quickstart story covering
  load/edit/patch/block/validate/run.
- Agent onboarding now links to `AGENTS.md` and `CLAUDE.md`; the old bundled
  skill path is not present in this checkout.
- `docs/python_composition_dsl_plan.md` remains active as the Layer 2
  architecture reference.

## Public API Notes

The authoritative M6 public import surface is recorded in
[`../../artifacts/m6-public-api.md`](../../artifacts/m6-public-api.md).

Use these current names in new documentation and examples:

- Load workflows with `load_workflow_any`, `workflow_from_ready`,
  `workflow_from_id`, `workflow_from_file`, or `load_workflow_json`.
- List ready templates with `ready_template_ids`.
- Run workflows with `run`, `run_sync`, `run_embedded`, or
  `run_embedded_sync`.
- Use top-level namespaces `image`, `video`, `blocks`, `patches`, and `router`
  for Layer 2 composition.
- Export ComfyUI API JSON with `VibeWorkflow.compile("api")`.

Compatibility aliases remain available:

- `workflow_from_template` is a back-compat alias for `workflow_from_id`.
- `load_template` is a back-compat alias for `load_workflow_json`.

New documentation should prefer the current names above. The old audit claim
that these aliases and loader exports were missing is superseded by
`artifacts/m6-public-api.md`.

## Migration Notes

No standalone migration-notes file exists in this checkout, so the v2.7 API
rename/alias guidance is recorded here.

- Replace older "template" language with "workflow" unless the object is a
  curated ready template under `ready_templates/`.
- Prefer `workflow_from_id` over `workflow_from_template` in new code.
- Prefer `load_workflow_json` over `load_template` in new code.
- Prefer `VibeWorkflow.compile("api")` for API JSON export examples.
- Prefer top-level runtime helper imports from `vibecomfy`, such as
  `from vibecomfy import run_embedded_sync`.

## Validation Follow-ups

- `tests/test_agent_skill_sync.py` and `scripts/sync_agent_skill.py` still encode
  the pre-M7 agent-doc contract where `AGENTS.md` is the canonical skill file
  and `CLAUDE.md` is a symlink to it. M7 deliberately inverted that relationship:
  `AGENTS.md` is now a thin bootstrap and `CLAUDE.md` is the canonical long-form
  guide. Updating that synchronization test/script is a follow-up code task, not
  part of this documentation-only reconciliation.
