## 2. Tests That Must Change Under Deletion-First

- **`test_node_shims.py`** — Both `test_generated_core_wrappers_are_importable_and_delegate` and `test_generated_pack_modules_import_and_exclude_helper_nodes` import through the shim layer. They'd shift to `vibecomfy.nodes._generated.<pack>`. The `test_prune_stale_thin_shims_*` tests *exercise the pruning mechanism itself* — they'd be rewritten to test that deletion works correctly.
- **`test_generated_node_wrappers.py`** — Imports from `vibecomfy.nodes.core` → `vibecomfy.nodes._generated.core`. The `.pyi` re-export assertion (line 93) must reflect the new layout.
- **`test_templates_module.py:1102`** — Same import shift.
- **`characterization/goldens/emitter/ltx2_3_*.ready.py.golden`** — Two golden files import from `vibecomfy.nodes.ltxvideo`. Would need regenerating with `from vibecomfy.nodes._generated.ltxvideo import ...`.

## 3. Accidental vs. Intentional

The `vibecomfy.nodes.<pack>` re-export layer is **intentional public API** — it provides a stable import path decoupled from `_generated/` internals. The characterization goldens are **intentional regression guards** for what the emitter produces. The `test_api_surface.py` (module-`__all__` snapshots) intentionally **omits** `vibecomfy.nodes` from its tracked exports, so the shim deletion doesn't affect that contract guard at all.
