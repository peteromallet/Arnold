## Node Shim Analysis

### Structure

Each `vibecomfy/nodes/<pack>.py` is an identical 4-line re-export:

```python
from vibecomfy.nodes._generated import <pack> as _generated
from vibecomfy.nodes._generated.<pack> import *
__all__ = list(_generated.__all__)
```

Plus a matching `<pack>.pyi` stub. **Zero business logic.** The `index.py` file is unrelated (node-listing utilities) and is **not** a shim.

**No code in `vibecomfy/` (the main package) imports through the shims directly** â€” only through `__init__.py`'s wildcard re-export.

### Tools that Own the Shims

`tools/generate_node_shims.py` generates all shim files via `_write_reexport_module()` / `_write_reexport_stub_module()`. The `_prune_stale_thin_shims()` function treats them as prunable artifacts.

### 1. Can Be Safely Deleted Now (all 38 files)

**All 19 `.py` shims + all 19 `.pyi` shims** at `vibecomfy/nodes/<pack>.py` and `vibecomfy/nodes/<pack>.pyi`. Every one is a mechanical re-export with zero logic. The `_generated/` modules are the source of truth, and `__init__.py` can re-export directly from them.
