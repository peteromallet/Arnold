### 3. Tests Needing Path Migration After Shim Deletion

**Critical finding: `vibecomfy.porting.wrappers` does not exist anywhere in the source tree.**

Four files import from it — they will `ImportError` at module load time:

| File | Broken Import |
|---|---|
| **`tests/test_wrapper_codegen.py`** (lines 18–19) | `from vibecomfy.porting.wrappers import codegen as wc`<br>`from vibecomfy.porting.wrappers.discovery import ClassSpec, InputFieldSpec` |

| **`vibecomfy/commands/nodes.py`** (lines 24–25) | `from vibecomfy.porting.wrappers import codegen as _wrapper_codegen`<br>`from vibecomfy.porting.wrappers import discovery as _wrapper_discovery` |
| **`scripts/demo_wrapper_codegen.py`** (lines 27–28) | `from vibecomfy.porting.wrappers import codegen as wc`<br>`from vibecomfy.porting.wrappers import discovery as wd` |

These are **top-level imports**, not lazy. `test_wrapper_codegen.py` and `test_wrapper_discovery.py` cannot possibly run. The `commands/nodes.py` module will fail on any import of the `vibecomfy.commands` package that triggers the `nodes` subcommand module.

**Action needed:** Either create `vibecomfy/porting/wrappers/` (with `codegen.py`, `discovery/__init__.py`, etc.) or delete the referencing files and update `vibecomfy/commands/nodes.py`.
