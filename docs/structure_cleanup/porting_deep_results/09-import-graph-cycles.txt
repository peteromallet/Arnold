## 1. Cycles Detected

### CYCLE A: `emit_kwargs.py` ↔ `emit/emitter.py`

```
emit_kwargs.py  ──top-level──►  object_info, widgets.*
```

**Severity: Moderate.** The lazy/function-level imports prevent a runtime crash, but the conceptual cycle forces cautious refactoring — any change in the shared boundary can cascade.

**Root cause:** The old monolithic `porting/emitter.py` was split into `emit_kwargs` (leaf) and `emit/emitter.py` (hub), but the split was incomplete: `emit_kwargs` still needs `_wrapper_module_for_class`, `_node_local_arity_check`, `RESERVED_WRAPPER_INPUT_NAMES` from the emitter. These should live in a shared leaf (e.g., `emit/_shared.py`).
