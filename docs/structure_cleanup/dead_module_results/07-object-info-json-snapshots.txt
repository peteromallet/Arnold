### Verdict: **DELETE — but with prerequisite updates**

The 3 JSON files in `vibecomfy/porting/object_info/` are **stale duplicates** of the identical files in `vibecomfy/porting/cache/object_info/`. They are:

1. ✅ **Not consumed** by the primary runtime (`consume.py` reads from `CACHE_DIR` only).
2. ✅ **Redundant** — identical copies exist in the canonical cache directory.
3. ✅ **Identified as deletable** by a prior cleanup brief (`10-safe-first-batch.txt`).

**However**, there is one blocking dependency:

`scripts/demo_wrapper_codegen.py` (line 40, 114) explicitly calls `wd.discover_pack(pack, sources=("snapshot",))` which reads from the legacy directory. Deleting the files would break this script.

### Recommended Action

**Delete the 3 files after making these two updates:**

1. **`scripts/demo_wrapper_codegen.py`** — Change `sources=("snapshot",)` → `sources=("cache",)` (or remove the source override entirely to use the default precedence which tries cache first).

2. **Update docs** (low priority) — Refresh the doc references in `wrapper_discovery.py`/`wrappers/discovery.py` docstrings and any template docs that still describe the legacy `snapshot` path as a live source.

The `--source snapshot` CLI option in `commands/nodes.py` can remain as-is: with the legacy directory gone, it will simply fail to find anything when explicitly selected, which is a clear signal that the source is defunct. You could also remove the CLI option, but it's harmless.

After these changes, delete:
