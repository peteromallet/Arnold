## Dead Module Audit 09: Runtime Duplicate Private Helpers

### Files Flagged
- `vibecomfy/runtime/config.py`
- `vibecomfy/runtime/session.py`
- `vibecomfy/runtime/server_process.py`

### Diagnosis: NOT DEAD — Behavior Refactor (Stale Extraction Drift)

All three files are actively imported and form a live dependency chain. However, they contain **divergent duplicates** of the same logic — a classic "extracted then evolved asymmetrically" pattern.

#### Dependency Graph (Who Imports What)

```
config.py  ←── server.py, watchdog_runtime.py, metadata.py, server_process.py
                (import SessionConfig, _comfy_server_argv)

server_process.py  ←── server.py (imports _spawn_comfy_server)
                   ←── config.py (imports _comfyui_executable)

session.py  ←── __init__.py, run.py, tests/*, tools/*
                (imports SessionConfig, EmbeddedSession, ServerSession, etc.)
```

No external consumers import `config.py` or `server_process.py` directly — they are internal runtime modules consumed by other runtime files.

#### Behavioral Divergence

This is a **real runtime divergence**, not just cosmetic:

| Code Path | Uses | Sage Attention? | Dir Flags? | Timeout |
|-----------|------|----------------|------------|---------|
| `server.py` (via `run()` API) | `config.py` & `server_process.py` old versions | ❌ No | ❌ No | 120s hardcoded |

Both paths are **live and executed** in different operational modes. Neither is dead.

### Verdict & Action

**Action: CONSERVATIVE — Do NOT delete any file. Defer consolidation.**

| File | Status | Rationale |
|------|--------|-----------|
| `runtime/config.py` | **Keep** (not dead) | `server.py`, `watchdog_runtime.py`, `metadata.py` import from it. The older `SessionConfig` (without `strict_drift`) is still the one consumed by those internal path. Deleting would break `server.py`. |
| `runtime/server_process.py` | **Keep** (not dead) | `server.py` and `config.py` import from it. The older `_spawn_comfy_server` is still live via the `run()` → `server.py` code path. Deleting would break `server.py`. |
| `runtime/session.py` | **Keep** (primary, not dead) | Widest import surface — `__init__.py` re-exports from here. This is the canonical "ahead" version. |

### Deferred Consolidation Plan

The pre-existing audit (`docs/structure_cleanup/package_results/04-runtime-subsystem.txt`) already documents this correctly. The recommended actions are:

1. **Merge `config.py` into `session.py`** — eliminate duplicate `SessionConfig` and helper functions. Re-route `server.py`, `watchdog_runtime.py`, `metadata.py` imports to `session.py`.
2. **Merge `server_process.py` into `session.py`** — eliminate duplicate `_spawn_comfy_server`. Re-route `server.py` import.
3. **Update `server.py`** to use the evolved `_spawn_comfy_server` (with configurable timeout, logging, sage-attention, directory flags) instead of the old one.

These are safe, low-risk refactors (private functions, no public API change) but are **not dead-code deletions** — they are behavioral alignment. Defer to MP-5 alongside the `ServerSession`/`ExternalSession` sharing epic.
