### Moves / deletions to DEFER

These are structurally risky in a first batch because they require coordinated updates across multiple registries.

| Change | Risk | Why defer |
|---|---|---|
| **Rename any ready_template file** | Medium–High | `coverage.json`, `ready_regeneration.json`, `asset_manifest.json`, pre-commit hooks, and the generated `template_index.json` all reference template paths. A rename requires updating all of these. Safe only after running `sources sync` and verifying parity. |
| **Move JSON corpus files between subdirectories** | High | `workflow_index.json`, `external_workflow_index.json`, and `coverage.json` all hold path references. Moving a file without updating all three breaks discovery. |
| **Delete stale `coverage.json` entries for non-existent templates** | Medium | Some `coverage.json` entries reference `ready_template: true` without a corresponding `.py` file (e.g., the `wanvideo_wrapper_*` entries that say `"ready_template": true` but the `.py` may be missing if regeneration dropped them). Needs a `diff --check` pass. |
| **Delete deprecated migration docs** | Low | `docs/migration/v23_to_v24.md`, `v25_to_v26.md`, `v25_to_v27.md` — historical but harmless. Safe to clean but distracting in a first batch focused on template/corpus/recipe only. |
| **Flatten `scripts/maintenance/` into `scripts/`** | Low–Medium | The `_regen_templates.py` and `_fix_t6.py` scripts are prefixed with `_` suggesting private status. Moving them would break any automation that references `scripts/maintenance/`. Verify no callers first. |
