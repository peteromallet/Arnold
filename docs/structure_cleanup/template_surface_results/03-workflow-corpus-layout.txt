## Audit: `workflow_corpus/` Layout

### 1. Corpus Layout — Deliberate and Loader/Index Backed?  

**Yes.** The layout is intentional with two indexing paths:

| Corpus dir | Index written to | Loader |
|---|---|---|
| `workflow_corpus/official/` | `workflow_index.json` (repo root) | `vibecomfy/ingest/sources.py` → `index_workflows()` |
| `workflow_corpus/custom_nodes/` | `external_workflow_index.json` (repo root) | same pipeline |
| `workflow_corpus/manifests/` | *(consumed directly, not indexed)* | `vibecomfy/templates.py`, `vibecomfy/registry/static_contract.py` |
| `workflow_corpus/input/` | *(fixtures, not indexed)* | `vibecomfy/fixtures.py`, `vibecomfy/testing/_fixtures.py` |

**Workflow ID = filename stem** (`.json` removed). The indexer in `vibecomfy/ingest/index.py` walks `*.json` under `official/` and `custom_nodes/` and sets `id: path.stem`. This is the sole source of truth for all downstream lookups (`_workflow_path.py`, `registry/library.py`, `search/bootstrap.py`).

The two root indexes (`workflow_index.json`, `external_workflow_index.json`) are **generated artifacts** — regenerated via `vibecomfy sources sync`. They live *outside* the corpus at repo root, which is correct.

Minor cleanliness issues found:

| Issue | Path | Severity |
|---|---|---|
| macOS metadata file | `workflow_corpus/.DS_Store` | Low — should be gitignored or removed |

No `.schema.json` files, `.pyc` files, or stale conversion artifacts were found inside the corpus. The `ready_templates/` directory does contain `__pycache__/` dirs (its own problem), but those are outside the corpus boundary.

### 4. Safe vs. Risky Changes

#### ✅ Safe (ID-stable)

| Change | Why |
|---|---|
| **Add `workflow_corpus/README.md`** | No IDs touched |
| **Remove `workflow_corpus/.DS_Store`** | macOS junk, zero index impact |
| **Add new `.json` workflows** | New stem = new ID; existing IDs unchanged |
| **Add new subdirectories** under `official/` or `custom_nodes/` | Indexer walks recursively |
| **Update `manifests/coverage.json`** | References IDs independently; run `vibecomfy sources sync` afterward |
| **Update `manifests/ready_regeneration.json`** | Same as above |
| **Add/remove fixture files in `input/`** | Not indexed; consumed by `vibecomfy.fixtures` only |
| **Modify JSON content** of any workflow (node positions, model refs, etc.) | Filename unchanged → same ID |
| **Re-run `vibecomfy sources sync`** | Regenerates indexes from same files → same IDs, just refreshed paths |

#### ❌ Unsafe / ID-breaking

| Change | Consequence |
|---|---|
| **Rename any `.json` file** | New stem → new ID in re-index. All `workflow_index.json`, `external_workflow_index.json`, `coverage.json`, and `ready_regeneration.json` references will break until updated |
| **Move a `.json` to a different directory** | Path in index changes. Coverage/regeneration manifests that reference the old path break |
| **Delete a `.json` referenced by any index/manifest** | Orphaned references in all downstream consumers |
| **Rename `official/` → anything else** | Hardcoded default in `sources.py` line 19 and `sources` CLI argument default |
| **Rename `custom_nodes/` → anything else** | Hardcoded default in `sources.py` line 20 |
| **Move `manifests/` outside corpus** | Breaks 6 files with hardcoded `workflow_corpus/manifests/coverage.json` paths |

---

### Summary of Recommendations

1. **Add `workflow_corpus/README.md`** documenting the ID-stem contract, `official` vs `custom_nodes` split, `manifests/` role, and index regeneration workflow. Low effort, high clarity.

2. **Remove `workflow_corpus/.DS_Store`** and add `*.DS_Store` to `.gitignore`. Trivial.

3. **Do not move `manifests/`** — it's correctly placed inside the corpus as metadata *about* the corpus, and six source files hardcode the path.

4. **No generated artifacts are misplaced** — the two root indexes are correctly outside the corpus.

5. **No filenames should be renamed** without coordinated updates to all indexes and manifests — the stem is the global ID.
