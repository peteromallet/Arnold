### Q1: Is the category layout consistent and loader-backed?

**Yes, consistently backed by the loader — but the README is wrong.**

The loader in `vibecomfy/registry/ready.py` (line 328) uses `root.rglob("*.py")` with no directory whitelist — any subdirectory is valid. The `template_index.json` include rule (`find ready_templates -type f -name '*.py' ! -name '_*' | sort`) likewise accepts all categories. So `audio/` and `smoke/` work at runtime.

**Problem: the README only documents 3 categories.** Line 7 says:

> Templates are organized by category (`edit/`, `image/`, `video`)

Missing: **`audio/`** and **`smoke/`**. The README mentions audio *content* at line 13 ("image/edit...") but never lists `audio/` as a directory category. The `smoke/` directory is not mentioned at all.

`audio/` has 4 templates that are `pure_python` source (verified by `ready_template_source_info` check). `smoke/empty_image_red` is a `generated` marker template used for runtime smoke validation (`capability: runtime_smoke`).

**Verdict:** Layout is functional but **under-documented** in the README.

```

```

These are **safe to delete** — `.pyc` files are auto-regenerated at import time. The loader (`_template_paths` at `ready.py:328`) explicitly excludes `__pycache__/` via `rglob("*.py")` so they are invisible to runtime.

#### b) `.DS_Store` (macOS metadata)

```
ready_templates/.DS_Store
```

Safe to delete. Add to `.gitignore` if not already there.

#### c) `VALIDATION.md` — stale content, not harmful at runtime

The **46-template** count is 18 behind reality. The validation commands and methodology are still valid; the document just needs the count updated (or removed if the process is now CI-enforced).

**Verdict:** The `__pycache__/` and `.DS_Store` are noise, not runtime hazards, but they clutter the tree. `VALIDATION.md` is harmless.

| Change | Risk |
|--------|------|
| **Move/rename any `.py` file** | Changes template IDs; breaks `ready_template_ids()`, `template_index.json`, referenced ids in tests (`test_ready_templates.py` snapshot list, `test_runpod_matrix.py` scope config, coverage.json). |

| **Restructure directories (e.g., flatten categories)** | Would change all template IDs; large blast radius. |
