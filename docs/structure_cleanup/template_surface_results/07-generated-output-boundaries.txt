## Audit: Generated Output Boundaries Around Templates and Corpus

### 1. Separation of Generated Artifacts from Source ✅

The project maintains a **clean boundary**:

| Area | Purpose | Status |
|---|---|---|
| **`out/`** | All runtime output (runs, cache, scratchpads, subagent reviews, pre-mortems, runpod artifacts, migration reports) | ✅ Fully gitignored (`out/` in `.gitignore`). All files found inside are generated/debug artifacts. |
| **`artifacts/`** | Committed historical baseline/review evidence (M1–M5 milestone evidence) | ✅ Intentionally tracked. `artifacts/README.md` explicitly documents this as non-runtime-output. |
| **`ready_templates/`** | Source Python scratchpads (authoring surface) | ✅ Tracked source. Each file is a `# vibecomfy: generated` template — these are the *source texts* that agents edit, not runtime output. |
| **`recipes/`** | User-facing compositions built from ready templates + patches | ✅ Tracked source. Small set (7 files). |
| **`workflow_corpus/`** | Source ComfyUI JSON workflows and test fixtures (images, video, audio) | ✅ Tracked source. Clean separation from generated Python. |

The `.gitignore` correctly excludes:
- `out/` (line 14)
- `__pycache__/` (line 2), `*.pyc` (line 7)
- `.DS_Store` (line 6)
- `*.patch` (line 30)
- Runtime indexes like `workflow_index.json`, `template_index.json`, `node_index.json` (lines 10–13)

Docs references (`README.md`, `docs/`, `tests/`) consistently use `out/scratchpads/` and `out/runs/` as the canonical locations for generated artifacts — no leakage into source dirs from code.
