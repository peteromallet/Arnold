### Recommended Structure Changes (minimum viable)

**1. Extract `vibecomfy/workflow/types.py`** (highest ROI)
- Move: `VibeWorkflow`, `VibeNode`, `VibeEdge`, `VibeInput`, `VibeOutput`, `WorkflowSource`, `WorkflowRequirements`, `RawWidgetPayload`, `ValidationIssue`, `ValidationReport`, `Handle` (or keep `Handle` in `handles.py`)
- New file has zero `from vibecomfy` imports (only stdlib / dataclasses)
- `workflow/__init__.py` re-exports `from .types import *` — zero code changes needed in callers
- All lazy-import workarounds for the `testing↔workflow` cycle can become eager imports

**2. Relocate `_derive_output_kind` out of `templates.py`**
- Move it to `registry/ready_template.py` (already loaded by templates.py)
- Deletes 3 lazy imports in emitter files

**3. Strip `workflow` import from `metadata.py`**
- Replace `VibeInput`/`VibeNode`/etc. type annotations with `Any` or plain dict — these are constant sets, not factories
