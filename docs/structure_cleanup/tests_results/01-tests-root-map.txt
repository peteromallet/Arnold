### 2. Root-Level Files That Earn Their Placement

**Untouchable — must stay at root:**

- **`tests/__init__.py`** — Makes `tests` a package. Required by the multiple `from tests._cli_helpers import ...`, `from tests._runtime_session_helpers import ...`, `from tests.smoke import ...`, `from tests.support.corpus_schema import ...` imports used across the suite.

- **`tests/conftest.py`** — Root conftest doing all of:
  - `sys.path.insert(0, ...)` — repo root setup
  - Module cache busting (pops stale `vibecomfy` modules)
  - `_KNOWN_FAILURES_FILE = pathlib.Path(__file__).parent / "known_failures.txt"` — relative path literal
  - `pytest_configure` / `pytest_sessionfinish` — RunPod budget helpers imported from `tests.smoke._runpod_helpers`
  - `pytest_collection_modifyitems` — runpod marker filtering
  - `pytest_terminal_summary` — known-failures baseline gate
  - Autouse fixtures for workflow context isolation and ComfyUI import pollution cleanup
  - All **must** be at suite root for correct scoping.

- **`tests/known_failures.txt`** — Referenced by `conftest.py` line 27 with a relative path. Contains 238 entries that map to `tests/test_*.py::test_*` node IDs. Moving it breaks the baseline gate entirely.

- **`tests/_cli_helpers.py`** — Imported by `test_cli_misc.py`, `test_cli_port.py`, `test_cli_models_fetch.py` as `from tests._cli_helpers import ...`. Underscore prefix prevents pytest collection.

- **`tests/_runtime_session_helpers.py`** — Imported by `test_runtime_session_config.py`, `test_runtime_session_embedded.py`, `test_runtime_session_validation.py`, `test_runtime_session_server.py`, `test_runtime_session_run_untracked.py` as `from tests._runtime_session_helpers import ...`.

**All `test_*.py` files at root (~150+ files)** — These are the main unit test suite. They earn their placement by convention:

- `testpaths = ["tests"]` in `pyproject.toml` discovers them directly
- Every node ID in `known_failures.txt` is relative to `tests/` root (e.g., `tests/test_schema.py::test_...`)
- Any move of a test file would require:
  1. Updating every entry in `known_failures.txt` that references it
  2. Updating CI scripts or shell aliases that target specific paths
  3. Updating any `docs/` or comments referencing the old path

No other root-level files exist. The directory is flat with only test files, two helper modules, conftest, `__init__.py`, and the known-failures baseline.

### Summary

The `tests/` directory is a healthy but **flat** layer. Its only structural issues are:

1. **No README** — zero navigation docs for 13 subdirectories and 150+ root test files
2. **Flat test root** — all core unit tests live at `tests/` root by convention, not by necessity; reorganizing is deferred due to `known_failures.txt` node ID anchoring
3. **Clean separation** — subdirectories are well-named and appropriately scoped; no files are in the wrong subdirectory
4. **Immobile spine** — `conftest.py`, `__init__.py`, `known_failures.txt`, and the two underscore-prefixed helpers (`_cli_helpers.py`, `_runtime_session_helpers.py`) form the structural spine and must not move

**The single highest-value action is writing `tests/README.md`.** Second is writing READMEs for `characterization/`, `intent/`, and `security/`. Everything else is either immovable, safe to delete only if you accept the import-breakage, or deferred.
