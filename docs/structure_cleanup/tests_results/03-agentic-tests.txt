- **No large committed evidence packs found under `tests/`.** The golden tests (`test_agentic_golden_m4.py`, `test_agentic_golden_m5.py`) generate evidence in `tmp_path` at runtime and assert on shape/contents — they do not commit the generated evidence.

### 4. Path contracts that prevent moving these files

Every agentic test file depends on one or more of the following path contracts:

#### a) `Path(__file__).resolve().parent.parent` → repo root

Used in **all** agentic test files to navigate to `agentic/` source:

| File | Expression | Resolves to |
|------|-----------|-------------|
| `test_agentic_structural.py` | `Path(__file__).resolve().parent.parent / "agentic" / "scenarios" / ...` | `{repo}/agentic/scenarios/` |
| `test_agentic_golden_m4.py` | `Path(__file__).resolve().parents[1] / "agentic"` | `{repo}/agentic/` |
| `test_agentic_golden_m5.py` | (same pattern as M4) | `{repo}/agentic/` |
| `test_agentic_contract.py` | (same pattern) | `{repo}/agentic/` |
| `test_agentic_adapter.py` | (inline via `build_faking_structural_chain` import) | N/A (uses imports, not paths) |
| `test_agentic_success_rate.py` | `Path(__file__).resolve().parents[1]` | `{repo}/` — also `REPO_ROOT` for `scripts/` |

**Contract:** These tests **must** remain at exactly `tests/test_agentic_*.py` (depth 1 from repo root) for the `parent.parent` / `parents[1]` navigation to continue working. Moving them into a subdirectory like `tests/agentic/test_structural.py` would break every path resolution.

**Fix recommendation:** Factor the repo-root resolution into a `tests/support/` helper (e.g., `repo_root()`) so the path contract is centralized. All agentic tests should import from that helper rather than recomputing `Path(__file__).resolve().parent.parent`.

#### b) Import-based contracts

| Import | File(s) | Risk |
|--------|---------|------|
| `from agentic.actors import ...` | `test_agentic_structural.py`, `test_agentic_adapter.py` | These are stable `agentic.*` imports — no path dependency |
| `from agentic.adapter import VibeComfyProjectAdapter` | `test_agentic_adapter.py`, `test_agentic_contract.py` | Stable |
| `from agentic.runner import ...` | `test_agentic_runner.py`, `test_agentic_contract.py` | Stable |
| `from agentic.actors_m4 import _M4_BUILDERS` | `test_agentic_golden_m4.py` | Stable |
| `from agentic.actors_m5 import _M5_BUILDERS` | `test_agentic_golden_m5.py` | Stable |
| `pytest.importorskip("sisypy")` | `test_agentic_runner.py`, `test_agentic_adapter.py`, `test_agentic_contract.py` | Runtime guard, not a path issue |

**Contract:** Imports from `agentic.*` require the `agentic/` package to be on the Python path. This is satisfied by running tests from the repo root (or having the repo in `sys.path`). Moving test files deeper would not break these imports as long as the Python path is correct.

#### c) File-reference contracts

| Reference | File | Dependency |
|-----------|------|-----------|
| `REPO_ROOT / "scripts" / "agentic_success_rate.py"` | `test_agentic_success_rate.py` | Hard-coded relative path to sibling `scripts/` dir |
| `REPO_ROOT / "tests" / "fixtures" / "agentic_success_rate.json"` | `test_agentic_success_rate.py` | Hard-coded path to fixture |
| `Path.home() / "Documents" / "reigh-workspace" / "sisypy"` | `test_agentic_structural.py` | **Brittle developer-machine path** (introduced in `_load_recovery_scenario` and `_load_scenario_yaml`). This path is a fragility risk — it won't work on CI or other machines without the same directory layout. |

**The `~/Documents/reigh-workspace/sisypy` path** in `test_agentic_structural.py` (lines 186-188, 373-375, 388-390) is a serious path contract problem. It hard-codes a developer's local workspace path. This should be replaced with either:
- A `PYTHONPATH`-based import (the tests already use `pytest.importorskip("sisypy")` elsewhere)
- An environment variable (`$SISYPY_PATH`)
- A pytest fixture that installs or locates Sisypy

#### d) `subprocess` call contract

`test_agentic_success_rate.py` invokes `[sys.executable, str(SCRIPT_PATH), ...]` (line 28+). This requires that `scripts/agentic_success_rate.py` remain at that exact path relative to `tests/`. If tests move under `tests/agentic/`, the `REPO_ROOT` computation would need adjustment (it currently uses `parents[1]` which would become `parents[2]`).
