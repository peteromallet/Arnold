### 2. External Service Dependencies

- **GPU / RunPod** — All `tests/smoke/test_*runpod*.py` files (5 tests). Gated by `@pytest.mark.runpod` or `@pytest.mark.runpod_full`. Require `RUNPOD_API_KEY` and provision real GPU pods with budget caps ($2 default, $15 for `--runpod-full`).
- **ComfyUI server** — `tests/test_comfy_nodes_live_smoke.py` requires a running ComfyUI instance at `VIBECOMFY_COMFYUI_URL`. Requires Playwright.
- **ComfyUI submodule** — `tests/test_embedded_comfy_model_paths.py` imports from `vendor/ComfyUI/comfy.cli_args`.
- **Node.js** — `tests/test_layer4_smoke.py` and `tests/e2e/` require Node.js.
- **Anthropic** — `tests/intent/` tests marked with `intent_ci` may use the optional `[intent]` extra.
- **Network** — RunPod tests (network to runpod.io), live ComfyUI smoke (network to running server). All other tests mock/stub network calls.

### 6. Findings & Recommendations

#### 🔴 Issues

1. **Stale E2E artifacts committed** — `tests/e2e/test-results/` and `tests/e2e/playwright-report/` contain hundreds of KB of stale test traces and HTML reports from local runs. The `.gitignore` already lists them, but they were committed before the gitignore entry. **Action:** `git rm -r --cached` these directories.

2. **`gpu` marker is defined but unused** — `pyproject.toml` defines a `gpu` marker, but no test in the repo uses `@pytest.mark.gpu`. The default filter `-m 'not gpu'` is a no-op. Consider either removing the marker or tagging the RunPod pod-launch tests with it.

#### 🟡 Minor / Advisory

3. **No top-level testing docs** — Add a `tests/README.md` (or augment the main `README.md`) covering:
   - How to run unit/runtime tests: `pytest tests/test_runtime_*.py`
   - How to run smoke tests: `VIBECOMFY_COMFY_SMOKE=1 pytest tests/test_layer4_smoke.py`
   - How to run RunPod tests: `pytest --runpod tests/smoke/`
   - How to run e2e tests: `node tests/e2e/run.mjs [--comfyui-dir ...]`
   - How to run browser unit tests: `node --test tests/browser/*.test.mjs`
   - Budget defaults and env vars for RunPod

4. **`test_smoke/` directory naming** — Tests in `tests/smoke/` are properly marked `runpod`/`runpod_full` but the directory name `smoke` is generic while all tests are RunPod-specific. Consider renaming to `tests/runpod/` to make the purpose explicit, or keep as-is since the markers are clear.

5. **`test_z_image_only.py` duplicates `test_p1_runpod.py`** — Both do essentially the same thing (single z_image smoke on one pod). Unless there's a subtle difference in the remote payload, this is redundant. The P1 test uses typed handle construction; the z_image-only test uses `load_workflow_any`. If both are needed for different validation surfaces, add a comment explaining the distinction.

6. **No `__init__.py` in `tests/smoke/`** — The file is empty (single blank line). Works fine for namespace packages, but could use a docstring explaining the package purpose.

7. **Runtime tests are all monkeypatched but not marked** — They could benefit from a `runtime` marker for selective execution, though the `test_runtime_*.py` naming convention makes them easy to target.

### 7. Does Anything Need Deleting?

**No expensive tests are duplicated or stale enough to delete.** The existing RunPod tests serve distinct purposes:
- `test_p1_runpod.py` — Typed handle API (P1 priority)
- `test_z_image_only.py` — `load_workflow_any` + `run_embedded_sync` path
- `test_layer2_runpod_ops.py` — Verb-native `image.t2i()`, `video.t2v()`, `video.i2v()` across 6 templates
- `test_layer2_runpod_dropped.py` — Tests templates that don't fit the main ops flow (GGUF, edit)
- `test_layer2_runpod_matrix.py` — Multi-pod parallel production matrix

The only possible consolidation: **`test_p1_runpod.py` and `test_z_image_only.py` could merge** since both test one template on one pod. But the different API surfaces (typed handle vs. reference loader) may justify keeping them separate. Not recommending deletion.
