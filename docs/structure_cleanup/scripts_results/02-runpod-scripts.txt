## RunPod Script Audit — References, Risks, Safe Actions

### Current Inventory (8 files under `scripts/`)

| File | Role | Invoked-by |
|------|------|------------|
| `runpod_runner.py` | Core shim over `runpod-lifecycle`. Exports `run_pod`, `run_pod_detached`. Imports `runpod_artifacts`. | Every other runpod script |
| `runpod_artifacts.py` | Post-run artifact reader: TSV parsing, manifests, MD reports. | `runner.py`, `e2e_matrix.py` |
| `runpod_matrix_plan.py` | Pure data model: `MatrixRow`, `CorpusMatrixPlan`, scope filtering. No RunPod calls. | `corpus_matrix.py`, `e2e_matrix.py`, tests |
| `runpod_matrix_remote.py` | Workflow patching executed **on-pod**. No RunPod calls. | Called from remote bash in `corpus_matrix.py` |
| `runpod_corpus_matrix.py` | Heavy matrix runner (~890 lines). Builds remote script from plan; calls `run_pod_detached`. | Direct execution |
| `runpod_model_matrix.py` | Model-level matrix runner (140 lines). Calls `run_pod` (synchronous). | Direct execution |
| `runpod_e2e_matrix.py` | E2E regeneratable-template matrix (681 lines). Has own argparse, dry-run, cost est. Duplicates constants from `runner.py`. | Direct execution |
| `runpod_validate.py` | Corpus smoke test (50 lines). Calls `run_pod_detached`. Name implies validation of RunPod infra — it's actually a smoke test. | Direct execution |

### Coherence Assessment

**Three natural layers exist** but are flattened into one namespace:

1. **Core runtime** — `runner.py`, `artifacts.py` (thick shim + post-processing)
2. **Library modules** — `matrix_plan.py`, `matrix_remote.py` (no RunPod calls, reusable)
3. **Entry-point orchestrators** — `corpus_matrix.py`, `model_matrix.py`, `e2e_matrix.py`, `validate.py`

**Issues:**

- **Mixed abstractions**: `matrix_plan.py` and `matrix_remote.py` are named `runpod_*` but never call RunPod. They're library modules that happen to serve RunPod orchestrators.
- **Name misleading**: `runpod_validate.py` validates the *corpus*, not RunPod infrastructure. `runpod_matrix_remote.py` is on-pod code, not remote orchestration.
- **Constant duplication**: `runpod_e2e_matrix.py` re-declares `DEFAULT_UPLOAD_EXCLUDES`, `REMOTE_ROOT`, `ROOT` to avoid importing `runner.py` during dry-run — a design smell.
- **Import fragility**: Cross-imports via `scripts.runpod_*` dotted paths are tight-coupled; moving files breaks everything.

### Risks

1. **Flat namespace clutter**: 8 `runpod_*` files in `scripts/` will grow. No structural hint for newcomers about which are entry points vs. libraries.
2. **`runpod_e2e_matrix.py` version drift**: Duplicated constants (`DEFAULT_UPLOAD_EXCLUDES`, `REMOTE_ROOT`) will silently diverge from `runner.py`'s originals.
3. **Refactoring cost highest in `corpus_matrix.py`** (~890 lines, 1 inline bash script). Splitting this is high effort; better to leave structurally alone for now.
4. **`runpod_validate.py` misnamed**: Someone looking for "validate" will check this first; it's actually a smoke-test runner.

### Safe Actions (lower risk, high value)

1. **Create `scripts/runpod/` subdirectory**. Move **only** the library modules (no entry-point breakage, easiest revert):
   - `runpod_matrix_plan.py` → `runpod/matrix_plan.py`
   - `runpod_matrix_remote.py` → `runpod/matrix_remote.py`

2. **Re-export from top-level stubs** for backward compatibility:
   ```python
   # scripts/runpod_matrix_plan.py → becomes stub:
   from scripts.runpod.matrix_plan import *  # noqa
   ```

3. **Rename `runpod_validate.py` → `runpod_smoke_corpus.py`** (or alias via CLI), since it runs a managed smoke test not infrastructure validation.

4. **Deduplicate `runpod_e2e_matrix.py` constants**: import `DEFAULT_UPLOAD_EXCLUDES`, `REMOTE_ROOT`, `ROOT` directly from `runner.py` even in dry-run mode (the import is cheap — only `sys.path` manipulation and env-bridging).

### Higher-Risk Changes (defer)

- Moving `runner.py` or `artifacts.py` under `runpod/` — deepest import tree, touches tests, all matrix scripts, and the test suite.
- Splitting `corpus_matrix.py`'s inline bash into a separate file — 890 lines with delicate escape/quote handling.
- The test files (`test_runpod_matrix.py`, `test_runpod_runner.py`, `test_conftest_runpod_markers.py`, `smoke/test_layer2_runpod_*.py`, `smoke/_runpod_helpers.py`) would all need import updates — coordinate with any move.
