## Classification & Recommendations

### 1. `vibecomfy_debug.py` (14 lines) → **DELETE**

A back-compat shim that does nothing but delegate to `vibecomfy debug`. The CLI command already exists. Keeping the script is dead weight and confuses discoverability. Remove it; anyone still calling it gets a clear `module not found` that points them to the real entry point.

### 2. `generate_agent_contract_js.py` (271 lines) → **STAY in `scripts/`**

Build-time code generator — reads Python source of truth from `contracts.py` and emits deterministic JS. This is the kind of one-shot "run once per contract change" script that belongs flat in `scripts/`. It's not a daily dev tool, it's a release step. No change needed.

### 3. `run_local_agent_comfy.sh` (77 lines) → **STAY in `scripts/`**

Developer launch script with environment wiring (symlinks, PYTHONPATH, megaplan fallback). Shell scripts don't become CLI commands naturally. It's fine here.

### 4. `sync_agent_skill.py` (118 lines) → **MOVE to `scripts/maintenance/`**

Installs the canonical `CLAUDE.md` into agent skill directories (`.claude/skills/`, `agents/openai.yaml`). It's a configuration sync utility, not something a human runs frequently. The `scripts/maintenance/` subdirectory already exists for this class of script.

### 5. `_agent_edit_prompt_dump.py` (211 lines) → **MOVE to `scripts/maintenance/`**

Underscore prefix already signals "internal." Dumps LLM prompts for debugging agent-edit behavior during development. Belongs with the other maintenance/utility scripts.

### 6. `agentic_success_rate.py` (411 lines) → **MAKE a CLI command**

This is the heavyweight: an evaluation harness that calls LLM APIs, validates task completion, manages budget, and produces structured results. At 411 lines with dry-run mode, dataclass config, and four distinct task categories, it's matured beyond a loose script. Register it as `vibecomfy eval` or `vibecomfy benchmark` under `vibecomfy/commands/`. This gives users discoverable help text, avoids `python scripts/...`, and lets CI invoke it via `vibecomfy eval --dry-run` instead of fragile path references.
