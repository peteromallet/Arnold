## 3. Should Docs Link to These Indexes?

**Yes.** Several docs already reference scripts/tools by path (e.g., `docs/templates/adding_templates_models.md`, `docs/migration/v25_to_v26.md`, `docs/megaplan_chains/excellence_epic/handoff-m1.md`). A canonical README in each directory would let those docs link via `see scripts/README.md` instead of duplicating invocation details inline — reducing staleness.

## 4. Generated-Command Reference Standardization

The invocation style is **inconsistent across docs**:

| Pattern | Used for | Risk |
|---------|----------|------|
| `python scripts/foo.py` | Plain direct calls | Works if cwd is repo root |
| `uv run python scripts/foo.py` | RunPod corpus matrix calls | More portable |
| `python -m tools.foo` | All tools (consistent ✓) | Good — keep this |
| `.venv/bin/python scripts/foo.py` | CI steps | CI-specific |

**Recommendation:** Standardize docs to use `uv run python scripts/<name>.py` for scripts and `python -m tools.<name>` for tools. Update existing doc references accordingly. The `scripts/README.md` should canonically document the preferred invocation pattern so all docs can reference it.
