## 1. Simplest rule for `scripts/`

**"Run by path, may need `sys.path` hacks, may leave artifacts."**  
Ad-hoc/operational code invoked as `python scripts/name.py` (not `python -m`). Includes RunPod deployment, debugging probes, one-off migrations, shell wrappers. Private library modules shared within the scripts package (like `runpod_artifacts.py`) are fine. Module-level `sys.path.insert` for direct invocation is the tell.

---

## 2. Simplest rule for `tools/`

**"Importable module with a CLI gate, invoked `python -m tools.name`."**  
The developer toolbelt — CI gates, validation gates, metadata refresh, template conversion, code generation. These have `if __name__ == "__main__"` with `argparse`, are tested via `from tools.name import ...`, and never need path hacks for normal use.
