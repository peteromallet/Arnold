## Audit 07: Blocks, Patches, Ops, Router — Findings Report

### 1. Organization Consistency

| Surface | Pattern | Public API | Private | Plugin Extension |
|---|---|---|---|---|
| **blocks/** | Domain modules with `@block`-decorated functions | `__init__.py` exports decorator, protocol, `Handles` | `_utils.py` (correct) | External `register_block()` via `extras.py` |
| **patches/** | Factories returning `Patch` + singleton `Patch` objects | `__init__.py` lazy re-exports via `__getattr__` | `types.py` internals, `requirements.py` utility | External `register(patch)` via registry |
| **ops/** | Public dispatch + private `_impl` + registry | `__init__.py` re-exports; modules support `__getattr__` for plugins | `_namespace.py`, `_common.py` | External `register_op()` via registry |
| **router/** | Pair of `_core.py` + `_rules.py` behind `__init__.py` | `__init__.py` re-exports `pick`, `RouterResult`, `Rule`, etc. | Both files correctly prefixed | External `register_route()` via `extras.py` → `router_rules` |

**Verdict:** The four surfaces follow a **consistent layering**: private `_` helpers → public domain modules → `__init__.py` re-exports → plugin API in `extras.py`. The naming conventions and decorator/registry patterns are uniform. **No structural inconsistency.**

8. **The top-level `router.py` deletion** — verify no CI scripts or agent workflows reference it before removing.

**Summary: 2 immediate deletions, 1 merge decision, 5 stability deferrals.**
