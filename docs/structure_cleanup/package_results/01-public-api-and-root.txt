These are implementation details, **not** public API. They fall into categories:

#### a) `_`-prefixed helpers (conventionally private)
| File | Role |

| `_compile/` (5 files) | Core compilation engine: `_helpers.py`, `_graph.py`, `_resolve.py`, `_widgets.py` |

| `_graph_utils.py` | **Shim** — `from vibecomfy._compile._graph import *` re-export |

| `_widget_aliases.py` | **Module swap shim** — `sys.modules[__name__] = _compile._widgets` |
| `_workflow_helpers.py` | **Shim** — `from vibecomfy._compile._helpers import *` re-export |

**Issue**: The four shim files (`_graph_utils.py`, `_helper_resolve.py`, `_widget_aliases.py`, `_workflow_helpers.py`) exist solely because `vibecomfy.workflow` and `vibecomfy.ir.workflow` used to import from different locations. They're fragile compatibility bridges — but they're imported by `workflow.py` (line 8-10) and `ir/workflow.py` (line 9-11), so removing them would break core types.

### 4. What Absolutely Must Not Move

1. **`vibecomfy.workflow`** — The single most imported module. Hundreds of test files, all docs/cookbook examples, recipes, tools, and internal modules import from it. Moving anything in `__all__` would break everything.

2. **`vibecomfy.templates`** — Documented in migration guides and every cookbook example. Used from `docs/cookbook/*.py`, `docs/migration/*.md`. Any rename would invalidate published docs.

3. **`vibecomfy.cli`** (`build_parser`, `main`) — The CLI entry point, tested by `test_cli_misc.py` and `test_doctor_models.py`.

4. **`vibecomfy.testing`** — Documented in `docs/testing/user_code.md`, used in `docs/testing/user_code_examples/`.

5. **`vibecomfy.errors`** (`VibeComfyError` et al.) — Imported by `workflow.py` transitively, used in migration docs.

6. **`vibecomfy.handles`** (`Handle`) — Core type, imported by `workflow.py` and used throughout tests.

7. **`vibecomfy.utils`** (`find_repo_root`) — Imported from many internal modules and tests.

8. **`vibecomfy.runtime`** (subpackage) — The `run_*` lazy exports in `__init__.py` and direct imports from recipes (`run_embedded_sync`).

9. **`vibecomfy.cli_loader`** — Used in recipes, docs examples, scripts.

10. **`vibecomfy.ops`**, **`vibecomfy.blocks`**, **`vibecomfy.patches`** — Documented public subpackages imported by tests, recipes, agentic scripts.
