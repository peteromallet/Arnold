### 2. Stale Caches / Snapshots That Should Move or Be Ignored

#### 🔴 CRITICAL: 3 stale JSON snapshot files in `object_info/`

Three large JSON files live directly in the source-code `object_info/` subpackage directory:

- `vibecomfy/porting/object_info/ComfyUI-KJNodes@runpod-snapshot.json` (702 KB)
- `vibecomfy/porting/object_info/ComfyUI-LTXVideo@runpod-snapshot.json`
- `vibecomfy/porting/object_info/rgthree-comfy@runpod-snapshot.json`

These are **exact duplicates** of the files already in `cache/object_info/`. The consumer (`consume.py`) uses `CACHE_DIR = Path(__file__).parent.parent / "cache" / "object_info"` — it **never reads** from the local `object_info/` directory. These 3 files are **dead code** that should be **deleted** (or moved into `cache/object_info/` if they're not there — but they are already there).

#### 🟡 Generated cache directory: `cache/`

The `cache/` directory contains:
- `class_inventory.json` (267 KB — repo-checked-in generated artifact)
- `object_info/index.json` (105 KB — class_type → filename map)
- `object_info/provenance.json` (5 KB — pack provenance metadata)
- `object_info/` + 23 per-pack JSON snapshot files

This is a generated build artifact. It **should not be committed** long-term (or should have a regenerate-on-demand story). If it IS intentionally committed (to avoid a ComfyUI dependency at build time), then at minimum a **`.gitignore`** is missing in `cache/` — the directory is currently naked with no ignore rules for `__pycache__` or any other transient artifacts.

#### 🟡 No `.gitignore` anywhere in `porting/`

There is no `.gitignore` at any level of the porting subtree. All `__pycache__/` dirs (7 subdirectories) are version-controlled artifacts. While this is harmless (`.pyc` files are typically committed unless excluded), it means any transient file dropped in `porting/` could accidentally be committed.

### 4. Imports / Backward-Compat Shims That Prevent Moving Modules

#### 🟡 Backward-compat re-export shims (flat → subpackage)

Several **flat root-level `.py` modules** are thin re-export wrappers that delegate to subpackage modules:

| Flat file | Delegates to | Pattern |
|-----------|-------------|---------|
| `edit_lint.py` | `vibecomfy.porting.edit.lint` | `from edit.lint import *` |
| `edit_ledger.py` | `vibecomfy.porting.edit.ledger` | `from edit.ledger import *` |
| `edit_session.py` | `vibecomfy.porting.edit.session` | `from edit.session import *` |
| `edit_types.py` | `vibecomfy.porting.edit.types` | `from edit.types import *` |
| `widget_aliases.py` | `vibecomfy._compile._widgets` | Direct imports from `_compile._widgets` |
| `widget_schema.py` | `vibecomfy._compile._widgets` | Direct imports from `_compile._widgets` |

These exist to preserve backwards compatibility while the code was being reorganized. **Before you can move/rename the subpackages, you must audit all external callers** across the codebase for `from vibecomfy.porting.edit_lint import ...` or `from vibecomfy.porting.edit_session import ...` style imports and update them to the canonical subpackage path.

Ways to find these:
```bash
# Search for flat-file imports that go through the shim
rg "vibecomfy\.porting\.edit_lint\b" --type py
rg "vibecomfy\.porting\.edit_ledger\b" --type py
rg "vibecomfy\.porting\.edit_session\b" --type py
rg "vibecomfy\.porting\.edit_types\b" --type py
rg "vibecomfy\.porting\.widget_aliases\b" --type py
rg "vibecomfy\.porting\.widget_schema\b" --type py
```

#### 🟡 Circular-dependency risk zones (from `__init__.py` cross-imports)

The top-level `porting/__init__.py` imports from:
- `emit/emitter.py` (EmissionDiagnostic, etc.)
- `edit/session.py` (EditSession, etc.)
- `edit/types.py` (FieldChange)
- `parity.py` (class_type_counter, etc.)
- `provenance.py` (various)
- `report.py` (various)
- `identity/codec.py` (build_reverse_map, etc.)
- `strict_ready.py` (various)

These create **dependencies** that chain through the package graph. For example, `porting/__init__.py` → `emit/emitter.py` → `emit/node_kwargs.py` → `object_info/consume.py` → `cache/object_info/`. Any attempt to move `emit/` or `edit/` out of `porting/` would require resolving these import chains.

#### 🟡 Internal cross-subpackage imports

- `object_info/serialize.py` imports `from vibecomfy.porting.object_info.consume import CACHE_DIR, INDEX_PATH, _WIDGET_LIKE_TYPES` — prevents moving `consume` without updating `serialize`
- `edit_session_ir_utils.py` imports from `edit/ops.py`, `widgets/schema.py` — couples `edit_session_*` flat files to subpackages
- `lowering.py` imports from `canonical_coords.py`, `uid.py` — circular-safe
- `latency.py` imports from `layout/` and `ui_emitter.py` — crossing subpackage boundaries
