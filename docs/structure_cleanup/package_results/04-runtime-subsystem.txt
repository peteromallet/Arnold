### Q3: Are module names/layout coherent enough for navigation?

**Issues found:**

1. **`config.py` vs `session.py` duplication.** `config.py` (157 lines) defines `SessionConfig` and `apply_memory_profile_override`. `session.py` (1461 lines) defines *another* `SessionConfig` dataclass (lines 150-161) with the same fields plus an extra `strict_drift` field. The two versions have drifted:
   - `config.py` has: `memory_profile`, `vram_policy`, `reserve_vram_gb`, `cache_policy`, `disable_smart_memory`, `warm_policy`, `auto_flush_vram_threshold_gb`, `port`, `extra`
   - `session.py` has all of those plus `strict_drift: bool = False`
   - `session.py` imports `_partition_comfy_config` from `config.py` indirectly but defines its own `SessionConfig.from_dict()` and `from_workflow_metadata()`. The `config.py` version appears unused except as a partial import source.
   
   **Action:** `config.py` should be consolidated into `session.py` (or re-export `session.SessionConfig`). The duplicate `SessionConfig` is a maintenance hazard.

2. **`server_process.py` vs `session.py` overlap.** `server_process.py` (57 lines) defines `_spawn_comfy_server`. `session.py` (lines 1317-1351) defines the *same* function with identical signature. `server_process.py`'s version is used by `server.py`; `session.py`'s version adds logging and env setup. This is a copy-paste drift risk.

3. **`watchdog_runtime.py` is a thin wrapper** (70 lines) that just calls `watchdog.py` functions. It exists to break a circular import (`session.py` → `watchdog.py` → `session.py`). The functions are re-exported back into `session.py`. This is fine as an import-cycle break but the naming `watchdog_runtime.py` vs `watchdog.py` is confusing — they feel like the same module.

4. **`run.py` vs `session.py`.** `run.py` calls into many `session.py` internals (prefixed with `_`). The line between "top-level run orchestration" and "session internals" is blurry — `run.py` duplicates the attempt bundle, drift enforcement, queue, and watchdog logic that also lives in `session.py`'s `_run_untracked`. Some consolidation could reduce duplication.

5. **`model_policy.py` imports `_model_assets_from_workflow` from `session.py`** (line 74), creating a back-import. This is the opposite direction of what the layout suggests (`model_policy` → `session` when it logically should be `session` → `model_policy`).

**Coherence score:** 7/10. The module boundaries are sensible but there are 3-4 duplication/drift points that will cause maintenance bugs.

### Q4: What should be documented instead of moved?

**Document, don't move:**

1. **`eval.py` + `preview_types.py`** — tightly coupled to the `vibecomfy runtime eval-node` CLI. Document the preview injection map (`preview_types.py`) to explain why LATENT → `VAELoader` + `VAEDecode` + `PreviewImage` is hardcoded, and why SD1 is the fallback. Move would break CLI contract.

2. **`attempt.py`** — the "write attempt.json before every queue boundary" invariant is a correctness rule. Document why it must happen before queue (not after) — it's for crash-recovery traceability. No move needed; it's a single-purpose module.

3. **`drift.py`** — per-process cache `((lockfile_mtime, workflow_id) → dict)`. Document that the cache expiry is coarse (file-mtime), so it won't detect manual edits to lockfiles between runs in the same process. This is a known tradeoff vs re-scanning every call. No move needed.

4. **`model_policy.py`** — the mode hierarchy (`embedded` vs `managed_local_server` vs `explicit_remote_server_*`). This is the closest thing to a "runtime deployment model" doc. Convert the RuntimeModelMode literal into an ADR-style doc rather than moving into a different module.

5. **`comfy_nodes/agent/runtime.py`** module docstring (lines 1-30) is already excellent self-documentation. No move needed.

**Move (defer to MP-5 or later):**

| Source | Target | Rationale |
|---|---|---|
| `config.py` | → merge into `session.py` | Duplicate SessionConfig causes drift (strict_drift missing in one) |
| `server_process.py` | → merge into `session.py` | Duplicate `_spawn_comfy_server` |
| `watchdog_runtime.py` | → inline into `session.py` or rename to `_watchdog_helpers.py` | Confusing name mirroring `watchdog.py`; 70-line thin wrapper |
| `model_policy.py` back-import | → move `_model_assets_from_workflow` consumer into `session.py` | Reverse dependency direction |
