## 2. Is moving "tracked evidence files" safe?

**There's nothing to move.** `agentic/evidence/` is a *virtual prefix* in scenario YAML manifests and adapter logic — not a real directory holding tracked assets. The three files (`actions.jsonl`, `compiled_api.json`, `metadata.json`) are **runtime generated output** produced by builder actors, not canonical fixtures or examples.

The `evidence/` prefix is resolved by `adapter.py`'s `_candidate_evidence_paths()` and `_assessor_candidate_evidence_paths()` methods, which strip it to find the actual files in the staging directory. If you moved anything, you'd just need to update the prefix stripping logic.

---

## 3. Best destination if they should move

The `evidence/` prefix should be **removed entirely** from the scenario YAML files and adapter code. Replace it with a **flat `required_frozen_evidence`** list using bare filenames (e.g., `compiled_api.json` instead of `evidence/compiled_api.json`).

**Rationale:** The prefix is noise — the scenario runner already knows it's looking at evidence files. All evidence lives in a dedicated staging directory; the extra prefix adds a pointless indirection that confuses readers into thinking there's a `agentic/evidence/` tree. Dropping it simplifies `adapter.py` (remove both `_candidate_evidence_paths` methods entirely) and makes the YAML manifest self-documenting.
