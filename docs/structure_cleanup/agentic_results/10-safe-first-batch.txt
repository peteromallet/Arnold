## 3. Changes to explicitly defer

- **Renaming `actors.py` → `actors_m2/`** — breaks imports in `adapter.py` (lines 19–33) and `tests/test_agentic_adapter.py`. Zero behavioral gain for high churn.
- **Renaming/merging `actors_m4/` or `actors_m5/`** — `adapter.py` lines 34–35, `tests/test_agentic_golden_m4.py:25`, `tests/test_agentic_golden_m5.py:27` all import by exact subpackage path.
- **Moving `scenarios/` or `briefs/`** — path-dependent defaults in `runner.py` (lines 82–91) and golden tests.
- **Renaming scenario YAMLs from underscore to hyphen convention** — the runner's `load_scenario` call already normalizes via the YAML `name` field, so it's safe but cosmetic churn.
- **Removing `build_positive_structural_chain` / `build_recovery_structural_chain` fallback paths** — they are tested directly in `test_agentic_structural.py` and correspond to real scenario YAMLs (`chaining_positive.yaml`, `image_to_video_chain_recovery.yaml`).
- **Changing `__all__` exports** in `actors_m4/__init__.py` or `actors_m5/__init__.py` — `adapter.py` imports `_M4_BUILDERS` and `_M5_BUILDERS` by name.

## 4. Rationale

The three recommended changes are purely mechanical — they fix git hygiene (tracked `.pyc` bloat), align the package's docstring with its actual contents, and eliminate redundant O(n) lookup chains without altering any observable behavior. None of them change imports, API surfaces, or evidence output paths, so all existing tests pass verbatim and the Sisypy harness continues to classify success/failure identically. Deferring the higher-risk renames avoids breaking the six import sites across `adapter.py` and three test files that depend on exact subpackage paths.
