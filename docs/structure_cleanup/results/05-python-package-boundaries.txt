## 2. Safe Moves at Top Level

- **Move `_debug_*.py` ‚Üí `scripts/`**: Already git-ignored at root. To un-ignore, add them to `scripts/` and remove the `.gitignore` line. Update `_regen_templates.py` and `_fix_t6.py` similarly ‚Äî they are one-shot maintenance scripts.
- **Move `recipes/` ‚Üí `vibecomfy/recipes/` or keep**: Recipes reference `vibecomfy` but are not in the wheel. They're standalone example files (no `__init__.py`). Minimal risk.
- **Move `agentic/briefs/` ‚Üí `docs/`**: Markdown files, no import risk.
- **Move `agentic/actors_m4/`, `actors_m5/` ‚Üí `scripts/`**: These are generated actor scripts. `actors.py` already imports `vibecomfy`; safe to relocate if you update import paths.

## 3. Risky Moves That Should Wait

- **Renaming or restructuring `vibecomfy/`**: Will break:
  - CLI entrypoint (`vibecomfy = "vibecomfy.cli:main"`)
  - ComfyUI custom_nodes entrypoint (`vibecomfy = "vibecomfy.comfy_nodes"`)
  - Pytest plugin (`vibecomfy = "vibecomfy.testing._pytest_plugin"`)
  - `[tool.hatch.build.targets.wheel] packages = ["vibecomfy"]`
  - All internal relative imports
  - All `from vibecomfy.xxx import yyy` across tests, scripts, agentic, recipes
- **Moving `tests/` ‚Üí `vibecomfy/tests/`**: Breaks `testpaths = ["tests"]` and `pytest11` plugin discovery.
- **Changing `scripts/` or `tools/` to shipped packages**: Must update `pyproject.toml` `packages` list and audit for hatch artifacts.
- **Deleting `vendor/`**: Live runtime dependency for ComfyUI-interop tests.
