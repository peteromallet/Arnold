- **`SECURITY_AUDIT_NOTES.md`** — an audit document at root → should be `docs/security_audit_notes.md`.
- **`finalize.json`** — a JSON task plan at root → belongs in `docs/plans/finalize.json` or similar.
- **`node_index.json`, `template_index.json`, `asset_manifest.json`** — generated indexes at root → could sit in `out/` (regenerable) or a `data/` directory; root is noisy.
- **`.agents/skills/desloppify/SKILL.md`** — semantically agent config, not clearly belonging to `agentic/` or anywhere exposed.

## 3. Minimal Migration Path (No Import/Test Breakage)

**Phase 1 (zero-risk moves):** Move root debug scripts and audit notes, leaving shims.
- `_debug_json.py` → `scripts/_debug_json.py` with a symlink or import shim at root (debug scripts are run directly, so a symlink suffices).
- Move `SECURITY_AUDIT_NOTES.md` → `docs/security/audit_notes.md` (no imports to break).
- Move `finalize.json` → `docs/plans/finalize.json`.

**Phase 2 (generated indexes):** Add `.gitignore` entries for the root JSON files and regenerate into `out/` or a new `data/` folder. Update any CI/scripts that reference the root paths.

**Phase 3 (`.agents/`):** Either rename to `.agent_skills/` or merge into `agentic/runtime_skills/` if the harness should own it. Since `.agents/` is a dotfile (hidden), the naming confusion with `agentic/` is less urgent.

## 4. Naming Conflicts Worth Resolving

| Conflict | Diagnosis | Recommendation |
|---|---|---|
| **`agentic/` vs `.agents/`** | `agentic/` = test harness; `.agents/` = runtime skill definitions. Both "agent" but different concerns. | Accept `.agents/` as hidden config. No urgent rename needed, but document the distinction in `agentic/README.md`. |
| **`artifacts/` vs `out/`** | `artifacts/` = checked-in milestone evidence; `out/` = generated runtime output. The names don't communicate this polarity. | Rename `artifacts/` → `evidence/` or `milestones/` to signal it's curated, not generated. Low priority. |
| **`ready_templates/` vs `recipes/`** | Both are user-facing workflow code. `ready_templates/` = single-workflow; `recipes/` = compound/compositional. Boundary is blurry when a template chains inside its Python. | Consider merging `recipes/` into `ready_templates/compositions/` long-term, or rename `recipes/` → `compositions/`. Not urgent. |
| **`workflow_corpus/` vs `ready_templates/`** | One is JSON (serialized graphs), the other is Python (generator code). Complementary but different representations. | Names are clear enough, but consider nesting `workflow_corpus/` under `ready_templates/` as `ready_templates/.corpus/` if it's purely derived. |
