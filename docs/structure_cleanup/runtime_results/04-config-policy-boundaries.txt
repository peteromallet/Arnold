### Decision 1: `vibecomfy/runtime/policy.py` — DELETE (dead duplicate)

**Evidence:**
- `policy.py` defines `_maybe_flush_for_policy()` and `_free_vram_gb()`.
- **Zero** modules import from `vibecomfy.runtime.policy` or from `.policy` relative import (verified via ripgrep).
- The **exact same two functions** are duplicated at `vibecomfy/runtime/session.py` lines 1358–1380, and those are the ones actually called (lines 341, 550 in `session.py`).
- The `policy.py` versions are **dead code** — likely a leftover from an earlier refactor where these were factored out then re-inlined.

**Action:** Delete `vibecomfy/runtime/policy.py`. No import references to update. Confirm with `git log --follow` to see if it was ever called externally.

### Decision 2: `vibecomfy/metadata.py` vs `vibecomfy/runtime/metadata.py` — NOT duplicates

The doc in `docs/structure_cleanup/package_results/06-ir-compile-schema.txt` already flags this module for possible relocation. This is a **low-priority rename opportunity**, not a bug.
