## Ranked Findings

### P0 — BUG: `drift.py` uses `hashlib` without importing it

**File:** `vibecomfy/runtime/drift.py`, line 193  
`hashlib.sha256(source_path.read_bytes()).hexdigest()` will raise `NameError` at runtime in the source checks branch. `hashlib` is not in the module-level imports (lines 13–21) and there is no late import. The branch is exercised whenever a lockfile entry has `source_sha256` entries. The other three `runtime` modules that use `hashlib` (`session.py`, `metadata.py`, `attempt.py`) all import it correctly — this was simply missed.

**Fix:** Add `import hashlib` to the imports block at the top of `drift.py`.

---

### P1 — DEAD MODULE: `watchdog_runtime.py` is entirely orphaned

**File:** `vibecomfy/runtime/watchdog_runtime.py` (70 lines, two functions)  
**Reality:** Zero imports — nothing in the codebase references `vibecomfy.runtime.watchdog_runtime`. `session.py` has its own **verbatim copies** of `_start_watchdog` (session.py:1395) and `_finalize_watchdog` (session.py:1423), identical modulo whitespace to the ones in `watchdog_runtime.py`. The `watchdog_runtime.py` module was extracted as an import-cycle break during the `session.py` decomposition (per `docs/structure_cleanup/package_results/04-runtime-subsystem.txt`), but the decomposition was never completed — `session.py` kept its private copies and the new file was never wired in.

**Action:** Either delete `watchdog_runtime.py` (it's dead code) or finish the extraction: remove the dups from `session.py` and import from `watchdog_runtime.py` instead. Given the module already exists, the latter is ~20 minutes of work and reduces `session.py`.

### P2 — DUPLICATE: `fingerprint.py` and `session.py` share identical code

**File:** `vibecomfy/runtime/fingerprint.py` (29 lines) vs `session.py` lines 45–46 + 1442–1461  
Both define `OVERRIDES_INCLUDE: set[str] = set()`, `OVERRIDES_EXCLUDE: set[str] = set()`, and `model_fingerprint(api_dict)`. The functions are byte-identical. `session.py` does **not** import from `fingerprint.py` — it carries its own copy. `fingerprint.py` itself has zero importers outside its own file:

```
$ grep -r 'fingerprint' vibecomfy/ --include='*.py'
vibecomfy/runtime/fingerprint.py  (self)
```

(Plus test references and docs.) So `fingerprint.py` is also de facto dead as a standalone module.

**Action:** Pick one canonical location. If the fingerprint is meant to be a shared utility, `session.py` should import from `fingerprint.py` and drop its copy. If it's only used in `session.py`, delete `fingerprint.py`. Either way, eliminate the duplication.

### P4 — `watchdog.py` and `watchdog_runtime.py` serve distinct conceptual layers

They are not duplicates. `watchdog.py` (641 lines) is the full execution observer: `Watchdog` class, WebSocket loop, VRAM polling, diagnosis heuristics, report serialization. `watchdog_runtime.py` (70 lines) is a **lifecycle shim** that constructs/stops the watchdog and writes the report. The confusion is purely naming: `watchdog_runtime.py` sounds like "the real watchdog" when it's actually just a two-function helper. Renaming to `_watchdog_lifecycle.py` (as the docs already recommend) would clarify this immediately.
