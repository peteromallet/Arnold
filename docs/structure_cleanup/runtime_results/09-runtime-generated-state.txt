## Runtime Package — Generated-State Audit

### 1. Static file inventory under `vibecomfy/runtime/`

**No JSON, log, cache, snapshot, or data files exist directly inside `vibecomfy/runtime/`.** The directory contains 30 source modules only — all `.py` files with one `README.md`. No generated artifacts are checked into the package tree.

### 2. Files created at runtime (under repo root)

All runtime file creation writes **outside** `vibecomfy/runtime/` into the repo root or OS temp dirs.

#### A. `out/runs/<run_id>/` — per‑run state directory
Created by both `run.py` and `session.py` (`EmbeddedSession` and `ServerSession`). Every run gets:

| File | Writer | Trigger |
|---|---|---|
| `out/runs/<run_id>/attempt.json` | `attempt.py:write_attempt_json()` (via `atomic_write_json`) | Called before every queue boundary in `run.py:82`, `session.py:364` (embedded), `session.py:566` (server) |
| `out/runs/<run_id>/metadata.json` | `atomic_write_json()` directly in `run.py:113`, `session.py:404/608` | After queue completes |
| `out/runs/<run_id>/watchdog.json` | `watchdog.py:write_report()` → one‑line header + pretty‑printed JSON body | Via `_finalize_watchdog()` in `session.py:1435` (also `watchdog_runtime.py:66`) |
| `out/runs/<run_id>/comfy.log` | `server_process.py:_spawn_comfy_server()` (ab‑write buffered) | Managed server subprocess stdout/stderr |
| `out/runs/<run_id>/embedded.log` | `session.py:347` | Embedded run log |

**`eval_node()`** (`eval/prompt.py:51`) also creates `out/runs/eval-node-<timestamp>/comfy.log` but writes **no** `attempt.json`, `metadata.json`, or `watchdog.json`.

**No cleanup exists.** Directories pile up in `out/runs/` indefinitely. This is the primary cleanup candidate.

#### B. `out/sessions/<id>/` — session persistence layer
Created by `commands/session.py` (not in runtime/ but consumes runtime internals):

| File | Content |
|---|---|
| `out/sessions/<id>/pid` | OS PID of the managed server process |
| `out/sessions/<id>/url` | Server URL |
| `out/sessions/<id>/config.json` | Serialised session config |
| `out/sessions/<id>/source_revision` | Git HEAD for stale‑session detection |
| `out/sessions/<id>/server_argv.json` | Server CLI argv |
| `out/sessions/<id>/daemon_argv.json` | Daemon spawn argv (if daemon mode) |

`session.py:active_session_metadata()` reads these; `_cleanup_session_files()` removes them when a session is stale/dead. However, **no process‑exit or crash‑safe cleanup** exists — abandoned sessions can leave debris.

#### C. Temp files (self‑cleaning)
- **`_local_library_yaml.py`**: Module‑level `tempfile.TemporaryDirectory(prefix="vibecomfy_lib_yaml_")` registered with `atexit`. Re‑renders `vibecomfy_library.yaml` inside the temp dir on every `acquire_library_yaml()` call. Cleanup is reliable.
- **`ensure_env.py`**: `tempfile.NamedTemporaryFile(prefix="vibecomfy-ensure-", suffix=".object_info.json", delete=False)` → explicitly `unlink()`ed in the same function's `finally` block (lines 879–880). Cleanup is reliable.

### 3. Runtime modules that **read** but don't create repo‑root files

- `attempt.py:_read_lockfile_snapshot()` — reads `custom_nodes.lock` (optional, not created by runtime)
- `session.py:_embedded_configuration_for_session()` — reads `extra_model_paths.yaml` at CWD if present (optional, user‑provided)

### 4. README documentation assessment

`vibecomfy/runtime/README.md` is **8 lines** and describes only *what* the package contains — it says nothing about:

- The `out/runs/` directory tree or its lifecycle
- The `out/sessions/` persistence layer
- Generated state boundaries or cleanup policy
- Temp file behaviour (`_local_library_yaml`, `ensure_env` temp files)

This is a **documentation gap**. Callers have no documented contract about which files are ephemeral vs. persistent, or how to clean them.

### 5. Cleanup candidates

| Priority | Item | Action |
|---|---|---|
| **High** | `out/runs/` with no cleanup policy | Add a cleanup TTL (e.g. keep last N runs, or `VIBECOMFY_RUNS_TTL_HOURS` env var, or a `vibecomfy runs prune` command) |
| **Medium** | `out/sessions/` stale session debris | The `_cleanup_session_files()` path only fires on re‑probe. A daemon‑side exit handler or eventual‑consistency cleanup would help. |
| **Low** | `README.md` silence | Document file‑creation boundaries, temp file lifecycle, and cleanup mechanism |
| **Low** | `eval/prompt.py` bypasses run‑dir conventions | `eval_node()` creates `out/runs/eval-node-*` without `attempt.json`/`metadata.json` — inconsistent with the rest of the package |

### 6. Notable: no `out/` or `sessions/` directories exist yet

At audit time, `out/runs/`, `out/sessions/`, and all runtime‑generated log/JSON files are **absent** — the package has not been executed in this working copy. The `external_workflow_index.json` at repo root is a checked‑in index, not runtime‑created.
