## Runtime Eval Split Audit

### Diagnosis: Unresolved Duplication

The flat files (`runtime/eval.py`, `eval_plan.py`, `eval_prompt.py`, `preview_types.py`) are **stale legacy copies** of the `runtime/eval/` package. The package is the canonical version. The flat files persist only because 3 external import sites still target them.

### Decision: Delete All 4 Flat Files

**1. `runtime/eval.py`** → **Delete safe.** `queue_eval_subgraph` is dead code (raises `NotImplementedError`, zero callers). `compile_eval_subgraph` is re-exported from `vibecomfy/runtime/eval/__init__.py`.

**2. `runtime/eval_plan.py`** → **Delete safe.** Exact subset of what the package provides.

**3. `runtime/eval_prompt.py`** → **Delete safe.** Exact subset. Internal import `from vibecomfy.runtime.eval_plan` is rewritten in the package as `from .plan`.

**4. `runtime/preview_types.py`** → **Delete safe.** Byte-identical to the package version.

### One Concern: Shadowing Risk

Currently `from vibecomfy.runtime.eval import compile_eval_subgraph` is ambiguous (flat file `eval.py` vs package `eval/__init__.py`). Deleting the flat file removes the ambiguity — definitely resolves to the package. **Do not merge without deleting the flat files first**, or the shadowing persists silently.
