### 3. Hidden Risk of Duplicate Implementations

The **16 text_encoder modules** each import from `transformers_compat.py`. The broken file masks a real risk: if you delete it and switch to direct `from transformers import ...`, you lose the graceful fallback to non-Fast tokenizers. However:

- Modern `transformers` has shipped `*TokenizerFast` for **every** tokenizer type since v4.20.0 (Dec 2022).

- The **real hidden risk** is leaving this broken file in place: developers see `transformers_compat` imported and assume it works. It doesn't. That's worse than deletion.
