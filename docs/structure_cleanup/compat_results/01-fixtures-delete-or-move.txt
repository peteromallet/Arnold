## Compatibility Layer Audit 01: Fixtures Delete Or Move

### 1. Which file should own the implementation?

**`vibecomfy/fixtures.py`** — it's the real module. `vibecomfy/testing/_fixtures.py` is a byte-identical dead duplicate with zero importers anywhere in the codebase.

Crucially, `vibecomfy/testing/fixtures.py` (84 LOC) is a **completely separate file** — it provides pytest test helpers (`vibecomfy_workflow_factory`, `dry_runtime`, etc.). That file is fine and should stay. The duplicate is `testing/_fixtures.py` (the smoke-fixture copy).

### 2. Is `vibecomfy/fixtures.py` a hard public CLI/API contract?

**Yes — it's a public CLI contract.** `python -m vibecomfy.fixtures` is invoked by:
- `scripts/runpod_corpus_matrix.py` (lines 130-132): `$PY -m vibecomfy.fixtures copy --target input`
- Documented as the bootstrap path in `docs/runtime/incompatibilities.md` and multiple planning docs

Python-side: `tests/test_fixtures.py` imports `from vibecomfy import fixtures` and calls `fixtures.available_fixtures()`, `fixtures.copy_smoke_fixtures()`, `fixtures._main()`. No other Python importers.

**Policy call**: per your "shim only for extreme public-contract" rule, this qualifies. The `-m vibecomfy.fixtures` invocation is a hard contract for the runpod matrix bootstrap shell script. The module-level API (`copy_smoke_fixtures`, etc.) is internal/test-only and could be moved.

### 3. Is deletion safe now? What cleanup still helps?

**`vibecomfy/testing/_fixtures.py`** — safe to delete immediately. Zero importers. It's a stale copy.

**`vibecomfy/fixtures.py`** — cannot be deleted without breaking the `-m` CLI contract. But it can be structurally improved:

**Recommended immediate action:**
1. **Delete** `vibecomfy/testing/_fixtures.py` — dead duplicate, safe.
2. **Keep** `vibecomfy/fixtures.py` as the sole implementation — simplest path, no shim needed.

**Optional improvement (lower priority):**
3. Move the 266 LOC of implementation into `vibecomfy/_fixtures_smoke.py` (a private module), then make `vibecomfy/fixtures.py` a thin shim that re-exports everything + preserves the `__main__` block. This mirrors the pattern used elsewhere and keeps the public contract surface minimal.

### 4. Exact tests/commands to verify

After deleting `testing/_fixtures.py`:

```bash
# 1) The smoke-fixture test suite still passes
python -m pytest tests/test_fixtures.py -x --no-header -q

# 2) The CLI entrypoint still works
python -m vibecomfy.fixtures list

# 3) Script import works
python -c "from vibecomfy.fixtures import copy_smoke_fixtures, available_fixtures; print(len(available_fixtures()))"

# 4) No remaining references to the deleted file
rg "testing._fixtures" --type py

# 5) Full test suite (optional)
python -m pytest tests/ -x --no-header -q --ignore=tests/known_failures.txt 2>/dev/null | tail -5
```

**No other call sites need updating** because `testing/_fixtures.py` was never imported by any Python code — it was only named in docs' dead-module inventories.
