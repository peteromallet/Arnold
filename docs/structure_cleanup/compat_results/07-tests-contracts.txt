### 2. Intentional public API vs. accidental compatibility

**Hard public contracts (keep shim or barrel):**

- **`vibecomfy/fixtures.py`** — `python -m vibecomfy.fixtures` is invoked by `scripts/runpod_corpus_matrix.py` and documented in `docs/`. The `__main__` entrypoint is a real deployment contract. The module is NOT a shim — it owns the implementation. `vibecomfy/testing/_fixtures.py` is the accidental duplicate (delete it).

- **`vibecomfy/schema` barrel** — Intentionally re-exports 17 public names (~34 test files rely on it). Already clean per audit 05. No action needed.

- **`vibecomfy/diagnostics` barrel** — Already a clean re-export of `findings.py` + `health.py`. Only 2 test files import from it. Already clean per audit 04. No action needed.

**Accidental/internal (may break but are not public API):**

- **`vibecomfy/commands/doctor.py`** — Tests import it directly as `doctor_cmd`, monkeypatch its internals (`_doctor_warnings`, `_cmd_doctor`, `_nodepack_lockfile_drift`), and call private helpers. These protect **internal command structure**, not public contracts. Expect churn when commands are reorganized.

- **`vibecomfy/_agent_edit_debug.py`** — Tests import the private module and monkeypatch its `_listener_pid`, `_fetch_runtime_status`. Purely internal.

- **`vibecomfy/cli.py`** — `build_parser` is the CLI entry point. Semi-public but expected to stay stable by design.

# No test needed — nothing imports it.

# Wave 3 (CLI commands — if commands are reorganized):

# Full confidence sweep:

**Bottom line:** Only `vibecomfy/testing/_fixtures.py` is safe to delete immediately (zero imports, byte-identical to `vibecomfy/fixtures.py`). The `vibecomfy/fixtures.py` file must stay as a `__main__`‑supporting module — it's a real public CLI contract, not a compatibility shim. The diagnostics and schema barrels are already clean.
