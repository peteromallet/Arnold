### Duplicate Functionality Found

**1. `_extract_rebaseline_recovery` — duplicated in two files (functionally identical)**

- **`contracts.py`** lines 828–855: `_extract_rebaseline_recovery(response)` — used by `_public_error_outcome_from_response` and `failure_outcome_payload`
- **`routes.py`** lines 147–177: `_extract_failure_recovery(response)` — used by `_validated_failure_response` and `_promote_accept_rebaseline_recovery`

Both walk the same nesting order: top-level → outcome → debug contexts, looking for `rebaseline_recovery` in `agent_failure_context.issues`. The `routes.py` version is named differently and lives in a different file but the logic is byte-identical.

**2. `_now()` timestamp utility — duplicated in two files**

- **`session.py`** line 102–103: `def _now() -> str: return time.strftime(...)`
- **`audit.py`** line 42–43: `def _now() -> str: return time.strftime(...)`

Identical implementation. Trivial but unnecessary duplication.

**3. `_extract_json_object` — near-identical in two files**

- **`provider.py`** lines 110–124: parses stripped text, handles markdown fences, raises `MalformedModelJSON`
- **`worker.py`** lines 48–68: same structure but additionally uses `raw_decode` for trailing-data resilience and raises generic JSON errors

These are *not* clean to unify because `worker.py` has the more robust `raw_decode` fallback and must be self-contained (subprocess). Acceptable as-is.

**4. Route normalization — overlapping logic in provider.py and runtime.py**

- **`provider.py`** lines 486–548 `_resolve_agent_route(route)`: maps route strings, adds ToS/guidance/granular browser-key policy
- **`runtime.py`** lines 117–121 `_normalize_route(route)`: maps auto/anthropic/codex → "arnold"
- **`runtime.py`** lines 135–147 `_agent_id_for_route(route)`: maps to Arnold dispatch agent IDs ("hermes", "codex", "claude")
- **`runtime.py`** lines 400–407 `_requested_route(route)`: normalizes claude→anthropic, codex→openai-codex

These serve different concerns (UI policy vs. dispatch routing) so the overlap is nominal, not duplicated logic — but it means a route change must be made in 3–4 places.

**Priority 1 — Remove duplication (no behavior change):**

1. **Delete `_extract_rebaseline_recovery` from `routes.py`**, import it from `contracts.py` instead.

   - `audit.py` line 42–43 → `from .session import _now`
   - No rename needed (same name, same implementation)

**Priority 2 — Low-risk extract (improves cohesion):**

3. **Extract message builders from `provider.py` into `contracts.py`** (or a new `messages.py`):
   - `build_messages()` (lines 390–421)
   - `build_delta_messages()` (lines 424–453)
   - `build_batch_messages()` (lines 182–360)
   - `build_delta_messages()` (lines 424–453)
