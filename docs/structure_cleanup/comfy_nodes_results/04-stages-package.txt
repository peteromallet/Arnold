### 4. Divergence Discovered

`stages/agent.py` `_stage_agent()` calls:
```python
messages = build_messages(task=state.task, python_source=state.python_before)
```
`agent/edit.py` `_stage_agent()` calls:
```python
messages = build_messages(task=state.task, python_source=state.python_before,
                          execution_mode="sandboxed_loose")
```

The `execution_mode` kwarg is **missing** from the `stages/` copy. If ever swapped in, this would change prompt construction for the full agent path. This confirms the `stages/` copies are stale and not synchronized.

### 5. Should `stages/` Remain a First-Class Package?

**No** — not in its current form.

The `stages/` package is an unfinished extraction that was abandoned mid-refactor. Six of seven files are fully dead; the seventh (`humanize.py`) is 99% dead code co-located with 3 live utility functions that happen to be imported by `session_io.py`.

**Recommendation:** Delete the package. Move the 3 live symbols from `humanize.py` into `agent/_humanize.py` (or inline them in `session_io.py` — each is a one-liner wrapper or small utility).
