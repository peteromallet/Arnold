### 1. Proposed Mental Model

- **`docs/`** = **Source of truth** — hand-authored or reviewed project documentation, specifications, architecture decisions, migration guides, templates, cookbooks. *Curated, durable, versioned.*

- **`artifacts/`** = **Process deliverables** — outputs of specific milestone steps (audit findings, baseline snapshots, codegen outputs, test logs). *Useful for traceability but are *intermediate* — not final docs, not throwaway runtime.*

- **`out/`** = **Expendable scratch** — logs, run cache, remote job results, subagent reviews. *Regenerable, machine-generated, gitignored.*

### Core problem: `artifacts/` is a **hybrid zone**. It holds (a) milestone audit docs that belong in `docs/audits/` and (b) generated code/test artifacts that belong in `out/`.

---

### 2. Recommendation: **Split** `artifacts/`

**Keep the directory concept** but re-scope. It should hold only **milestone process artifacts that inform future decisions** (baselines, audit findings, symbolic maps). Everything else belongs in `out/`.
