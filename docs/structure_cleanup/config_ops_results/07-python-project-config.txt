### Pre-commit hook concerns:

All three hooks reference valid Python modules. However, using `language: system` with `pass_filenames: false` means:
- Hooks run in the user's current virtualenv, which may or may not have `vibecomfy` installed.
- `pass_filenames: false` combined with `files: ^ready_templates/.*\.py$` is harmless but confusing — the pattern matches but no filenames are passed.

**Recommendation:** Either switch to `language: python` (or `python_venv`) with `additional_dependencies` for proper isolation, or keep as-is but document that hooks need an editable install.
