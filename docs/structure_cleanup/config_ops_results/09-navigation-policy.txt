## Audit Summary

### Q1. Does root README explain root config/ops files enough?

**No.** The Repository Layout table only listed 6 of the 18+ non-hidden items at root. Config files (`pyproject.toml`, `cloud.yaml`, `version_matrix.json`, `asset_manifest.json`, `.pre-commit-config.yaml`, `.importlinter`, `AGENTS.md`) and important directories (`scripts/`, `tools/`, `vendor/`, `.github/`, `agentic/`) had zero mention. A reader cannot learn the project's structural shape from the README alone.

### Q2. Should new `docs/ops/`, `docs/config/`, `.github/README.md`, or `vendor/README.md` be added?

| Candidate | Verdict |
|---|---|
| `docs/ops/` | **No** — `scripts/README.md` already covers ops well. |
| `docs/config/` | **No** — too few config files to warrant a directory. |
| `.github/README.md` | **Optional** — 6 CI workflows exist with no summary, but workflows are well-named. |
| `vendor/README.md` | **Yes** — added. Explains what is vendored, why, and how to update the submodule. |

### Q3. Smallest useful navigation improvement?

**Update the Repository Layout table** in the root README to include all root-level directories and config files. This is a single-table edit — no new files, no structural changes, immediate payoff for anyone landing on the repo.

### Edits Made

1. **`README.md`** — Repository Layout table expanded from 6 rows to 18 rows, covering every non-hidden root path (`scripts/`, `tools/`, `vendor/`, `.github/`, `agentic/`, `pyproject.toml`, `cloud.yaml`, `version_matrix.json`, `asset_manifest.json`, `.pre-commit-config.yaml`, `.importlinter`, `AGENTS.md`). Existing rows (`scripts/`, `tools/`, `agentic/`) link to their dedicated READMEs.

2. **`vendor/README.md`** — New file. Explains the purpose of the vendor directory, the Single submodule (ComfyUI LTS fork), the upstream URL and pinned branch, and how to update it. This closes the gap where `vendor/` was a black box to newcomers.
