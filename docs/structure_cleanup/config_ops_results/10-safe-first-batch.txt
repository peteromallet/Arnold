## Proposed First Batch

### 🟢 NOW — exact deletions (zero risk)

```
# 1. macOS noise files (not in .gitignore but harmless rot)
rm  out/.DS_Store
rm  vendor/.DS_Store

# 2. Generated gitignored caches (safe: regenerated by `sources sync`)
rm  workflow_index.json
rm  node_index.json
rm  template_index.json
```

These three `.json` index files are gitignored, deterministic outputs of `python -m vibecomfy.cli sources sync`. Deleting them is the same as a clean checkout. No tool will break — they'll either regenerate on next `sources sync` or work without them (the CLI has fallback paths).

### 🟢 NOW — exact edits (cosmetic, no behavioral change)

**Edit `.gitignore`** to add the two `.DS_Store` files that currently exist and rot:

```
--- a/.gitignore
+++ b/.gitignore
@@ -1,7 +1,9 @@
 .venv/
 __pycache__/
 .pytest_cache/
 .ruff_cache/
 .import_linter_cache/
 .DS_Store
+out/.DS_Store
+vendor/.DS_Store
 *.pyc
```

Cancel — `.DS_Store` at the root level already glob-matches all `.DS_Store` files recursively. So these are already covered. No edit needed.

**Actually, let me check — `.DS_Store` in gitignore without a leading slash...** A bare `.DS_Store` pattern in gitignore matches `.DS_Store` at any level of the tree. So both `out/.DS_Store` and `vendor/.DS_Store` are already gitignored. They just exist on disk. So deletion is still safe, no `.gitignore` edit needed.
