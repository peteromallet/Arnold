# Artifact Classification

This document classifies the files and directories produced or consumed by the
VibeComfy repo.  The categories are fixed to: **source**, **generated committed
source**, **generated local build output**, **runtime output**, **diagnostic
export**, and **scratch**.

## Source

Files humans edit and review.

| Path | Owner | Notes |
|---|---|---|
| `vibecomfy/` | backend team | Python source for agent-edit, porting, executor, etc. |
| `tests/` | QA/team | Python and browser tests. |
| `docs/` | team | Architecture, API, and design documentation. |
| `scripts/` | team | Helper scripts and audits. |
| `ready_templates/` | templates team | Template packs and supporting assets. |

## Generated committed source

Files generated by a build/codegen step but committed intentionally so the
runtime does not need the generator.

| Path | Generator | Owner | Notes |
|---|---|---|---|
| `vibecomfy/_compile/_widgets.py` | internal widget compile step | backend | Frozen widget metadata. |
| `vibecomfy/comfy_nodes/web/*.js` (bundled) | bundler | frontend | When a bundle is committed instead of built at runtime. |

## Generated local build output

Files produced by local development commands.  These should never be committed.

| Path | Generator | Cleanup |
|---|---|---|
| `.venv/` | `uv` / `python -m venv` | Delete and recreate as needed. |
| `node_modules/` | `npm` / `pnpm` | `rm -rf node_modules` |
| `__pycache__/` | Python | `find . -type d -name __pycache__ -exec rm -rf {} +` |
| `*.pyc`, `*.pyo` | Python | same as above |

## Runtime output

Files produced by running ComfyUI / VibeComfy.  These are environment-specific
and must not be committed.

| Path | Owner | Notes |
|---|---|---|
| `out/` | runtime | Sessions, runs, editor output. |
| `out/editor_sessions/` | `session.py` | Per-session turn artifacts and lifecycle files. |
| `out/runs/` | runtime | Log files and run records. |
| `input/` | runtime | User-provided images and other inputs. |
| `models/`, `custom_nodes/` | ComfyUI | External model weights and custom node packs. |

`make clean-artifacts` deletes `out/` as local development hygiene.  Use it
only for disposable local sessions; do not run it against production,
customer-support, or support-critical session directories.

## Diagnostic export

Files produced explicitly for support, debugging, or issue reporting.  They are
not part of normal UI/chat state.

| Path | Producer | Consumer | Notes |
|---|---|---|---|
| `<session>/turns/<turn>/audit/audit.json` | `audit.py:write_audit` | issue bundle, CLI debug | Redacted durable record of a turn. |
| `<session>/turns/<turn>/response.json` | `edit.py` | `session.py:iter_turn_records`, CLI debug | Durable turn response. |
| `<session>/turns/<turn>/request.json` | `edit.py` | CLI debug | Durable turn request. |
| `<session>/session_state.json` | `session.py` | `session.py`, CLI debug | Lifecycle and baseline state. |
| `vibecomfy_debug bundle` | `vibecomfy/commands/_agent_edit_debug.py` | support / triage | Tarball exported by the CLI debug tool. |

## Scratch

Transient files that should not persist.

| Path | Producer | Cleanup |
|---|---|---|
| `chain_run.pid` | local chain runner | `rm chain_run.pid` after run |
| `temp/counter.txt` | local scratch | `rm temp/counter.txt` |
| `comfyui_arnold_debug*.png` | local debug render | `rm comfyui_arnold_debug*.png` |
| `.megaplan/plans/<plan>/` | megaplan engine | kept for traceability; do not hand-edit |
