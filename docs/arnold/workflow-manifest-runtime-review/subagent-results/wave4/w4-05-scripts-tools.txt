## Workflow-Manifest-Runtime Plan Artifact Inventory

### Plan Framework

The plan is a **6-milestone chain** (`chain.yaml`) targeting a clean-break migration of the Megaplan/Arnold repo from legacy `arnold.pipelines.megaplan`/native-runner surfaces onto a neutral explicit-node manifest/compiler/runtime architecture (`arnold.workflow` + `arnold.kernel` + `arnold.execution`). The product lands in a new namespace package: `arnold_pipelines.megaplan`. Three waves of sub-agent review (2× DeepSeek, 1× Codex) have already been completed and synthesized into `review-synthesis.md`, with corrections folded back into the milestone briefs.

### Remaining Gaps

1. **No non-package Python imports old `megaplan` directly.** Zero scripts/tools import `import megaplan` or `from megaplan`. The legacy top-level `megaplan` package appears absent from the current tree — only `arnold.pipelines.megaplan` exists as the current product package.

2. **`_gen_golden_traces.py` and `tools/m4_oracle_bisect.py` are the most entangled with old surfaces** (`arnold.pipeline.native`, `arnold.runtime.*`, `arnold.pipeline.executor`). Both are explicit M6 deletion targets per the plan.

3. **Path migration gap:** Five scripts (`backfill`, `generate_arnold_docs`, `live_watchdog`, `record_workflow_next_parity`, `check_pipeline_id_registry`) import from `arnold.pipelines.megaplan.*` or `arnold.pipeline.*`. None import the old `arnold.pipelines.megaplan` surface. All will need path repointing when M4 moves product to `arnold_pipelines.megaplan`.

4. **`sync-skills.sh`** has a hardcoded source path that must change post-M4.

5. **The `agent/` compatibility shim** is the single clearest clean-break violation — it keeps `import agent` alive. M5 must inventory it explicitly; M6 must delete or re-charter it.

6. **No `acceptance/` or `supabase/`** Python files import old surfaces — they are data/config only.

7. **`scripts/m4_oracle_bisect.py`** is a duplicate of its `tools/` sibling — plan should consolidate.
