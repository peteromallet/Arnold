Here is the `.megaplan` runtime-state inventory, organized by artifact class, with shape/path examples, role, disposition, hook, and gaps.

---

## 1. Plans (`plans/`) — ~120 plan directories

**Shape:** `plans/<slug>-<timestamp>/` containing phase artifacts: `plan_v1.md`, `plan_v1.meta.json`, `state.json`, `critique_v*.json`, `execution_batch_*.json`, `execution.json`, `finalize.json`, `gate.json`, `faults.json`, `review.json`, `execution_trace.jsonl`, `execution_audit.json`, `final.md`, etc.

Some plans carry a nested `.hermes_state/` subdirectory (SQLite `.db` + WAL/SHM files for executor and gatekeeper state) and `verification/` with raw logs.

Some plans carry a nested `.megaplan/runs/` subdirectory — plans that self-host their own pipeline run history (e.g., `folder-audit-pipeline-phase-1-20260616-2249`, `mimo-distributed-workflow-fencing`).

**Role:** Per-plan execution state, receipts, phase outputs, and locks (`.plan.lock`). These are the primary runtime capsules.

**Disposition:** KEEP. Vast majority are historical execution records. ~3–4 Shannon stress/smoke plans are archival; the rest carry pipeline development provenance.

**Hook:** Plans link to epics and locks by name match (slug prefix). Telemetry `.ndjson`/`.seq` files mirror plan names.

## 2. State Locks (`.state-locks/`) — 17 `.lock` files

**Shape:** `.state-locks/<epic-name>.lock`. All files appear empty (0 bytes).

**Role:** Global-advisory mutual exclusion for epic-level pipelines. Locks span epics that have plan directories and telemetry (e.g., `milestone-1-native-python-20260617-1719.lock`).

**Disposition:** KEEP. Active concurrency control.

**Gap:** Empty lock files rely on filesystem `O_EXCL` — no content check, no TTL, no owner PID. Stale-lock recovery depends on operator manual cleanup.

## 3. Epics (`epics/`) — 15 epic directories

**Shape:** `epics/<name>/` with `_journal/` subdirectory and `events.jsonl`.

**Role:** Journal of events across all plans within an epic. Mirrors the telemetry stream.

**Disposition:** KEEP. All active.

**Hook:** 1:1 mapped to state locks and telemetry files. Locks guard these epics.

## 4. Telemetry (`telemetry/`) — 26 files (`.ndjson` + `.seq` pairs)

**Shape:** `telemetry/<epic-name>.ndjson` and `<epic-name>.seq`. One pair per epic. Some epics have only one format.

**Role:** Append-only event journals. `.ndjson` = newline-delimited JSON; `.seq` = binary sequence counter.

**Disposition:** KEEP. Append-only runtime journals.

## Summary Gaps

1. **Empty lock files:** No TTL, owner PID, or auto-recovery. Stale locks require manual intervention.
2. **Recursive `.megaplan`** in briefs and plans: No documented convention for when subdirectories get their own pipeline state.
3. **Bare chain state:** `plans/.chains/chain-58e7820d33e0.json` — only one chain artifact at plans root. Unclear if this is canonical or stale.
4. **No archive policy:** `mimo-boundary-investigation/` is clearly superseded but has no archival marker or migration path.
5. **`_journal/`, `blobs/`, `db-blobs/`** are plumbed but empty — reserved for future runtime journal and artifact content-addressed storage.
