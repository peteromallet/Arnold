## Verdict

The conformance gates in expectation 9 are well-intentioned but miss critical dimensions while leaving several existing gates too vague to be machine-enforceable. The current list is **too broad** (terms like "import graph checks" and "behavior goldens" lack precise scope) and **missing** key gates: type-check/stub conformance, console entrypoint verification, package metadata validation, cross-branch contamination checks, import-error quality gates, and live-execution parity (not just golden parity).

---

## Missing Conformance Gates

| # | Missing Gate | Why It Matters |
|---|---|---|
| A | **Type-check / `py.typed` gate** | Import greps can't detect re-exports or stale `__all__` entries. A `mypy`/`pyright` clean pass on the public surface proves no old types leak. |
| B | **Console entrypoint removal gate** | M6 mentions deleting entrypoints but provides no machine check. Must grep `pyproject.toml` `[project.scripts]` and installed `bin/` for old command names. |
| C | **Package metadata gate** | `pyproject.toml` package discovery, `packages`, and `[tool.setuptools]` must not list deleted packages. No milestone checks this. |
| D | **Import-error quality gate** | M6 Done Criteria #1 checks `import megaplan` fails, but a bare `ModuleNotFoundError` is ambiguous. The gate must verify it raises a **specific** migration error with a migration path message. |
| E | **Cross-branch contamination gate** | After M6 merges, a gate must verify `origin/main` doesn't reintroduce deleted surfaces from merge artifacts or stale branches. Currently untested. |
| F | **Live-execution parity gate** | Golden fixtures cover static output but not live runtime with real backends. At minimum, one live-backed smoke path (even with a cheap model) must pass before M6 deletion. |

---

## Gates That Are Too Broad

| Gate | Problem | Recommended Scoping |
|---|---|---|
| "Import graph checks" | Vague — what scanner? What rules? | Define per-milestone forbidden-pattern lists: neutral→product (M1), product→deleted-public (M4), no old names in any public `__init__.py` (M6). |
| "Docs/scaffold checks" | Doesn't name the scanner or forbidden tokens. | Must name specific greps: `PipelineBuilder`, `Stage`, `Edge`, `ParallelStage`, `run_pipeline`, `native_runner`. Add whitelist for internal-only docs. |
| "Behavior goldens" | Too broad — covers every test fixture. | Scope to: (a) canonical Megaplan planning flow goldens (M1/M4), (b) shipped-pipeline output snapshots (M5), (c) CLI parser snapshots (M5). Explicitly list volatile fields to normalize. |

---

## Corrections by Milestone

- **M1**: Add Gate A (type-check boundary) and Gate D (import-error quality) to Done Criteria. The import-boundary tests already exist; add a `py.typed` marker and `mypy --strict` pass on `arnold/workflow/` and `arnold/kernel/`.
- **M4**: Add Gate F (live-execution parity) as a Done Criterion: one live-backed smoke run of the canonical Megaplan manifest, even with minimal model, must pass before M4 is Done.
- **M5**: Add Gate B (entrypoint inventory) scanning `pyproject.toml` for old entrypoint names. Add Gate C (package metadata check).
- **M6**: Add Gate E (cross-branch contamination) as the final gate, running the full forbidden-import and deletion-gate suite against the merged result, not just the feature branch.

## Proof Gates (Final)

1. **M6 final gate suite must pass against the merge commit**, not the feature branch.
2. `pip install .` in a clean venv → `import megaplan` raises a **named migration error** with docs link.
3. `mypy --strict` or `pyright` clean on the public surface.
4. `pyproject.toml` contains zero references to deleted package names or entrypoints.
5. At least one live-backed workflow run (not fake backend) matches golden behavior before deletion is authorized.
6. Forbidden-token scans against `docs/`, `README.md`, and scaffold templates find zero hits outside explicit whitelists.
