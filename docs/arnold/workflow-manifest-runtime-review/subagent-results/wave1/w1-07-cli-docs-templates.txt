## Verdict

**The plan correctly identifies the big surfaces — CLI commands, core docs, shipped pipelines — but misses several operator-facing artifacts that would leak the old public surface after M6.** The M5 brief names five CLI verbs and three doc pages; the actual public/operator surface is larger and includes auto-generated artifacts that the plan never audits.

---

## Missed Surfaces

### 1. Generated skills (`_codex_skills/*/SKILL.md`)
Eight generated skill files under `arnold/pipelines/megaplan/data/_codex_skills/` (megaplan, megaplan-prep, megaplan-cloud, megaplan-observe, megaplan-epic, megaplan-bakeoff, megaplan-tickets, babysit) are consumed by the agent runtime. None appear in M5's migration checklist. The generation logic in `arnold/pipelines/megaplan/cli/skills.py` must produce new-surface references — otherwise agents receive stale instructions referencing `PipelineBuilder` or legacy paths.

### 2. Auto-generated projection docs
`docs/reference/arnold-projections.md` is generated by `scripts/generate_arnold_docs.py`. It currently describes `native_program`, `@pipeline` generator, and `driver=("native", ...)`. After cutover this must describe `arnold.workflow.Pipeline` and `WorkflowManifest`. The plan never addresses regenerating this artifact.

### 3. Package metadata (`pyproject.toml`)
The description reads *"Megaplan as the flagship planning pipeline plugin"*. After M6, Megaplan is a product package under `arnold_pipelines`, not the identity of the wheel. Console entrypoints and optional-dependency groups also need audit.

### 4. Package contract and authoring docs
`docs/arnold/package-contract.md` and `docs/arnold/package-authoring-contract.md` teach native-first `@pipeline` generators. After cutover they must teach `arnold.workflow.Pipeline(..., steps=[...])`. Neither is in M5's touchpoints.

### 5. Package disposition registry
`docs/arnold/package-disposition.yaml` enumerates available pipelines. Post-cutover it must not list packages that were deleted in M6, and must point to new canonical paths.

### 6. Scaffold template
`arnold/pipelines/_template/SKILL.md` is the authoring scaffold for new pipelines. It must model the new explicit-node style. The `creating-a-new-pipeline.md` guide mirrors this.

### 7. Console entrypoint dispatch
`arnold/cli/__init__.py` delegates to `arnold.pipelines.megaplan.cli.arnold`. After M6 deletes `arnold.pipelines.megaplan`, this lazy import breaks. The plan never names this dependency.
