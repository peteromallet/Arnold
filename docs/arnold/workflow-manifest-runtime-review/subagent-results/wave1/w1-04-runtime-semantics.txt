## Verdict

Expectation 4 is **90% correct** — but conflates generic runtime *mechanism* with product *policy* in two places, and one item is already known to be unimplemented.

### Correctly Generic Runtime Behavior

The neutral Arnold runtime (`arnold.execution` + `arnold.kernel`) should own these as policy-blind primitives:

- **Append-only events/journals** — foundation of audit, replay, resume. No product dependency.
- **Artifact hash binding** — content-addressed artifact identity is universal.
- **Resume cursors** — `(manifest_hash, reentry_id, scope_stack, event_sequence)` is pure runtime coordinate space.
- **Replay** — deterministic event-journal scanning with hash verification. Generic.
- **Fake backends** — in-memory execution for testing. Any runtime needs this.
- **Branch routing, loops, retries, fanout/fanin, subpipelines** — all are structural topology concerns. The runtime walks the manifest graph; `decide=`/`until=`/`reducer=` are importable product-owned functions, but the routing *mechanism* is generic.

### Product Policy (NOT runtime-owned)

- **Capability/effect enforcement** — The *enforcement mechanism* (fail-closed capability check before node dispatch) is generic. But the capability *vocabulary* (`run_shell`, `human`, `subjective_judgment`) and effect *operations* (`git_push`, `pr_create`) are product-registered extensions. M1 kernel contracts define the enforcement seam; products populate the registry.
- **Retry idempotency details** — The runtime enforces that retry-scoped effects declare `idempotency_key`. The *policy* for what constitutes idempotency per operation type is product data in the manifest.
- **Fanout merge/reduce strategy** — Runtime provides the fanout/merge mechanism; `majority_vote` vs. `weighted_vote` vs. `first_past_post` is product pattern choice.

### Must Be Explicitly Deferred

**Human suspension/resumption is the critical item.** Ticket `01KTPVTAZRJZTVRDCB4SP5M9QA` (open, severity MEDIUM) already diagnoses this: *"Suspend/resume is convention, not a runtime primitive — the neutral executor ignores Suspension/SUSPENDED."* The current evidence_pack implementation writes a `Suspended` envelope to disk and accidentally halts; "resume" restarts a fresh pipeline. The runtime has no `SUSPENDED` terminal exit, no deadline enforcement, no resume-payload validation.

**Correction**: M3 must implement SUSPENDED as a first-class terminal exit with durable event persistence and a generic rehydration hook. But the *human-gate-specific* schema (deadline, `on_timeout`, `resume_input_schema`, display artifacts) is product policy deferred to M4 (product migration) — Megaplan's human-gate becomes the first consumer proving the generic mechanism.
