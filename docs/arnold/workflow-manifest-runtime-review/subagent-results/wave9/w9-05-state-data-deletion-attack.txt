## Deletion-Attack Analysis — Wave 9

### How Old State Could Falsely Pass Deletion

**1. Archived/projected state still readable on disk.** M4 classifies old `.megaplan` state as `archive` or `project read-only`. If surviving CLI/operator code contains residual `open()`/`json.load()` calls pointing at `.megaplan/plans/*/state.json`, those files remain the de facto authority even though M6 import/wheel gates are green. The deletion plan scans imports, not filesystem reads.

**2. Incomplete event-journal field coverage.** M3's event journal may omit fields that old `state.json` carried — gate signal history, cost accumulators, override audit trails. If `arnold workflow status` falls back to reading old state for "completeness," the old file remains authoritative. M6 golden tests verify happy-path parity, not exhaustive field coverage.

**3. Nested `.hermes_state` / `.megaplan` directories as undiscovered authorities.** Subpipeline runs can write nested state directories. If these aren't enumerated in M4's historical-state ledger and surviving code discovers them via `os.listdir()`/`glob`, they become silent side-channel authorities after M6.

**4. Lock files surviving as coordination gatekeepers.** Stale `.megaplan/plans/*/lock` files with old PIDs/TTLs. If surviving orchestration code still checks these locks (or if lock cleanup is classified as `ignore`), the system remains dependent on old lock state for correctness. M6 gates don't test lock resolution.

**5. Legacy resume cursors blocking migration.** In-flight runs suspended before M6 carry old-format resume cursors in `state.json`. If M3/M4's cursor translation is incomplete, those runs can only resume from old state — the old file stays load-bearing. M6 passes because no *new* runs need it.

**6. Projection-as-authority drift.** If projection code that reads old state to produce event-journal views is retained past M6 (classified as `project read-only` with no sunset), operators consult the projected view but the underlying read path keeps old state as the ground truth. The projection becomes a permanent dependency.

**7. Generated artifacts embedding old paths.** M5 regenerates skills, projection docs, package-disposition data, and scaffolds. If any regenerated artifact contains references to `.megaplan`, `state.json`, or old-format schemas, the conceptual authority of old state survives even if code-level imports are clean.

### Evidence Required to Prove No Old State Authority Remains

1. **Filesystem-access trace** from the full test suite (installed-wheel smoke, CLI integration, operator commands) proving zero reads from `.megaplan/` outside sunset-expired migration code.

2. **Field-coverage matrix**: every key path in old `state.json` schemas mapped to its event-journal projection, with golden tests proving identical values for canonical runs. Gaps documented with explicit abandonment rationale.

3. **Nested-state ledger**: `find` output of all `state.json`, lock files, and `.hermes_state` directories under `.megaplan` matched against the historical-state inventory with zero undiscovered entries.

4. **Resume-cursor audit report**: every old `state.json` checked for cursor/checkpoint fields; each entry marked `translated → manifest-coordinate`, `quarantined`, or `run completed/expired`. No untranslated surviving cursors.

5. **Lock-staleness report**: all `.megaplan` lock files listed with PID liveness, TTL, and disposition (`archived`, `deleted`, or `stale-broken → deleted`).

6. **Generated-artifact scan output**: `rg state.json\|\.megaplan` across all `_codex_skills/`, `_composed/`, projection docs, scaffolds, disposition data, and registry JSON — must return zero hits.

7. **Installed-wheel proof**: the built wheel, in a clean environment with `.megaplan/` removed, executes the canonical Megaplan workflow end-to-end with golden-parity results.

Only when all seven evidence artifacts are committed and green can M6 be considered truly closed. Without them, old `.megaplan` state survives as a silent authority — deletion gates pass on paper while the old files remain the real source of truth for resume, status, cost tracking, lock coordination, and operator inspection.
