## Wave 9: Deletion Attack — How Tests, Fixtures, and Inventories Could Falsely Pass After Deletion

Assuming M6 is nominally "green" but old semantics are not actually dead, here are the top ways tests and fixtures could be keeping the old native/runtime world alive or masking missing parity.

---

### 1. Top Ways Test/Fixture Deletion Could Falsely Pass

**A. Xfails and skips that mask real import/behavior failures.** If any test that exercises old `_pipeline`, `native_runner`, `PipelineBuilder`, `Stage`, `Edge`, or `run_pipeline` has been marked `@pytest.mark.xfail` or `@pytest.mark.skip` — especially with condition strings that evaluated to True because the old module now fails to import — those tests pass silently while the break is real and no new equivalent covers the gap. M6 Done Criteria do not require a sweep of xfail/skip markers to prove each is not a disguised deletion casualty.

**B. Whitelist rows acting as permanent keepalives.** M5 permits whitelisting only zero-public-surface internal coverage, with mandatory owner and expiry before M6. If a whitelisted pipeline, test root, generator script, or `test_*.py` is not burned down to migrate/delete before M6 closes, old execution paths survive under an "internal coverage" label. M6 correctly gates on this ("M6 cannot start while any M5 whitelist row still protects old native semantics"), but if that gate is checked by script output rather than human audit, a row reclassified at the last moment without actual migration is a false pass.

**C. Characterization/oracle fixtures locked against old runtime, then never re-compared.** M5 requires old-behavior traces locked before migration and new event journals compared for semantic equivalence. If the comparison harness is disabled (xfail, skip, conditionally skipped in CI), or if the new oracle is regenerated from the new runtime — so it always matches itself — then the gate is circular and proves nothing. A single golden diff that is normalized away (e.g., a volatile field silently added to the normalization spec during M4/M5/M6 without amendment) erases an entire class of parity evidence.

**D. Root-level `test_*.py` files not discovered by scanners.** M5 inventories `test_*.py` as a category, but M6's AST scan and `git ls-files` cover "code, tests, scripts, tools, generators, docs, package metadata, and generated artifacts." If a root-level test imports `_pipeline` or old runtime surfaces through a private helper, or uses `importlib.import_module(…)` with a string constructed at runtime, AST scans miss it. If the test is also not in any pytest collection path (no `conftest.py` root, not in `pyproject.toml` test paths), it survives undeleted and un-audited.

**E. Dynamic imports and lazy `__getattr__` shims in test support code.** M6 requires catching `importlib`, `__getattr__`, `__all__`, and console entrypoints in production. But `tests/conftest.py`, `tests/helpers/`, characterization harnesses, and golden generators can all hold lazy imports that resolve old paths. If the scan is production-only, a `conftest.py` that does `_pipeline = lazy_import("arnold._pipeline")` survives and keeps the import real.

**F. Fake-backend tests that never touch real product dispatch.** If M3/M4 fake-run tests are the only coverage for Megaplan parity, and the deletion of `native_hooks.py` or `native_runner.py` breaks real capability/effect handler registration, nothing fails because the test backend bypasses the product registries entirely. The plan requires live-backed smoke (M4 Done Criteria #7), but if that smoke is itself marked xfail or conditioned on a backend that is unavailable in CI, the gap is hidden.

**G. Editable-mode-only green CI.** The plan requires installed-wheel tests at M4 and M6. If conformance gates run only in editable mode — where old `arnold/pipelines/megaplan/` and `_pipeline/` directories resolve via local path — they pass. In a built wheel those directories are absent. If the wheel smoke is a separate step that doesn't block merge, deletion appears green while the wheel is broken.

**H. Generated artifacts not actually regenerated before the freshness check.** M6 Done Criteria #13 requires a passing generated-artifact freshness check. If the check runs against artifacts regenerated from old generator code that still lives under `scripts/` (which M5 inventories but may not migrate), then the check is a closed loop — the old generator produces old-format output, and the freshness check (which compares hash of generator to hash of output) shows them matching.

---

### 2. Exact Gates and Plan Edits Needed

| # | Gap | Edit |
|---|-----|------|
| 1 | No xfail/skip audit | Add to M6 Done Criteria: "Every xfail and skip marker in the merge-result test tree is justified by a non-legacy reason (e.g., platform, network, upstream bug) and no marker references a deleted module, class, function, CLI command, or runtime path." |
| 2 | Whitelist burn-down is a pre-condition but not provably closed | Add to M6 constraints: "Before M6 declares Done, the M5 whitelist inventory must be empty. No row may transition from whitelist to re-chartered internal coverage during M6 without an independent review that includes AST/dynamic-import proof the row has zero contact with deleted surfaces." |
| 3 | Characterization comparison is not idempotently re-executable | Add to M5 Done Criteria: "The characterization comparison harness supports a `--strict` mode that re-derives old-behavior traces from locked VCS snapshots of the pre-migration codebase and fails if any old trace is missing, regenerated, or normalized beyond the M1 locked volatile-field spec." |
| 4 | Root `test_*.py` and test helpers escape dynamic-import scans | Add to M6 Done Criteria: "Dynamic-import and lazy-import scans cover `tests/`, `conftest.py`, `scripts/`, `tools/`, `data/`, and characterization harnesses, not only production package paths." |
| 5 | Fake-backend-only parity | Add to M4 Done Criteria: "Live-backed smoke is not xfailed, skipped, or conditioned on a backend flag that CI does not satisfy. The merge gate must include at least one real-backend run or document the explicit acceptance of the risk." |
| 6 | Editable vs installed-wheel asymmetry | Add to M6 constraints: "Merge-result conformance gates must pass from a built wheel installed in a clean venv, not from an editable checkout. The editable checkout may be used for authoring but the gate is the wheel." |
| 7 | Generated-artifact circularity | Add to M5 constraints: "Generator code for skills/docs/registries/disposition/scaffolds must be migrated to import only surviving public surfaces before regeneration. The freshness check must compare the migrated generator's source hash, not the old generator's hash." |
| 8 | Conformance allowlist re-chartering loophole | Add to M6 Done Criteria #10: "Any re-chartered entry must additionally pass a dynamic-import scan proving it never resolves a deleted module through `importlib`, `__getattr__`, `__all__`, console dispatch, installed-wheel path fallback, or type-stub forwarding." |
| 9 | Resume/replay using old state fixtures | Add to M6 constraints: "Tests under `tests/arnold/execution/test_resume*.py` and `tests/arnold/kernel/test_replay*.py` must not load `.megaplan/` state files, `state.json`, or old pickle/checkpoint formats except in an explicit migration-projection test that asserts the migrated form matches the event journal and then marks the old fixture for archival." |
| 10 | Manifest amendment protocol bypass | Add to M6 Done Criteria: "Any manifest/kernel contract change discovered during M4/M5/M6 must be recorded in `workflow-manifest-amendments.md` with a version bump and must back-propagate to M1/M2/M3 contract tests. M6 cannot close with an amendment version newer than the one all conformance tests assert against." |

### 3. Evidence Required Before Deleting Old Tests or Old Runtime Code

Before any deletion, the following must be proven (not assumed):

1. **Import-graph proof:** A dynamic-import scan covering all Python files (production, tests, scripts, generators, conftest, tools, data loaders) shows zero resolutions of any path in the M6 deletion list. AST-only is insufficient.

2. **Installed-wheel import proof:** `python -c "import megaplan"` and analogous imports for every deleted surface fail with `ModuleNotFoundError` in a clean venv from the built wheel. Editable-mode success is not evidence.

3. **Semantic parity evidence:** The characterization comparison harness, run in strict mode against locked pre-migration traces, shows pass for every node outcome, decision, capability invocation, suspension point, and deterministic artifact hash in the canonical fixture matrix.

4. **Xfail/skip justification audit:** Every xfail/skip marker in the merge-result test tree has a non-legacy reason, and a grep for deleted module/class/function/CLI names across all marker strings returns zero results.

5. **Whitelist inventory is empty:** No M5 whitelist row survives into M6; every row burned down to migrate or delete with VCS evidence.

6. **Generated-artifact loop broken:** Generator code is migrated to import only surviving surfaces, regenerated artifacts pass provenance/compile/fake-run/embedded-example checks, and the freshness check compares the migrated generator hash.

7. **Allowlist burn-down with dynamic-proof:** Every conformance allowlist entry is either removed or re-chartered with dynamic-import proof of zero contact with deleted surfaces.

8. **No old state in test fixtures:** No resume/replay/execution test loads a pre-migration `state.json`, pickle checkpoint, or `.megaplan/` blob except in a migration-projection test that asserts equivalence and marks the fixture for archival.

The deletion is safe only when the gates above are *closed and blocking*, not advisory. The plan already names many of these gates but does not lock them as pre-deletion required evidence — adding the ten edits above would close the loop.
