## Verdict

**The plan is directionally sound but has four scope gaps and one over-scoped ambiguity that will cause execution friction.** The clean-break target is well-defined, and the milestone sequencing is correct. ## Missing Surfaces

1. **`arnold.agent` integration points.** `arnold.agent` is a large existing neutral package (adapters, tools, providers, environments). M4 says product may depend on it, but no milestone specifies what agent/runtime contract M3 needs to finalize. The plan mentions "Route model/tool dispatch through neutral `arnold.agent`" in M4 but never defines the interface seam between `arnold.execution` and `arnold.agent`. **M3 must include a thin capability/effect dispatch contract that `arnold.agent` satisfies**, or M4 will discover the gap ad hoc.

2. **Old `arnold.runtime` disposition.** The existing `arnold/runtime/` package (envelope, resume, event_journal, dry_run, etc.) is not mentioned in any milestone. M3 builds `arnold.execution` and `arnold.kernel` as replacements, but the old runtime's fate — delete, merge, or refactor — is never declared. **M3 should add old `arnold.runtime` to its anti-scope and M6 should add it to the deletion list.**

3. **`arnold.control` and `arnold.conformance` surface status.** Both exist on `main`, both are referenced by Megaplan product code (`arnold.pipelines.megaplan`), but the plan never names them as staying, being replaced, or being deleted. They're absent from the surface-edges audit. If they're neutral Arnold packages meant to survive, they belong in the M1 import-boundary tests.

4. **`_pipeline` and bridge inventory is incomplete in M4.** M4's done criteria mention "Remaining legacy surfaces are inventoried for M6 deletion with blockers named," but there's no concrete inventory of `_pipeline`, `_bridge`, `_forward_m2_m3`, `_compatibility`, and old `builder.py` files in the touchpoints. M4 touchpoints list `arnold_pipelines/megaplan/` directories but don't include a committed inventory file like `docs/arnold/legacy-surface-inventory.md`.

This creates confusion about what "passes" the `import megaplan` gate.
