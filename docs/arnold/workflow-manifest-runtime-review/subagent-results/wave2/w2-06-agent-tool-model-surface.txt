## Verdict: **Plan Is Structurally Sound After Wave 1 Hardening, but Wave 2 Exposed Four Critical Gaps — Agent/Tool Dispatch, Live Smoke, State Migration, and Runtime Edge Semantics — That Will Block M3/M4 Without Amendment.**

---

### Missing Surfaces

**1. Agent/tool/model dispatch contract.** `arnold.agent` is a large existing neutral package (adapters for Codex/DeepSeek/Shannon, tools for browser/terminal/vision/MCP/session/search, providers, sandbox wrappers, budget/token tracking, approval gates). M3 builds `arnold.execution` and M4 says product "may depend on" `arnold.agent` — but **no milestone defines the capability/effect dispatch seam between the runtime and the agent**. Without it, M4 discovers that the runner cannot invoke a model, route through OpenRouter, enforce budget caps, sandbox a terminal call, or require approval without importing product policy. **Gap:** every tool in `arnold/agent/tools/` (browser, terminal, vision, MCP, session, file_ops, web, sandbox) is a capability that must be registered against a neutral runtime dispatch contract. The plan says "capability checks" in M3 but never names the agent-tool surface.

**2. Live smoke testing surface.** M4 done criteria #6 says "At least one live-backed smoke run succeeds" — but no milestone defines what "live" means (real API calls? real model inference? real file system?). The fake-backend tests in M3 are deterministic, but the jump from fake to live carries risks around credential injection, rate limiting, budget exhaustion, tool side-effects, and idempotency of real external calls. No smoke gate exists between M3 fake-run and M4 live-backed.

**3. State/artifact migration path.** M4 bundles "move or re-author" but no milestone addresses on-disk state migration (`plan_v*.md`, `state.json`, receipts, capsules under `.megaplan/`). A plan run today must be resumable tomorrow through the manifest runner. Without a committed migration inventory, M6 risks orphaned state.

**4. Runtime semantics: cancellation, timeout, escalation, compensation.** No milestone defines `CANCELLED` terminal state, node-level execution timeout, pipeline-level deadline, or compensating-action hooks after upstream side-effects. These are generic runtime concerns that M4 Megaplan will consume.

### Product/Core Boundaries at Risk

| Boundary | Status | Risk |
|---|---|---|
| `arnold/kernel/human.py` vs `suspension.py` | **Leaked.** Human gates are product; suspension is kernel. M1 touchpoints blur this. | Product policy leaks into neutral kernel. |
| Reducer dispatch | **Under-specified.** M3 says fanout reducers are generic but never draws the line between runtime mechanism and product-owned reducer function. | Megaplan-specific merge logic could embed in runtime. |
| `arnold.patterns` constructor count | **Over-scoped.** M2 lists 16 pattern constructors including `panel`, `tournament`, `merge` — these are product compositions, not universal primitives. | Baking Megaplan-specific patterns into neutral Arnold freezes wrong abstractions. |

### Over-Scoped Live Behavior

- M2's 16 pattern constructors exceed what's needed to build the canonical Megaplan shape. `panel`, `tournament`, and `merge` are product compositions.
- M6's "Delete top-level `megaplan` package" is imprecise — the package doesn't exist on `origin/main`; only the console entrypoint needs deletion.

### Proof Gates (concrete, testable)

1. **M3 agent-dispatch gate:** `arnold.execution` invokes a tool capability through a neutral dispatch registry. `arnold.agent` satisfies the contract. `grep` proves zero `arnold.agent` imports in `arnold/execution/`.
2. **M3 cancellation gate:** Fake-backend test: start pipeline, cancel mid-execution, assert `run_cancelled` event, children never start, resume confirms terminal state.
3. **M3 timeout gate:** Node with `timeout_seconds=0.1` that blocks transitions to `TIMED_OUT`. Pipeline with `deadline_seconds=1` terminates within tolerance.
4. **M4 live-smoke gate:** One canonical Megaplan run against real backend (real API, real model, real tools) succeeds with event journal parity to fake-backend run. Drift report if behavior diverges.
5. **M6 agent-surface gate:** Every tool in `arnold/agent/tools/` (browser, terminal, vision, MCP, session, sandbox) is reachable via the neutral capability dispatch contract — no tool requires direct `arnold_pipelines.megaplan` import to invoke.
