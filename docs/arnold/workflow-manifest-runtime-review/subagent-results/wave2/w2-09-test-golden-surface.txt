## Verdict: Plan is directionally sound but evidence-backed gates are thin; the existing test infrastructure tests the *old* architecture, not the *new* one.

---

### Missing Evidence (8 critical gaps)

1. **Installed-wheel smoke**: zero coverage. M6 requires it but no `tests/installed_wheel/` or pip-install-then-test pattern exists. Without this, deletion gates are unprovable.
2. **No `arnold.workflow` / `arnold.kernel` / `arnold.execution` packages exist**. All existing conformance/import-surface tests operate against `arnold.pipelines.megaplan.*` — the old architecture. M1-M3 evidence is aspirational only.
3. **Manifest v1 schema, hash, ref, event tests**: absent. M1 done criteria demand these; none exist.
4. **Neutral-to-product import-boundary conformance**: absent. M1/M3/M4 all require enforced separation; no AST scanner or import test proves it.
5. **Live smoke**: M4 requires "at least one live-backed smoke run." Not present.
6. **Pipeline inventory scanner**: M5 requires it; M6 depends on it for deletion traceability. Doesn't exist.
7. **Manifest amendment protocol doc**: M1 requires `docs/arnold/workflow-manifest-amendments.md`. Doesn't exist.
8. **Quarry/rejection inventory**: M1 requires a committed inventory mapping what's preserved vs rejected from `native-python-pipelines`. Doesn't exist.

### Weak Gates

- **M1 "no-regression rule"** (no golden update without explanation) is manual policy with zero enforcement. No CI hook, no scanner.
- **M2 canonical Megaplan-shaped conformance gate** lacks concrete pass/fail criteria. "Compile to manifest and it validates" is circular if the validator is the same code under test.
- **M3-to-M4 integration gate**: "fake-run the canonical manifest enough to unblock product migration" — no definition of "enough."
- **M5-to-M6 inventory traceability**: required but no tool verifies deletion targets are a closed subset of M5 outcomes.
- **Merge-result conformance** (M6): runs only against feature branch, not merge-equivalent. Manual step, easily skipped.

### Excessive Gates

- **M5 is overloaded**: shipped pipelines + CLI + docs + scaffolds + generated skills + composed rules + inventory + package metadata is one milestone. The plan itself acknowledges parallel M5a/M5b streams — split formally.
- **M4 shadow/dual-run seam**: building a comparison harness solely to delete it after parity creates a throwaway testing surface with its own maintenance burden. Replace with feature-flagged canary runs.
- **M1 hand-authored manifest**: done criterion #8 demands a hand-written branch+loop+fanout+subpipeline manifest before M2 begins. This is brittle and will drift against the M2 compiler; defer to M2 (compiler generates the canonical manifest).

### Scope Mismatches

- **Package naming conflict**: plan mandates `arnold_pipelines.megaplan` (underscore) as permanent home, but all existing code uses `arnold.pipelines.megaplan` (dot). The anti-scope claims "do not rename packages" while simultaneously requiring a move to an underscore-named package. Resolve before M1.
- **M2 neutral patterns encode Megaplan-specific concepts**: `agent`, `critique`, `tournament`, `panel` in `arnold.patterns` are Megaplan product concepts. These should live in `arnold_pipelines.megaplan.patterns`, not in the neutral layer.
- **`/tmp/workflow_decision_docs/` external dependency**: M1 source material lives outside the repo with no committed copy. Risk of lost context.

### Recommended Milestone Edits

| Milestone | Edit |
|-----------|------|
| **M1** | Drop hand-authored manifest criterion #8. Add committed snapshot of workflow decision docs. Add installed-wheel scaffold. |
| **M2** | Move Megaplan-specific patterns (`agent`, `critique`, `tournament`) to `arnold_pipelines.megaplan.patterns`; keep only generic topology builders in `arnold.patterns`. |
| **M3** | Define concrete M3-to-M4 gate: canonical manifest must fake-run through all routing paths, replay from 3 distinct resume cursors, and survive manifest-hash mismatch quarantine. |
| **M4** | Replace shadow/dual-run with feature-flagged canary. Require live smoke *before* migration sign-off. |
| **M5** | Split into M5a (pipelines + CLI) and M5b (docs + scaffolds + generated surfaces + inventory). Formal dependency: M5b depends on M5a. |
| **M6** | Add automated merge-result checkout script. Add installed-wheel gate as blocking CI step. Add scanner proving deletion list ⊆ M5 inventory. |

### Proof Gates (add to M1 as anchor)

1. **Goldens must compare, not just snapshot**: tests must re-derive golden fixtures and assert equality — not just capture and trust.
2. **No-golden-update enforcement**: CI step that fails if behavioral golden files change on a commit that only moves imports/packages (AST-detectable).
3. **Import-boundary scanner**: AST-based tool run in CI proving `arnold.workflow`, `arnold.kernel`, `arnold.execution` never import `arnold_pipelines.megaplan`.
4. **Installed-wheel conformance**: `pip install . && python -c "from arnold.workflow import Pipeline"` must pass before M3 sign-off; identical gate for M6 deletion checks.
5. **Manifest hash determinism**: separate-process round-trip test proving identical DSL input produces byte-identical manifest hash.
