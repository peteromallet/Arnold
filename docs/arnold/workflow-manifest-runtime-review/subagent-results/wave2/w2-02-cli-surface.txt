## Verdict: M5 CLI Scope Is Underspecified — 6 Planned Verbs vs. 40+ Existing Commands

The plan names six `arnold workflow` verbs (`check`, `manifest`, `dot`, `dry-run`, `run`, `resume`) but treats the existing 40+ operator commands as invisible. The M4→M5→M6 sequencing is sound, but the CLI transition mapping is a plan defect, not an open question to resolve during execution.

---

## Missed Surfaces

**Operator commands not addressed by M5:**
- `arnold workflow status`, `trace`, `inspect` — M4 Done Criterion #5 requires these project from manifest events/artifacts. M5 doesn't list them.
- `arnold workflow override` — the 11-routed-action override system needs a new CLI surface; M5's six verbs don't include it.
- The full `megaplan` command set (init, list, config, setup, audit, feedback, contract, epic, brief, ticket, step, loop, user-action, quality-gate) — no mapping of old→new/delete/migration-error exists.
- `arnold pipeline` (current top-level dispatch) — fate unspecified.

**Artifact surfaces not addressed:**
- `arnold/cli/__init__.py` lazy-imports from `arnold.pipelines.megaplan.cli.arnold` — M6 will break this. M5 never names this dependency.
- Parser snapshot regeneration (`test_cli_parser_snapshot.py`, `test_arnold_parser_snapshot.py`) — not in M5 scope.
- Console entrypoint (`megaplan` script) — no transition or deprecation plan.
- Installed-wheel CLI smoke tests — not in M5/M6 done criteria.

**Overlap with wave 1 review:** The `w1-07-cli-docs-templates` subagent already flagged generated skills, projection docs, package metadata, and entrypoint dispatch. These remain unincorporated into M5's touchpoints.

## Accidental Compatibility Creep

1. **`arnold pipelines run` → `megaplan run` delegation**: survives as a hidden forwarding path that M6 deletion would break. The plan never maps this transition.
2. **`arnold/cli/__init__.py` lazy import**: creates a silent dependency on `arnold.pipelines.megaplan` that M6 deletes. W1-08 already corrected the deletion list but didn't address this dispatch chain.
3. **`megaplan` console script**: external scripts and docs reference this name; no deprecation visibility period is mandated (contra W1-10's recommendation).
4. **Over-scoped retention risk**: `arnold pipelines upgrade-cursor` and `arnold pipelines new --driver native` encode native-runner assumptions that should be deleted, but no explicit deletion decision exists.
