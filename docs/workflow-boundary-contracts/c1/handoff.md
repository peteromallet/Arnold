## 11. Design Decisions and Constraints

### SD1: Observe-Only
C1 is strictly observe-only. It freezes schemas, builds matrices, captures fixtures, and adds diagnostics without changing producer completion order, lifecycle mutation, watchdog behavior, repair queues, chain advancement, cloud status, or auditor behavior.

### SD2: Scoped Baseline + Full-Suite Hard Gate
The finalize baseline is scoped to concrete C1 test selectors for fast feedback. The full-suite `pytest` command is a mandatory hard acceptance gate via `always_run` and `full_suite_fallback=true`. This is an explicit full-suite opt-in, not an implicit fallback.

### SD3: Typed Unknown Markers
Missing standalone producer categories receive typed `unknown` or `non_conformant` fixture markers for schema coverage but do not count as real producer-driven health coverage. C2–C6 will add real producers; C1 documents the gap.

### C2–C6 Sunrise
- 22 of 35 contracts (63%) lack real producers at C1 freeze — documented in `coverage_and_exceptions.md`.
- Tiebreaker sub-step receipts (4 contracts), review receipts (5), finalize receipts (3), replan receipt (1), parent rejoin receipt (1), and override receipts (8) are targeted for C2 migration.
- Transaction/outbox migration, runtime reconciliation (CP1–CP12), NDJSON sequence gap detection, and idempotent retry of external effects are targeted for C4–C6.
- Evidence pack boundary mapping is targeted for C3.
- Full 35-contract producer coverage is targeted for C6.
