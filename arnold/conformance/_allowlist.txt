# This is a ratchet baseline. Remove entries as each transitional coupling is

#
# --- M5 Allowlist Entry Rationales ---
#
# 1. import-coupling arnold.cli
#    `arnold.cli` is the canonical Arnold CLI forwarder. It deliberately imports
#    Megaplan package names to provide backward-compatible `megaplan pipelines`
#    and `megaplan pipelines doctor` commands. This coupling is intentional
#    infrastructure, not accidental extraction leakage. Remove only after M7
#    when old `megaplan` CLI paths are fully retired.
#
# 2. package-name-staleness arnold.cli
#    Same rationale as (1). The forwarder module mentions active Megaplan package
#    names in string constants for subprocess forwarding. These are stale only if
#    the Megaplan package names themselves change; until then, they are accurate.
#
# 3. package-name-staleness arnold.cli.forwarder
#    Same rationale as (1) and (2). The forwarder sub-module uses Megaplan package
#    names in CLI argument construction.
#
# 4. package-name-staleness arnold.pipeline.registry
#    The registry resolver searches for packages under both `arnold.pipelines`
#    and `megaplan.pipelines` discovery roots. The `megaplan` string literals in
#    registry path construction are needed for backward compatibility with
#    packages that still live under the Megaplan namespace. Remove when the
#    Megaplan pipeline packages are fully relocated.
#
# 5. semantic-coupling arnold.pipeline.step_io_policy
#    `step_io_policy` exposes `decision_blocks_read()` and
#    `decision_blocks_write()` helpers that reference `PlanState` and phase-name
#    vocabulary. This is the last remaining semantic-coupling allowlist entry
#    (Slice 32 removed the stale schema_registry entry). Resolve when the policy
#    module is decontaminated or moved to a package-owned location.
#
# 6. public-workflow-layering arnold.pipelines.evidence_pack.steps
#    Evidence-pack steps expose `Stage` and `StepResult` in their public method
#    signatures for integration with the neutral runner surface. This is
#    package-local runtime integration, not a public workflow authoring API.
#    Revisit when the public authoring DSL is settled (M2).
#
# 7. public-workflow-layering arnold.pipelines.evidence_pack.hooks
#    Evidence-pack hooks implement `ExecutorHooks` methods with `Stage`,
#    `StepResult`, and `StepContext` annotations. Same rationale as (6):
#    package-local runtime integration using the neutral hook contract.
#    Revisit when the public authoring DSL is settled (M2).
#
# --- End M5 Allowlist Rationales ---
