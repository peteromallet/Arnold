### 3. Missing Central Abstraction

**There is no single `ProviderConfig` or `PROVIDERS` registry.** The current architecture scatters provider identity across:

- 3 dicts in `pool.py` (key vars, base URL vars, default URLs)
- 3 dicts in `deepseek.py` (same 3 concepts, duplicated)
- 1 dict each in `_pipeline/preflight.py` and `cloud/preflight.py`
- 1 enum + 2 dicts in `model_seam.py`
- 1 `elif` branch in `resolve_model()`

**Recommended**: A single `@dataclass ProviderRegistration` at `arnold/agent/providers/pool.py` (the SSoT) that captures `name`, `env_key_var`, `env_base_url_var`, `default_base_url`, `display_name`, and `model_family`. Each downstream module would read from this registry instead of maintaining its own dict. The `resolve_model()` function could use a dispatch table keyed by prefix rather than an if/elif chain.
