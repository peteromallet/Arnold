On adapter failure (timeout/rate-limit/provider error), catch the exception and iterate through fallbacks. The dispatch layer is the *only* place with enough context to distinguish transient failures from permanent ones (bad prompt, auth error).

### 4. `arnold/agent/contracts.py` — `AgentRequest` (L306)

**Change**: Add `fallback_specs: list[str] | None = None` field so callers can thread fallback chains through the request object.

### 5. The dispatch call-site (wherever `AgentRequest` is constructed from profile specs)

The code that converts `phase=spec` strings into `AgentRequest` objects needs to detect the `||` delimiter and populate `fallback_specs`.

## Recommended TOML Syntax

Two options, ranked:

**Option A — Inline array (recommended)**:
```toml
[profiles.resilient]
execute = ["codex:gpt-5.5", "codex:gpt-5.4", "hermes:deepseek:deepseek-v4-pro"]
plan    = "premium:low"   # single string still works, full backward compat
```

**Option B — Delimited string (fallback if TOML arrays feel too foreign)**:
```toml
execute = "codex:gpt-5.5 || codex:gpt-5.4 || hermes:deepseek:deepseek-v4-pro"
```

**Recommend Option A**. TOML arrays are native and unambiguous. The `||` delimiter is an internal serialization detail — users never see it. The parser in `_validate_profile_map` normalizes the array to the delimited string for storage.

---

## Compatibility Hazards

1. **`||` in model names**: DeepSeek/Fireworks model paths don't contain `||`, but a future provider might. Mitigation: validate that no spec in a fallback list contains `||`.

2. **`dict[str, str]` type footprint**: The profile map type is used in ~40 function signatures across profiles, policy, pipeline resolution, and tests. All of them assume values are atomic specs. Changing to `dict[str, str | list[str]]` is a massive refactor. **Solution**: serialize lists to delimited strings at parse time so the `dict[str, str]` contract holds. Only the dispatch layer and rewrite functions need to know about the delimiter.

3. **`phase_model` CLI flag and state persistence**: `phase_models` are persisted as `phase=spec` strings in plan state. If a fallback spec is serialized with `||`, the `=` split in `apply_profile_expansion` (L800-803) still works because the `=` is only between phase name and spec. The `spec` side can contain `||` harmlessly.

4. **`tier_models`**: These are also `dict[int, str]`. Fallback lists should be supported in tier entries identically, validated in `_validate_tier_models`.

5. **Profile layering**: `merge_profile_layers` does full-profile replacement (L328-334), so a project-layer fallback list entirely replaces a built-in string for the same profile. This is correct behavior.

6. **Inheritance (`extends`)**: `_resolve_with_inheritance` does `dict.update()` merging, so a child profile's fallback list for a phase correctly overrides the parent's. No changes needed.

## Concise Implementation Plan

1. **Parse**: In `_validate_profile_map` (profiles/__init__.py), detect `isinstance(raw_spec, list)`. Validate each element. Join with `" || "`. Store as string. Mirror in `_validate_tier_models`.

2. **Rewrite functions** (policy.py): Add a helper `_map_fallback_specs(spec: str, fn: Callable[[str], str]) -> str` that splits on `" || "`, applies `fn` to each, rejoins. Call it in all five rewrite functions (`apply_vendor_rewrite`, `apply_depth_rewrite`, `apply_critic_rewrite`, `apply_available_model_floor`, `apply_deepseek_provider_rewrite`).

3. **Contract**: Add `fallback_specs: list[str] | None` to `AgentRequest` in contracts.py.

4. **Call-site threading**: Where `phase=spec` strings become `AgentRequest` objects, split on `" || "` and populate `fallback_specs`.

5. **Dispatcher**: In `ArnoldDispatcher.dispatch()`, wrap the adapter call in try/except. On transient error (subclass `TransientAgentError`), pop the next spec from `fallback_specs`, re-resolve the adapter, and retry. Raise on terminal errors or exhaustion.

6. **Tests**: Add the six test cases above. Ensure all existing tests pass unchanged.

**Estimated blast radius**: ~8 files, ~120 lines of new/edited code, zero breaking changes to existing profile files.
