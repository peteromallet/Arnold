## 2. New Fixtures/Helpers Needed

**New conftest fixture** (in `tests/conftest.py` or a new `tests/arnold_pipelines/megaplan/conftest.py`):

- `fallback_chain_fixture(agent_spec_list: list[str]) -> FallbackSpecChain` — a parametrizable fixture that produces valid `FallbackSpecChain` objects from lists like `["codex:gpt-5.4", "hermes:deepseek:deepseek-v4-pro"]`.

- `mock_worker_that_fails_then_succeeds(failures: list[tuple[str, dict]], success: WorkerResult)` — a monkeypatch-ready mock that fails with the specified `CliError` codes/payloads in order, then returns success. Enables testing the attempt loop without real LLM calls.

- `mock_worker_that_exhausts_chain(failures: list[tuple[str, dict]])` — raises after all failures consumed, to test exhaust behavior.

**New test helper module** (e.g., `tests/helpers/fallback.py`):

- `make_fallback_retryable_error(code: str)` → `CliError` — factory for retryable infrastructure failures (`connection_error`, `worker_timeout`, rate-limit 429, provider 5xx).
- `make_fallback_permanent_error(code: str)` → `CliError` — factory for non-retryable failures (`blocked`, `worker_parse_error`, schema validation failure).
- `assert_routing_ledger_has_fallback_fields(ledger_entry)` — assertion helper to validate `configured_specs`, `attempted_specs`, `selected_spec`, `selected_index`, `fallback_triggered`, `fallback_reasons` are present and correct.

## 4. Highest-Value Characterization Tests Before Editing Production Code

These should be written and passing (golden) before any production changes:

1. **Single-string profile dispatch produces identical routing** — load the current `partnered-5` profile, run `profile_to_phase_models`, capture the exact routing ledger entries for plan/critique/execute phases, and assert they are unchanged after fallback code lands. Use `tests/arnold/pipeline/test_profiles.py`.

2. **`AgentMode.__iter__` four-value unpacking contract** — assert `tuple(AgentMode(...)) == (agent, mode, refreshed, model)` for every agent/provider combination. Already partially covered but make it explicit in `tests/arnold/pipeline/test_profiles.py` under `TestAgentSpecShape`.

3. **`--phase-model execute=codex:gpt-5.4` still suppresses tier routing** — drive `apply_profile_expansion` with a CLI override and assert `tier_models.execute` is cleared. Extend `tests/arnold_pipelines/megaplan/test_profile_policy.py`.

4. **Adaptive critique tier selection with persisted state unchanged** — `test_persisted_state_tier_models_survive_stripped_args` in `test_handler_behaviors.py` already captures this. Run it under coverage and snapshot the full `_resolved_agent_mode` for each check to use as a golden.

## 5. Tests That Must Explicitly Prove Single-String Behavior Unchanged

- `test_fallback_chain_single_string_still_works` — give `_validate_profile_map` a plain `"codex:gpt-5.4"` string, assert it normalizes to a single-element `FallbackSpecChain` whose `.primary` is that string and whose `.specs` tuple has length 1.

- `test_policy_rewrites_on_single_string_preserve_identity` — apply vendor/depth/critic rewrites to a single-string profile, assert the result is identical to pre-fallback behavior.

- `test_persisted_scalar_phase_model_still_loads_on_resume` — resume from a state.json with `"phase_model": ["execute=codex:gpt-5.4"]`, assert the decoded model is the same scalar string, not wrapped.

- `test_override_set_model_preserves_scalar_semantics` — an `override set-model execute=codex:gpt-5.4` must suppress tier routing and produce exactly one `AgentMode`, same as today.

- `test_agent_mode_iter_four_value_unpacking_preserved` — explicit characterization: `a, m, r, mdl = AgentMode(...)` must not break or change arity.

## 6. Likely Brittle Tests to Avoid

- **Mocking `run_step_with_worker` internals deeply.** The existing `model_dispatch_inventory.json` anchors on specific whitespace/signature patterns. Don't write tests that depend on the exact line-by-line structure of `_impl.py`; test through the public dispatch boundary instead.

- **Testing fallback inside `WorkerUnit` fanout directly.** The `WorkerUnit` abstraction is thin and delegates to `run_step_with_worker`. Test the chain-attempt behavior at the `run_step_with_worker` level, not inside each fanout variant. The existing `test_phase_modules_do_not_own_raw_agent_fanout` already guards the fanout boundary — don't add tests that duplicate that.

- **Hardcoding provider error strings in retryability classifier tests.** Use the enum-based `FallbackDecision` classifier. Don't write tests that assert on raw `"429"` or `"5xx"` string matching — those will rot when providers change error formats.

- **Testing cloud/chain surfaces before the internal model is stable.** The chain YAML parser and cloud preflight consume encoded `phase_model` strings. Write characterization tests here only after the encoding format (`__fallback_json__`) is finalized — otherwise you'll be updating both the serializer and the cloud test fixtures simultaneously.

- **Extreme parallel review topology change tests.** The design doc explicitly defers this to post-v1. Don't write tests asserting that parallel Hermes review falls back to a single Codex review — it's out of scope and will produce misleading test coverage.

- **Full end-to-end integration tests with real LLM calls.** The `test_live_smoke.py` surface exists but is expensive and non-deterministic. Keep fallback tests at the unit/contract level with mocked workers. A single smoke test that exercises the chain with real retryable failures is sufficient but should be in a separate `test_fallback_live_smoke.py` gated behind `--run-live`.
