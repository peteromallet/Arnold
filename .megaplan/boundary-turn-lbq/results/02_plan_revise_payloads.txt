## 1. Verdict

**Plan and revise must first remain structured-payload boundaries.** They must not become Markdown-only draft boundaries in the first (or near-term) BoundaryTurn migration. The design doc already calls this out explicitly (lines 195–197 and 338–346), and the code confirms it. The path forward is: wrap the existing structured payload in BoundaryTurn, preserving `plan_vN.md` *and* `plan_vN.meta.json` as paired canonical artifacts. Only later—after metadata parity is proven—should a Markdown draft surface be considered.

---

## 2. Metadata or behavior at risk if switched to Markdown-only

| At-Risk Metadata / Behavior | Why Markdown-only would lose it |
|---|---|
| **`questions`** (list of strings) | Unstructured prose can't reliably distinguish questions from commentary. Handler uses this for state summaries and response fields. |
| **`success_criteria`** (list of `{criterion, priority, requires}` objects) | Priority (`must`/`should`/`info`) and `requires` capability arrays drive gate validation, verifiability audits, and test-selection policy. Parsing these from Markdown is fragile and would regress. |
| **`assumptions`** (list of strings) | Used in plan-version metadata; unstructured extraction would conflate assumptions with plan prose. |
| **`changed_surfaces`** (list of file paths) | Drives the deterministic test blast-radius computation in `_derive_plan_test_blast_radius`. Missing or incomplete paths silently degrade test selection. |
| **`test_blast_radius`** (nested dict with `strategy`, `selectors`, `rationale`) | Model-proposed scoping merged with deterministic floor. Losing this structure means the system can only use the floor—no model-scoped narrow baselines. |
| **Revise-specific: `changes_summary`, `flags_addressed`** | `flags_addressed` drives `update_flags_after_revise()` and gate-transition logic. `changes_summary` populates history and receipts. Neither has a natural Markdown home. |
| **Revise-specific: `delta_from_previous_percent`** | Computed by `compute_plan_delta_percent()` comparing old vs. new plan text; stored in meta.json. Pure Markdown draft has no slot for this harness-computed field. |
| **Revise-specific: cache-hit guard** | `_write_plan_version` compares new plan hash against prior; byte-identical content raises `cache_hit_suspected`. This guard operates on the plan text but the detection + error path lives alongside metadata handling. |
| **Revise-specific: prior blast-radius carry** | When the model omits `test_blast_radius` in revise, the handler carries forward the prior version's blast radius. This state-machine behavior lives in the handler, not the draft. |
| **Imported decision criteria merging** | `_merge_imported_decision_criteria` injects harness-computed criteria derived from `imported_decisions` in state. These must survive promotion into meta.json. |
| **Plan-structure validation** | `validate_plan_structure(plan_text)` runs on the Markdown string, but the validation itself is triggered inside `_write_plan_version`, which writes both `.md` and `.meta.json`. Moving to Markdown-only doesn't break validation itself, but does break the paired-artifact model that records `structure_warnings` in meta. |
| **Verifiability flags** (`_build_verifiability_flags`) | Consumes structured `success_criteria` list; produces flag objects stored in the flag registry. Markdown-only eliminates the structured input this function needs. |

In short: the handler currently consumes a **structured JSON payload** whose `plan` field happens to be Markdown. The other fields are first-class structured data that drive state transitions, test selection, flag management, gate validation, and history. A Markdown-only draft would require the harness to parse all that structured data out of prose—a regression with no compensating benefit.

The following clarification should be added to `docs/arnold/megaplan-boundary-turn-design.md`:

1. **Stage Coverage table, `plan` row (line 195)**: append to the existing cell:
   > "The structured payload contains `plan` (Markdown string), `questions`, `success_criteria`, `assumptions`, `changed_surfaces`, and `test_blast_radius`. BoundaryTurn must capture this as a validated structured payload; `plan_vN.md` is promoted from `payload["plan"]` and `plan_vN.meta.json` from the remaining fields plus harness-computed additions (blast radius, imported criteria, structure warnings)."

2. **Stage Coverage table, `revise` row (line 196)**: append:
   > "Revise adds `changes_summary`, `flags_addressed`, and carries `test_blast_radius` from the prior version when the model omits it. Delta tracking and cache-hit detection are harness-computed during promotion and stored in `plan_vN.meta.json`."

3. **Preservation Checklists, Plan And Revise (lines 338–346)**: add an explicit sub-bullet:
   > "- `plan_vN.meta.json` must contain every field listed above; a test must verify that switching to Markdown-only would break at least 8 of the 10 metadata consumers enumerated in the metadata-at-risk analysis."

### Acceptance tests to add (migration step 4, line 251)

| Test ID | Description |
|---|---|
| `test_plan_boundary_preserves_structured_metadata` | Run `handle_plan`. Assert `plan_vN.meta.json` exists and contains keys: `questions`, `success_criteria`, `assumptions`, `test_blast_radius`, `hash`, `version`, `timestamp`, `structure_warnings`. |
| `test_plan_boundary_rejects_markdown_only_draft` | Simulate a model output that is pure Markdown (no structured wrapper). Assert BoundaryTurn validation fails with a classification of `invalid` and a diagnostic referencing missing required fields. |
| `test_revise_boundary_preserves_flags_addressed` | Run `handle_revise`. Assert `plan_vN.meta.json` contains `changes_summary` and `flags_addressed`; assert `update_flags_after_revise` was called and flag registry mutated. |
| `test_revise_boundary_carries_prior_blast_radius` | Run revise with model omitting `test_blast_radius`. Assert meta.json blast radius equals prior version's blast radius. |
| `test_revise_cache_hit_still_detected` | Feed revise a worker whose plan text hashes identically to the prior version. Assert `cache_hit_suspected` is raised even under BoundaryTurn. |
| `test_plan_imported_criteria_survive_boundary` | Run plan with `imported_decisions` in state. Assert merged criteria appear in `plan_vN.meta.json` success_criteria. |

These tests ensure that when BoundaryTurn wraps plan/revise (migration step 4), the structured-payload contract is preserved and no metadata consumer silently degrades. The Markdown-draft path remains a future possibility gated on achieving metadata parity—but it is not the next step.
