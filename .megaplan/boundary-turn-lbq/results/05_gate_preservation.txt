## 1. Verdict

**Yes, with one qualification.** The design doc's preservation checklist (lines 348–358) explicitly names every observable gate behavior, and the multi-stage semantics (lines 213–216) correctly specify reprompt draft reuse and seed capture. Nothing is omitted. The qualification is that the reprompt loop's two-attempts-one-canonical-write pattern creates mild tension with `BoundaryTurn`'s linear `capture → validate → promote` API, requiring the handler to call `validate()` without `promote()` on the first attempt — a pattern the design text supports (line 117: "Validation runs before state or canonical artifacts are touched") but the example API does not illustrate.

## 3. Specific Design-Doc Edits and Acceptance Tests

### Edit 1 (lines 213–216, after "reprompt attempts")

Add:

> The handler opens a turn for each attempt but calls only `validate()`, not `promote()`, on intermediate attempts. Only the final turn's `promote()` writes `gate.json` and `gate_carry.json`. Reprompt auto-downgrade (PROCEED→ITERATE after failed reprompt) remains handler logic outside BoundaryTurn.

### Edit 2 (preservation checklist, add to lines 348–358)

> - Gate reprompt auto-downgrade path (lines 967–981 of gate.py): when both attempts fail to resolve blocking flags, the recommendation is downgraded to ITERATE with appended rationale BEFORE canonical writes — this ordering must survive conversion.

### Acceptance Tests

1. **Two-turn gate with reprompt:** Model emits PROCEED on first attempt but leaves blocking flags. Handler opens `BoundaryTurn` → `validate()` → detects blocking flags, opens second `BoundaryTurn` with same draft path → `validate()` → still blocking → auto-downgrades to ITERATE → calls `promote()` ONCE. Assert `gate.json` contains `"recommendation": "ITERATE"` with appended downgrade rationale, `gate_carry.json` exists, and `gate_signals_vN.json` was written before turn 1.

2. **Single-turn PROCEED gate:** Model emits valid PROCEED with all flags resolved. Handler opens `BoundaryTurn` → `validate()` → no blocking flags → `promote()`. Assert exactly one `gate.json` written, `gate_carry.json` written, debt registry updated, flag events emitted, `last_gate` synced.

3. **Invalid recommendation fallback preserved:** Model emits `"recommendation": ""`. Assert `_normalize_gate_payload` infers PROCEED/ITERATE/ESCALATE from signals and the inferred recommendation reaches `gate.json`.

4. **No-progress streak terminates loop:** Two consecutive ITERATE rounds with no flag closure. Assert `_critique_terminate_branch` sets `STATE_BLOCKED` (with blocking correctness flag) or `STATE_GATED` (cosmetic only), and `gate.json` reflects the termination.

5. **Modified-invalid scratch = hard failure:** Hermes agent writes non-JSON to `gate_output.json`. Assert `ValueError` raised before any canonical artifact is written.
