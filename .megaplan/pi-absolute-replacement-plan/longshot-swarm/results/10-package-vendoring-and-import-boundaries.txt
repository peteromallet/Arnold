## Findings: Hermes/Shannon/Fan-Caller Deletion Boundaries

### 1. **pyproject.toml — Wheel Packages & Console Scripts**

The wheel ships three top-level packages (`["arnold", "arnold_pipelines", "agentbox"]`) and two console scripts (`arnold`, `agentbox`). Deleting Hermes/Shannon leaves no clean package-level carve-out — the vendored runtime IS `arnold/agent/`, so you can't exclude it from the wheel without breaking the whole `arnold` package. The `[agent]` optional-dependency extra is a documented no-op back-compat alias; removing it would break any external scripts/docs referencing `pip install '.[agent]'`.

### 2. **`arnold/agent/` — The Vendored Runtime Boundary**

The entire `arnold/agent/` tree IS the Hermes runtime: `run_agent.py`, `hermes_state.py`, `hermes_time.py`, `hermes_constants.py`, `hermes_cli/`, plus 33 shim files under `tools/`, `agent/`, `hermes_cli/` that dynamically import `arnold_pipelines.megaplan.*` via `importlib.import_module`. The `__init__.py` pre-registers `DeepSeekAdapter` (key: `"hermes"`) and `ShannonAdapter` (keys: `"claude"`, `"shannon"`) at module import time.

**Critical risk**: The `test_import_leak.py` boundary test documents that while the static surface is clean (zero `from arnold_pipelines.megaplan` in `arnold/agent/`), the 33 shim files create dynamic coupling. The test is marked `xfail` pending T5-T11 tool-layer migration. Deleting Hermes without also removing these shims would cause import-time failures.

### 3. **Multiple Competing Import Resolution Paths**

The same `(AIAgent, SessionDB)` runtime is imported through 4+ different paths:

| Caller | Resolution strategy |
|---|---|
| `workers/hermes.py::_import_hermes_runtime()` | `sys.path` manipulation → bare `from run_agent import AIAgent` |
| `skills/subagent-launcher/launch_hermes_agent.py` | 4-tier fallback: legacy `megaplan.agent` → `arnold.agent.run_agent` → vendored checkout → historical layout |
| `skills/subagent-launcher/fan.py` | Same 4-tier fallback + `sys.modules` identity hack |
| `arnold/agent/adapters/deepseek.py` | Direct `from arnold.agent.run_agent import AIAgent` |
| `arnold/agent/adapters/shannon.py` | Lazy `from arnold_pipelines.megaplan.workers.shannon import run_shannon_step` |
| `agent_adapters/shannon.py` (megaplan-level) | Same lazy import from megaplan workers |

There are **two parallel `ShannonAdapter` classes** — one at `arnold/agent/adapters/shannon.py` and one at `arnold_pipelines/megaplan/agent_adapters/shannon.py`. Both must be deleted or merged.

### 4. **Fan-Out / Fan-Caller Tangle**

Fan-out is spread across five modules with overlapping concerns:

- `_core/hermes_fanout.py` — Hermes-specific `scatter_gather`, `scatter_gather_processes`, `scatter_gather_checks`
- `_core/worker_fanout.py` — Generic worker fan-out dispatching through **Shannon, Codex, AND Hermes**
- `_core/process_fanout.py` — Process-level fan-out
- `skills/subagent-launcher/fan.py` — Standalone in-process fan (ThreadPoolExecutor + shared SessionDB)
- `agent_runtime/` — The "public vendorable" contract surface re-exporting `scatter_agent_units`, used by both hermes and worker fanout paths

`worker_fanout.py` explicitly documents that it drives "Claude (via Shannon interactive tmux), Codex, and Hermes." Deleting Hermes/Shannon requires untangling this tri-agent dispatch.

### 5. **Vendoring Docs & Audit Infrastructure**

- `docs/hermes-vendoring.md` — documents git subtree commands, tree audit, and fresh-environment smoke
- `arnold_pipelines/megaplan/audits/hermes_vendoring.py` — programmatic audit checking `hermes_cli/`, `hermes_state.py`, `run_agent.py` as **required** entries, scanning for import sites of `gateway/`, `cron/`, `honcho_integration/`
- `tests/archive/m5/pipelines/megaplan/audits/test_hermes_vendoring_audit.py` — archived test for the audit
- The audit defines `DEAD_WEIGHT_PATTERNS` that must stay absent — removing Hermes means flipping this entire audit inside-out

### 6. **Compatibility Adapters That Bridge Old/New**

- `_compatibility.py` — DSL→neutral pipeline projection referencing `COMMAND_HANDLERS` from the megaplan CLI (which drives Hermes phases)
- `store/compat.py` — `ArnoldStoreAdapter` bridging old dict-based API to new Store seam; not directly Hermes-coupled but part of the same adaptation layer

### 7. **agentbox & Shannon**

`agentbox/tmux.py` likely ties into Shannon's tmux session management. The `agentbox` console script is separately shipped in the wheel. Deleting Shannon requires auditing agentbox for tmux session dependencies.

### 8. **Installed-Artifact Risks**

- Wheel installs ship `arnold/agent/` in its entirety. After source deletion, stale `.pyc` files and the wheel's RECORD would still reference deleted modules until `pip uninstall arnold && pip install -e .`
- The `[agent]` extra no-op must be removed from pyproject.toml in sync with doc updates
- The `test_import_leak.py` xfail test will start passing (or need removal) once the 33 shim files are gone
- `conftest.py` markers include `"asyncio: async tests, primarily for the vendored agent runtime"` — must be cleaned up

### Recommended Deletion / Import-Graph Scans

1. **`rg -l "from arnold\.agent\.(run_agent|hermes_state|hermes_time|hermes_constants)"`** — find all direct Hermes imports (32 files found above)
2. **`rg -l "import (run_agent|hermes_state)\b" --glob '*.py'`** — find bare imports that rely on sys.path manipulation (14 files)
3. **`rg -l "from arnold_pipelines\.megaplan\.workers\.(shannon|hermes)"`** — find Shannon/Hermes worker callers (25 files for Shannon, fewer for Hermes)
4. **`rg -l "DeepSeekAdapter|ShannonAdapter" --glob '*.py'`** — find adapter registrations
5. **`rg -l "hermes_fanout|worker_fanout|process_fanout" --glob '*.py'`** — find fan-out callers
6. **`python -c "from arnold_pipelines.megaplan.audits.hermes_vendoring import audit_vendored_agent_tree; print(audit_vendored_agent_tree(Path.cwd()))"`** — run the existing audit to inventory the current vendored surface before deleting it
7. **`pip install -e . && pip show -f arnold | grep -E 'arnold/agent/(run_agent|hermes_)'`** — verify what the installed wheel actually ships under `arnold/agent/`
