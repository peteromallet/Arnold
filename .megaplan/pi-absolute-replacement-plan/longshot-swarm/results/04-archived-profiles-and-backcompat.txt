## Answer: Are Archived Profiles/Run Artifacts an External Compatibility Surface?

**No.** They are historical data only. Specifically:

1. **No external tooling reads them.** They live under `docs/archive/m5/`, a documentation-only path.
2. **The spec format is forward-compatible** ‚Äî `agent:model[:effort]` still parses. But the stage keys are wrong for the system-profile path.
3. **The `abstraction_level_validator` is subsumed** by the generic `stage_value_validators` mechanism. The archived copy is dead; the test that imports it is stale/broken.
4. **No run artifacts exist** that embed serialized profile references needing migration.

The only compatibility concern is internal: if someone tried to load an archived profile TOML as a **system profile** (Layer 3), they'd get a `ProfileLoadError` ("unknown declared stage prefix"). As a **pipeline-local profile** (Layer 2), it would work fine.
