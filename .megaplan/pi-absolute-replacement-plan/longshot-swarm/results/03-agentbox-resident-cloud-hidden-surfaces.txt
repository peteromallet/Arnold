### What's Missing: 8 Concrete Gaps

**1. AgentBox credential backend (`agentbox/credentials/`)**

Epic 8 mentions migrating Codex/Hermes OAuth shims *in cloud auth*, but `agentbox/credentials/registry.py` + `backend.py` is a separate, full credential subsystem with its own CLI (`agentbox credential push/test/list`). This is a distinct credential store that the facade must either absorb or route through.

**Edit:** Add to Epic 8 scope: "Inventory and reconcile `agentbox/credentials/` backend with facade-owned credential mediation. Ensure `agentbox credential *` CLI commands route through the facade's credential layer rather than maintaining a parallel credential store."

**2. AgentBox CLI as its own agent surface (`agentbox/cli.py`, `agentbox/__main__.py`)**

The `agentbox` CLI is not just an adapter — it's a standalone command surface (`bootstrap`, `run`, `status`, `logs`, `attach`, `cleanup`, `reconcile`, `guardian run`, `doctor checkup`, `notify test`, `services`). The Guardian runs via `python -m agentbox guardian run` and the Discord resident via `python -m arnold_pipelines.megaplan resident discord` — neither goes through any facade. Epic 8 only mentions the *adapter*, not the CLI as its own surface.

**Edit:** Add: "Route `agentbox` CLI commands (guardian run, services, cleanup, reconcile) through facade telemetry, run records, and credential mediation, or scope them out explicitly."

**3. AgentBox durable operations store as parallel state (`agentbox/operations.py`, `arnold.runtime.durable_ops`)**

AgentBox maintains its own file-backed operation state machine with `OperationState`, scheduled tasks, typed resources, and locks. Epic 8 says "AgentBox/resident launches produce facade run records" but doesn't address whether the AgentBox operation store remains authoritative or becomes a facade-backed view. Two state stores with overlapping semantics is a divergence risk.

**Edit:** Add: "Define whether the AgentBox operation store integrates with, delegates to, or remains separate from the facade run-record store. If separate, document the boundary and ensure `agent history` / `agent doctor` can observe both."

**4. The `resident/subagent.py` bypass**

`resident/subagent.py` calls `launch_hermes_agent.py` as a subprocess directly — this is a resident surface, NOT a cloud wrapper. Epic 8's gate only mentions deleting direct `launch_hermes_agent.py` calls "from cloud wrappers."

**Edit:** Expand gate: "Direct `launch_hermes_agent.py`, `codex exec`, or raw Claude calls are deleted from **both** cloud wrappers and resident tool dispatch after parity."

**5. Systemd lifecycle hooks and `ensure-megaplan-watchdog`**

Four systemd units + one path unit + one timer + the `ensure-megaplan-watchdog` script form the host-side lifecycle surface. These contain template variables (`{{WORKING_DIRECTORY}}`, `{{WORKSPACE_ROOT}}`, `{{PYTHON}}`) that need a bootstrap path. `ensure-megaplan-watchdog` restarts watchdogs via `docker exec ... tmux` — it's itself an unattended lifecycle hook that could bypass facade governance.

**Edit:** Add: "Inventory systemd units and `ensure-megaplan-watchdog` script. Ensure lifecycle hooks observe facade kill/restart policy and produce audit records. Document template-variable bootstrap path for clean-runner install."

**6. `scripts/megaplan_live_watchdog.py` — separate Python watchdog**

This is a distinct, Python-native watchdog supervisor (separate from the bash `arnold-watchdog` wrapper) that uses Arnold's execution engine directly (`from arnold.execution import run`), maintains its own NDJSON registry, has its own repair runner and retry loop. Epic 8 mentions "watchdog" generically but may assume only the bash wrapper.

**Edit:** Add: "Inventory `scripts/megaplan_live_watchdog.py` alongside cloud wrapper watchdog. Ensure both watchdogs route through facade or are scoped out with owner sign-off."

**7. AgentBox notification system (`agentbox/notify.py`)**

AgentBox has its own notification dispatch (`notify_test` CLI command). If the facade absorbs observability, this notification path should integrate rather than remain a parallel channel.

**Edit:** Add: "Route AgentBox notifications through facade telemetry or scope out explicitly."

**8. `agentbox/github.py` — CI-status and PR recording**

`agentbox/github.py` provides `ci_status_for_branch` used by the `MegaplanChainHandler.tick()` method to record CI status and PRs into operations. This is a GitHub integration surface that Epic 8 doesn't mention.

**Edit:** Add: "Ensure AgentBox GitHub integration (CI status, PR recording) produces facade-observable records."
