## Consumer Breakage Audit

### 1. Receipt Schema vs. Emitted Fields (Contract Gap)

**Evidence**: `receipts/schema.py:11-36` declares `class Receipt(TypedDict)` with 26 fields. `receipts/__init__.py:100-103` silently appends **four extra keys** not declared in the TypedDict: `shannon_plan`, `worker_channel`, `auth_channel`, `auth_metadata`.

**Who breaks**: Any code that type-narrows against `Receipt`, iterates receipt keys from the schema, or serializes receipts expecting the TypedDict shape. No current production consumers do this (they read via `getattr(worker, ...)` at emit time), but the schema is the documented contract.

**Missing gate**: `Receipt` TypedDict must add those four keys, or `build_receipt` must stop emitting them. Currently they ride through as untyped dict spillover.

### 2. COST_RECORDED Payload Contract — 5 Production Consumers

**Emitters** (2 sites):
- `workers/hermes.py:2583-2597` emits `{request_id, cost_usd, provider, model}`
- `workers/_impl.py:2203-2213` (`_emit_codex_execute_cost_recorded`) emits `{request_id, cost_usd, provider, model}`

**Consumers** (5 production readers, all reading `payload.get("cost_usd")` and `payload.get("model")`):
| Consumer | Fields Read | Break If |
|---|---|---|
| `observability/cost.py:126-147` | `model`, `cost_usd`, `request_id`, `provenance` | Rename/remove any of these |
| `observability/introspect.py:621-624` | `cost_usd` | Renamed |
| `observability/doctor.py:357-358` | `cost_usd` | Renamed |
| `observability/trace.py:286-289` | `cost_usd`, `provider`, `model` | Renamed |
| `auto.py:136` | Includes COST_RECORDED in `STALL_PROGRESS_EVENT_KINDS` (kind-only) | Removing the kind constant |

**Missing gate**: No schema declaration for `COST_RECORDED` payload shape. If a third emitter starts producing `COST_RECORDED` without `model` or `cost_usd`, the defensive `float(payload.get("cost_usd", 0) or 0)` in cost.py won't crash but will silently zero out costs. A typed `CostRecordedPayload` / TypedDict would catch this at static-analysis time.

**Consumers**:

**Missing gate**: `cache_read_tokens` is consumed in `cost.py:171` but never emitted — it's a dead read that silently returns 0. If a future emitter starts populating it, there's no contract documenting it. Same for `cost_usd`/`duration_s` in trace.py's LLM_CALL_END handler.

### 4. WorkerResult Fields Flow — Untracked Spillover

`workers/_impl.py:660-679` defines `WorkerResult` with 16 fields including `shannon_plan`, `rate_limit`, `worker_channel`, `auth_channel`, `auth_metadata`. The `from_agent_result`/`to_agent_result` projections at lines 682-730 carry all fields.

**Receipt path**: `build_receipt` reads `shannon_plan`, `worker_channel`, `auth_channel`, `auth_metadata` via `getattr(worker, ...)` — but `Receipt` TypedDict doesn't declare them (see #1).

**Missing gate**: If `WorkerResult` adds a field, `build_receipt` won't pick it up unless explicitly wired. There's no test that asserts all `WorkerResult` fields round-trip through the receipt construction path.

### 5. events.ndjson Envelope Schema — Unversioned Consumer Assumptions

The event envelope (`events.py:315-323`) pins `schema_version=1`. Consumers read:
- `fold.py:59` reads `kind == "state_written"` → `payload["state"]`
- `cost.py:120-191` reads `kind`, `phase`, `payload.{model,cost_usd,tokens_in,tokens_out,request_id,cache_read_tokens}`
- `trace.py` reads `kind`, `ts_utc`, `phase`, `payload.{phase,model,provider,tokens_in,tokens_out,cost_usd,duration_s,...}`
- `auto.py:130-138` reads `kind` only for stall-detection set membership
- `events_projection.py:37-51` projects `{seq, schema_version, ts_utc, kind, phase, payload, transaction_id, store_method, run_id?}`

All consumers use defensive `.get()` patterns, so adding fields won't break. **Removing or renaming** `seq`, `kind`, `ts_utc`, or `phase` would break all consumers.

### 6. Test Coverage Gaps

- `tests/.../test_native_golden_manifest.py:648-672` asserts `receipt_metrics` obligations (non-empty list of strings) but **does not** assert the full `Receipt` shape.
- `tests/.../test_handler_behaviors.py:147-151` constructs `WorkerResult(cost_usd=0.0, duration_ms=0)` — never asserts `shannon_plan`, `rate_limit`, or the three channel fields.
- **No test** asserts the `COST_RECORDED` or `LLM_CALL_END` payload shapes emitted by any worker match what consumers read.
- **No test** asserts `build_receipt` output matches the `Receipt` TypedDict.
