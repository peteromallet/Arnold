## 1. DeepSeek/Subagent/Hermes Mechanics Already in Repo

### Subagent Launcher (highest leverage)
**`subagent-launcher/SKILL.md`** + companion scripts — **fully reusable as-is**. Three dispatch pathways:

| Pathway | Invocation | Model | Best for |
|---|---|---|---|
| **Hermes agentic** (default) | `launch_hermes_agent.py --toolsets="file,web,terminal"` | DeepSeek V4 Pro/Flash, Kimi K2.7, Zhipu GLM, MiMo | Tool-using subagents with repo access |
| **Codex** | `codex exec --sandbox <level>` | GPT-5.5 | Sandboxed implementation |
| **Claude** | `launch_claude_agent.py --model=opus` | Claude Opus/Sonnet | Same-family isolation |

**Key mechanics already built:**
- **Fan-out** (`fan.py`): N agents in one process, shared import (~5-15× RAM savings vs N subprocesses). Briefs directory → per-task output + `_report.json`.
- **Escalation ladder**: MiMo (cheapest, fast scans) → DeepSeek Flash (non-reasoning, coding-tuned) → DeepSeek Pro (judgement) → Codex/Claude (complexity). "Start low, escalate on evidence" rule coded into tool selection.
- **Liveness detection**: stderr heartbeat (`[tool]`/`[done]` every 1-5s), `</dev/null` anti-wedge for Codex, `--task-timeout=1800` default.
- **Multi-lens jury**: Fan same prompt to Codex + DeepSeek + Kimi for high-stakes calls; divergence = signal.
- **Network sandbox awareness**: Codex's `CODEX_SANDBOX_NETWORK_DISABLED=1` documented; Hermes agents need outbound network to provider APIs — `danger-full-access` required for Codex-orchestrated Hermes subagents.

**File refs**: `~/.claude/skills/subagent-launcher/launch_hermes_agent.py`, `fan.py`, `launch_claude_agent.py`
