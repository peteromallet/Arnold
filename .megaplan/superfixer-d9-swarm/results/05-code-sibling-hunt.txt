### 1. PRIMARY BUG: `TypeError` in `_attempt_summary` → `build_investigation_context`

**File**: `/workspace/arnold-consolidation-20260714/arnold_pipelines/megaplan/cloud/repair_investigation.py`  
**Function**: `_attempt_summary` (lines 64–84), called by `build_investigation_context` at line 323

**Root cause** — line 81:
```python
"validation": [_text(item, 1000) for item in (report.get("validation") or [])[-8:]],
```

The `or []` short-circuit **only fires when `report.get("validation")` is falsy** (`None`, `""`, `[]`, `{}`, `0`, `False`). If a dev-fix agent writes `"validation": {"status": "passed", "tests": [...]}` (a **non-empty dict**), then `report.get("validation")` is truthy, `or []` does **not** trigger, and `[-8:]` is applied to a dict — raising `TypeError: unhashable type: 'slice'`.

Identical sibling risk at **lines 78–79**:
```python
"what_tried": [
    _text(item, 1000)
    for item in (value.get("dev_summary") or report.get("what_tried") or [])[-8:]
],
```
Same pattern: if `dev_summary` or `what_tried` is a non-empty dict/string (truthy but not sliceable-as-list), the same TypeError fires.

**Trigger chain**:
1. A dev-fix model writes a report with `"validation"` as a dict shape (perfectly reasonable — many models output structured validation objects)
2. The repair-loop wrapper persists it at `dev_report.validation` (line 4394)
3. Next repair iteration calls `build_investigation_context`
4. Line 323: `"prior_repairs": [_attempt_summary(item) for item in attempts[-max_prior_attempts:]]`
5. `_attempt_summary` hits the dict with `[-8:]` → **TypeError**
6. Context build fails → `run_repair_investigator_turn` sets `fixer_infrastructure_failure` (lines 6184–6188)
7. `require_investigation_before_mutation` fails closed (correctly), but repair loop is dead

### 5. SMALLEST FAIL-CLOSED FIX DESIGN

**Change one function**: `_attempt_summary` in `repair_investigation.py`, lines 64–84.

**Design principle**: Coerce non-list values to `[]` before slicing, so malformed shapes are silently treated as "no evidence" rather than crashing. This is fail-closed: no wrong evidence leaks through, and the context still builds successfully (just with empty validation/what_tried fields).

**Exact fix** (replace lines 77–81):

```python
# BEFORE (lines 77-81):
        "what_tried": [
            _text(item, 1000)
            for item in (value.get("dev_summary") or report.get("what_tried") or [])[-8:],
        ],
        "validation": [_text(item, 1000) for item in (report.get("validation") or [])[-8:]],

# AFTER:
        "what_tried": [
            _text(item, 1000)
            for item in (
                value.get("dev_summary")
                if isinstance(value.get("dev_summary"), list)
                else report.get("what_tried")
                if isinstance(report.get("what_tried"), list)
                else []
            )[-8:]
        ],
        "validation": [
            _text(item, 1000)
            for item in (
                report.get("validation")
                if isinstance(report.get("validation"), list)
                else []
            )[-8:]
        ],
```

**Why this preserves authority/review guards**:
- No new exceptions raised — context still builds
- No schema changes — output shape is identical for valid inputs
- No widening of accepted evidence — non-list validation/what_tried is discarded, never reinterpreted
- `_text()` already stringifies items safely — no new failure mode introduced
- The `"prior_repairs"` list is informational/narrative evidence for the investigator, not an authorization gate — omitting validation items from a prior attempt is strictly less risky than crashing the entire investigation

**Bonus guard** (optional, same function): Add an `isinstance` check on the `exact_error` fallback at lines 306–309 to ensure non-dict truthy values are caught:

```python
# Line 306-309 could add:
        "exact_error": (
            current.get("latest_failure")
            if isinstance(current.get("latest_failure"), Mapping)
            else phase_result if phase_result_blocked else {}
        ) or frozen_checkpoint.get("latest_failure") or {},
```

This prevents a truthy string `latest_failure` from being used as a dict downstream.
