# C2: Model Seam — Relocate to Generic + Wire Live

Condenses old m3 (the over-invested centerpiece). Bases off `arnold-generalized-pipeline @
944d10cc`. The migration already BUILT essentially the entire model-seam machinery — but it lives
in `arnold/pipelines/megaplan/model_seam.py` (megaplan-coupled, 1694 lines) and the model adapter is
NOT installed into the live invocation path. C2 relocates the family-agnostic core to generic
`arnold`, wires it live, and closes the residual coverage gaps.

## Outcome

Take the step⇄model seam (and the engine⇄worker capability sub-seam) onto the contract as a GENERIC
platform primitive, and make the degraded path explicit, bounded, observable, and never trusted
unless validated — without deadlocking a worker. The acute pain (Hermes-no-structured on execute,
Shannon never `--json-schema`, Codex `--output-schema` fresh-only) is already handled by the built
two-tier-trust + always-on-audit design; C2's job is to make that machinery a platform primitive and
actually live, not megaplan-private and dormant.

Three pieces of real work:

1. **Relocate the family-agnostic core to generic `arnold/pipeline/`.** The tokenizer table, budget
   math, the always-on structural audit, `RenderedStepMessage`, and the `StepInvocation` MODEL
   ADAPTER are platform concerns and belong next to the contract (token-budget ADJACENT to the
   contract, per the dossier). Only the megaplan prompt-key / worker-dispatch / step-name specifics
   stay in `arnold/pipelines/megaplan/`.
2. **Wire the model adapter into the live executor/dispatch path.** `install_model_step_adapter`
   exists in `model_seam.py` (registered name in `arnold/pipelines/megaplan/__init__.py:170`) but
   has NO caller — the model adapter is built but never installed into the live `StepInvocation`
   flow, so the seam's `model` slot still fails closed at runtime. C2 installs it so model steps
   actually route render→dispatch→capture through the adapter.
3. **Close the residual coverage + budget-correctness gaps**: the OWN-prompt-assembly coverage audit
   (every OWN model dispatch routes through `render_step_message`), the COMBINED-input budget
   (system+history+prompt+tools, not the bare prompt — the Hermes multi-turn gap), the DONT_TOUCH
   vendored-boundary tripwire, and the advisory-evidence projection cap at the SOURCE.

## Scope

IN:

- **Relocate** the model-seam core into generic `arnold/pipeline/` (proposed `arnold/pipeline/
  model_seam.py` or a model-adapter module): the local-first per-family tokenizer table (tiktoken
  `o200k_base` for GPT/Codex; HF `AutoTokenizer` for DeepSeek/Kimi/GLM; conservative byte-estimate
  `ceil(utf8/3)+20-25%` CJK-safe fallback), the budget math (`ModelBudget`/`ModelFamily`/
  `BudgetStatus`), the always-on structural audit (`audit_step_payload`/`schema_audits_step_payload`),
  `render_step_message` / `capture_step_output` / `RenderedStepMessage`, and the model `StepInvocation`
  adapter. Keep megaplan prompt-key/worker-dispatch specifics in `arnold/pipelines/megaplan/`.
- **Install the model adapter** into the live executor/dispatch path so a `kind="model"`
  `StepInvocation` no longer hits the reserved fail-closed slot; model steps route
  render→dispatch→capture through the adapter end-to-end. Prove on EXECUTE first (riskiest).
- **Coverage-audit test**: enumerate OWN model-dispatch sites and FAIL if any assembles a prompt
  without routing through `render_step_message`, so a future pipeline cannot silently reintroduce the
  hole. Route the OWN-UNGUARDED sites the dossier catalogued (resident loop, Hermes combined
  prompt+history + summary/repair follow-ups, the JSON-repair builder under Hermes) through the
  chokepoint where they still bypass it.
- **Combined-input budget**: the budget runs on the FULL assembled model input
  (system + history + prompt + tool results), not a bare prompt string — the Hermes
  `run_conversation` multi-turn gap is the template for the whole multi-turn class; the check lives
  where the pieces become `api_messages`.
- **Advisory-evidence projection cap at the source** (a token budget alone does NOT close this):
  large execute "deviation"/audit file-path lists are materialized uncapped as prose and pasted
  forward, then re-inlined into the next batch prompt. Cap+sample them (`"<N> paths (showing K): …
  — full set via ArtifactRef"`) BEFORE they enter the batch artifact and BEFORE re-inlining, so the
  budget operates on signal, not noise. (Proof-case in the dossier: a 337-file rename produced a
  225,059-char prompt that was 77.5% advisory path-list noise, the literal embodiment of the
  "advisory evidence serialized as prose and pasted forward" thesis.)
- **DONT_TOUCH boundary tripwire**: a sentinel test FAILS if the vendored/external boundary moves
  (Shannon starts passing `--json-schema`, Hermes-execute stops disabling `response_format` under
  tools, codex changes its session/resume model), so a declared-out-of-scope boundary cannot silently
  rot into an uncovered hole.
- Two-tier trust + always-on structural audit + one-shot envelope-only repair + media budgets are
  CONSUMED (already built); C2 makes them generic and live, and verifies their behavior post-relocation.

OUT:

- Migration of finalize/critique/review/gate and the long tail — execute is C2's vertical proof; the
  rest is the migration-delta in C4 (largely absorbed already; see C4).
- Suspension-aware composition (C3) and authoring-API enforcement (C4).
- Budgeting accumulation INSIDE a vendored/external session (`codex exec resume` rollout, Shannon/
  Claude tmux transcript, pure process-spawn) — the DONT_TOUCH side; handled by existing session
  rotation/compaction, left intact + observable. We do not claim to budget what we cannot see.
- Any change to the `ContractResult` type, the schema registry, the validator, or the mode ladder.

## Locked Decisions (carried from the prep dossier jury)

- `render_step_message` IS the MODEL ADAPTER of the `StepInvocation` seam — one of {model, tool,
  human, state}, NOT "the only seam." Unknown (non-model) adapter kinds fail closed at the registry.
- Two-tier trust: enforced wire-trust (Codex `--output-schema` fresh/read-only; Hermes
  `response_format` where tools off) + non-enforced prompt-template-plus-structural-audit (Shannon
  always; Hermes-execute; Codex-resume). No worker unusable; none untrusted.
- The structural-type audit runs ALWAYS, including in enforced mode — one uniform validation path,
  no per-tier branching (defense-in-depth; uniform & bulletproof over minimal-redundancy).
- Do NOT force a repair loop on Shannon/Hermes (cost); the structural audit is the universal closer.
  The bounded retry is EXACTLY ONE repair re-ask (envelope-only, no tools/redo) → terminal
  `worker_structural_audit_failed`; hard ceiling 2 model turns = no deadlock; unifies Codex + Shannon.
  (Resolved decision #3.)
- Token budget uses real per-family tokenizers (local-first table, keyed off `resolve_model`'s
  normalized string); fails at assembly time before dispatch; accounts for tool-manifest + large_refs
  overhead. Fail CLOSED on unknown family in enforced mode; fail SAFE (over-estimate + degraded
  telemetry) in non-enforced. (Resolved decision #3.)
- Budget the COMBINED model input (system + history + prompt + tool results), NOT the bare prompt.
- Large data crosses BY REFERENCE (by tool / `ArtifactRef`), not inlined.
- Media budgets (frame count, image resolution, audio seconds, file size), not only text tokens, so a
  vision/multimodal input is budgeted on its media dimensions (B1/N=1 overfit fix).
- DONT_TOUCH TRIPWIRE (pre-mortem risk 6): a sensor sits exactly where we stopped looking; it fires
  if the vendored boundary moves.
- Prove on EXECUTE first (riskiest; Hermes-no-structured).
- **NEW for C2 (relocation discipline):** the family-agnostic core is a PLATFORM primitive
  (`arnold/pipeline/`), token-budget ADJACENT to the contract; only megaplan prompt-key/worker
  specifics stay in `arnold/pipelines/megaplan/`. Relocation must change no behavior (a
  characterization-test-then-move discipline).
