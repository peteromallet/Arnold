## 3. Gap/Risk

**Concrete product consequence:** An extension author cannot package their work as a distributable artifact. There is no file format to declare identity, version, API compatibility range (`apiVersion`), dependencies (`dependsOn`), required permissions, or migration scripts. The platform cannot validate whether an extension is compatible with the host version before loading it, cannot resolve extension-vs-extension dependency conflicts, cannot run schema migrations when extension data formats evolve, and cannot enforce a permissions model (e.g., preventing an extension from calling export APIs it didn't request). This means M1 (basic extension packaging) is unimplemented and M14 (release-readiness compatibility gates) is structurally blocked — you cannot gate release compatibility on a manifest that doesn't exist.

**Dev consequence:** Any attempt to onboard a third-party extension requires manual code integration through React context wiring (`VideoEditorExtensionConfig`), which is neither documented as a public API nor stable across versions. The `@publicContract` annotations in `index.ts` and `browser.ts` do not include extension manifest or package types.
