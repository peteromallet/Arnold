## Megaplan Cloud SSH Provider Audit: Hetzner/SSH Native Persistent-Storage Path

### 1. Current Architecture Summary

```
Local (CLI)                          Remote (Hetzner/SSH host)
───────────                          ──────────────────────────
cloud.yaml ──► materialize_deploy_dir()
                ├── Dockerfile                rsync/scp ──► $remote_dir/
                ├── entrypoint.sh                        (default: /tmp/megaplan-cloud)
                ├── healthserver.py
                └── wrappers/
                                                      docker build -t $container .
                                                      docker run -d --name $container
                                                        --restart unless-stopped
                                                        --env-file .env
                                                        -p $port:$port
                                                        $container
                                                        ↑ NO VOLUME MOUNTS ↑
```

The entrypoint assumes `/workspace` is persistent (clone-if-missing, credential seeds, Claude refresh-token cache, plan state), but **the Docker container has no volume mount**. Every `docker rm -f` + `docker run` cycle wipes all state.

### 2. Hidden Hazards in the Current SSH Provider

| # | Hazard | Location | Severity |
|---|--------|----------|----------|
| **H1** | **No `/workspace` volume mount.** `deploy()` (ssh.py:116-136) runs `docker run -d` with no `-v` flag. All cloned repos, `.venv`, `node_modules`, plan state, auth seeds, chain state, and caches are lost on every redeploy. | `ssh.py:125-136` | **Critical** |
| **H2** | **Auth seeds silently lost.** `auth.py` writes codex/hermes OAuth to `/workspace/.creds/*` via `docker exec`. The `entrypoint.sh.tmpl` (lines 44-55) tries to seed from `/workspace/.creds/*` on boot. Without a volume mount, these never survive a redeploy, making `codex_auth: chatgpt` broken after the first restart. | `auth.py:48-63`, `entrypoint.sh.tmpl:44-55` | **Critical** |
| **H3** | **`remote_dir` defaults to `/tmp/megaplan-cloud`.** `/tmp` is often a tmpfs on Linux/Hetzner — wiped on host reboot. The Docker build context (Dockerfile, entrypoint.sh, wrappers) disappears, so `cloud build` after a host reboot fails silently or with confusing errors. | `spec.py:96` | **High** |
| **H4** | **`destroy()` unconditionally nukes `remote_dir`.** `ssh.py:192-196` does `docker rm -f && rm -rf {remote_dir}`. If `remote_dir` is moved to a persistent path that also holds workspace data, `destroy` would wipe user data. | `ssh.py:194-195` | **High** |
| **H5** | **`destroy()` ignores the `volume` parameter.** `ssh.py:192-193`: `del volume` — the method signature accepts `volume` but discards it. `ResourcesSpec.volume` is never used by the SSH provider. If we add a named Docker volume, `destroy` needs to clean it up. | `ssh.py:192-193` | **Medium** |
| **H6** | **SCP fallback is destructive.** When rsync is unavailable, the scp path does `rm -rf {remote_dir} && mkdir -p {remote_dir}` before copying. The rsync path just syncs incrementally. Inconsistent and destructive. | `ssh.py:95-107` | **Medium** |
| **H7** | **`deploy()` always does `docker rm -f` before starting.** No check for running state. This is fine post-fix (volume will persist), but combined with H1 it means every deploy is a hard reset. | `ssh.py:122-124` | **Low** (becomes acceptable with volume) |
| **H8** | **No health check or restart verification after `docker run`.** The `deploy()` method fires `docker run -d` and returns 0 immediately without verifying the container is healthy. | `ssh.py:125-137` | **Low** |
| **H9** | **`VALID_PROVIDERS` still includes `"railway"`** in `spec.py:22` and `cloud.yaml.tmpl:5`. Needs removal per constraints. | `spec.py:22`, `cloud.yaml.tmpl:5` | **Medium** |
| **H10** | **`cloud.yaml.tmpl` still documents `railway:` section** (lines 123-133) and references Railway in comments (lines 148-150 about `railway ssh --session agent`). | `cloud.yaml.tmpl:123-156` | **Low** |

#### 3.5 `SshProvider.deploy()` (ssh.py:116-137) — Add volume mount

**Before:**
```python
def deploy(self, deploy_dir: Path, *, secrets: dict[str, str]) -> int:
    del deploy_dir
    env_path = f"{self._ssh.remote_dir}/.env"
    env_lines = [f"PORT={self._spec.resources.port}"]
    env_lines.extend(f"{name}={value}" for name, value in secrets.items())
    self._remote_run(f"cat > {shlex.quote(env_path)}", input="\n".join(env_lines) + "\n")
    self._remote_run(
        f"docker rm -f {shlex.quote(self._ssh.container)} >/dev/null 2>&1 || true"
    )
    self._remote_run(
        " ".join(
            [
                "docker run -d",
                f"--name {shlex.quote(self._ssh.container)}",
                "--restart unless-stopped",
                f"--env-file {shlex.quote(env_path)}",
                f"-p {self._spec.resources.port}:{self._spec.resources.port}",
                shlex.quote(self._ssh.container),
            ]
        )
    )
    return 0
```

**After:**
```python
def deploy(self, deploy_dir: Path, *, secrets: dict[str, str]) -> int:
    del deploy_dir
    env_path = f"{self._ssh.remote_dir}/.env"
    env_lines = [f"PORT={self._spec.resources.port}"]
    env_lines.extend(f"{name}={value}" for name, value in secrets.items())
    self._remote_run(f"cat > {shlex.quote(env_path)}", input="\n".join(env_lines) + "\n")

    # Resolve workspace volume name (resources.volume overrides ssh.workspace_volume)
    volume_name = self._spec.resources.volume or self._ssh.workspace_volume

    # Ensure named Docker volume exists (idempotent)
    self._remote_run(f"docker volume create {shlex.quote(volume_name)} >/dev/null 2>&1 || true")

    self._remote_run(
        f"docker rm -f {shlex.quote(self._ssh.container)} >/dev/null 2>&1 || true"
    )
    self._remote_run(
        " ".join(
            [
                "docker run -d",
                f"--name {shlex.quote(self._ssh.container)}",
                "--restart unless-stopped",
                f"--env-file {shlex.quote(env_path)}",
                f"-p {self._spec.resources.port}:{self._spec.resources.port}",
                f"-v {shlex.quote(volume_name)}:/workspace",  # ← NEW
                shlex.quote(self._ssh.container),
            ]
        )
    )
    return 0
```

#### 3.6 `SshProvider.destroy()` (ssh.py:192-197) — Wire volume deletion

**Before:**
```python
def destroy(self, *, volume: str | None = None) -> int:
    del volume
    self._remote_run(
        f"docker rm -f {shlex.quote(self._ssh.container)} >/dev/null 2>&1 || true && rm -rf {shlex.quote(self._ssh.remote_dir)}"
    )
    return 0
```

**After:**
```python
def destroy(self, *, volume: str | None = None) -> int:
    volume_name = volume or self._spec.resources.volume or self._ssh.workspace_volume
    self._remote_run(
        f"docker rm -f {shlex.quote(self._ssh.container)} >/dev/null 2>&1 || true"
    )
    if volume_name:
        self._remote_run(
            f"docker volume rm {shlex.quote(volume_name)} >/dev/null 2>&1 || true"
        )
    # remote_dir is the build context on the host, safe to remove
    self._remote_run(f"rm -rf {shlex.quote(self._ssh.remote_dir)}")
    return 0
```
