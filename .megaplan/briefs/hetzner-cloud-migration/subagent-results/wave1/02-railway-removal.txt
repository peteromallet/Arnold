## Likely breaking tests/import surfaces

- **No test breakage expected.** The cloud test file (`tests/cloud/test_phase_command_shim.py`) has zero Railway references. `test_import_surface.py` tracks `megaplan.store`, `megaplan.workers`, `megaplan.cli`, `megaplan.chain`, and `megaplan.orchestration.evaluation` — none of which touch Railway. `RailwayProvider`/`RailwaySpec` are not in any tracked import surface.
- **User-facing break:** Any existing `cloud.yaml` with `provider: railway` will hit the `VALID_PROVIDERS` validation and get a clear error: `"provider must be one of local, ssh, fly; got 'railway'"`. That's the right behavior — no silent degradation.
- **`supports_session` becomes dead code** (only `RailwayProvider` set it to `True`). The `--session` CLI flag will always error with the existing message. No test asserts `supports_session=True` anywhere.
