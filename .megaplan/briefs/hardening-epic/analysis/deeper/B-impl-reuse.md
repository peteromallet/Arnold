## Claims in the design docs the code does NOT support

1. **`worker_did_work` reads `cli_provenance` for tool calls** (claude doc §2, line 88; codex doc §2). FALSE — `cli_provenance` is config-key snapshotting only (`shared.py:325`). The tool-call/diff signal lives in delegate `tool_trace` and `execution_batch_*.json`. The evidence provider must target the batch artifact, not PhaseResult.cli_provenance.
2. **`landed_diff` = `validate_execution_evidence` + `git diff --stat <base_ref>..HEAD`** (claude §2). `validate_execution_evidence` uses `git status` (working-tree) via `_collect_git_status_paths_with_nested_repos` (`:121`), NOT a base-ref diff. There is no base-ref/checkpoint capture today; `CompletionContext.git_base_ref`/checkpoint-at-start must be **built new**. The hollow-done check is the real abandonment catch, not a base-ref diff.
3. **"reuse `_capture_test_baseline` as the green-suite runner"** (both docs). Partially true — only the *runner* is reusable; baseline capture never re-runs or compares, so the green-suite *comparison* is genuinely new (docs acknowledge this for the gate, but both imply heavier reuse than exists).
4. **Declared no-op already has a home** (codex §6 implies `completion/noop.json` pattern, claude §3 cites "finalize.json:tasks[2].deferral"). No such field/artifact exists — `deferred_human` is criterion-scoped only. Must-build-new.

---

### 5-line summary

1. **2 of 6 evidence classes are real reuse:** landed-diff wraps `validate_execution_evidence` wholesale (phantom/hollow-done checks already nail the abandonment case); phase-coverage composes `PhaseResult` + `_latest_execution_batch_all_tasks_done` + `classify_criteria` with no new data.
2. **3 of 6 need extension of existing primitives:** green-suite reuses only the `_capture_test_baseline` runner (re-run + baseline-diff is new); worker-did-work reuses `tool_trace`/`execution_batch` activity (NOT `cli_provenance`); review-disposition is a new consumer of existing `review.json`/the silent force-proceed at `review.py:251`.
3. **1 of 6 is must-build-new:** the typed `Waiver` / declared-no-op artifact — nothing structured exists beyond criterion-level `deferred_human` and free-text `executor_notes`.
4. **The contract scaffolding (CompletionContract/Evidence/CompletionContext + base-ref checkpoint capture + the verifier wiring at `auto.py:1363`, `chain:1418`, `execute.py:211`, `review.py:248`) is all new code**, but every gather step except no-op-waiver and base-ref-diff bottoms out in an existing helper.
5. **Net: ~60% reuse, ~40% new** — heaviest correction is that worker-activity and landed-diff signals do NOT live where the design docs say (`cli_provenance`/`git diff base..HEAD`); they live in `execution_batch_*.json` and `git status`.
