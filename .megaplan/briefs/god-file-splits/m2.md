# Split comfy_nodes/agent/edit.py god-file (finish structural-decomposition m5-agent-edit)

## Outcome

Decompose `vibecomfy/comfy_nodes/agent/edit.py` (7236 LOC -- the largest file in the repo) into focused submodules (~<=700 LOC each) within `vibecomfy/comfy_nodes/agent/` -- **finishing** what `structural-decomposition` m5-agent-edit started (it moved the file into the `agent/` package and split some peers, but `edit.py` itself stayed/grew to 7236 LOC). The public surface (`vibecomfy.comfy_nodes.agent.edit` + its re-exports + the headless entry path) stays byte-for-byte identical. Agent-edit behavior stays byte-identical.

## Scope

**IN:** split `comfy_nodes/agent/edit.py` by pipeline stage/concern -- the agent-edit handler + `AgentEditState`, the batch-REPL turn loop (`_stage_*` functions), render/normalize layers, session/state-machine transitions, provider/runtime wiring, and the response-contract shaping. Land submodules in `vibecomfy/comfy_nodes/agent/` (e.g. `edit/` subpackage or focused sibling modules); `edit.py` becomes a thin facade re-exporting the public surface.

**OUT:** the agent-edit contract, wire format, and UI behavior must not change. Don't touch `provider.py`/`runtime.py`/`worker.py` behavior, the JS contract (`vibecomfy_roundtrip.js` ownership surface), or non-agent files.

## Constraints

- **Behavior-preserving, proven by the gate:** `tests/test_comfy_nodes_agent_edit.py` (large -- deep-imports of private symbols, the grind the prior epic flagged) + `tests/test_comfy_nodes_agent_backend_spine.py` + the full `make browser-smoke` (808/808, incl. the roundtrip/ownership contract tests) must stay green; `make fast` no new failures; `make check` green.
- **Topological risk:** `comfy_nodes/agent/edit.py` is imported widely (headless entry, routes, contracts). A circular import or broken private-symbol import path DESIGNED INTO THE PLAN can stop the suite collecting. The deep test imports of private symbols are the real hazard -- the split must preserve importability of every symbol tests reference, or update those test imports as part of the work. `python -c "import vibecomfy"` + full `__all__` must stay green.
- The headless-safe property (importable without aiohttp/ComfyUI when `VIBECOMFY_HEADLESS=1`) must be preserved.

## Done Criteria

- `comfy_nodes/agent/edit.py` reduced to a thin facade (<=~300 LOC); no submodule >~700 LOC.
- `tests/test_comfy_nodes_agent_edit.py` + `test_comfy_nodes_agent_backend_spine.py` green; `make browser-smoke` 808/808; `make fast` no new failures.
- `python -c "import vibecomfy"; [getattr(vibecomfy,n) for n in vibecomfy.__all__]` succeeds; headless import still works.
- `lint-imports`: 1 kept, 0 broken. `make check` green.

## Touchpoints

`vibecomfy/comfy_nodes/agent/edit.py` -> new submodules under `vibecomfy/comfy_nodes/agent/`. Possibly `tests/test_comfy_nodes_agent_edit.py` import lines if private symbols move (update them; don't break them).

## Anti-scope

Don't change the agent-edit contract, wire format, or UI behavior. Don't touch the JS ownership surface. Don't refactor `porting/emitter.py` or `porting/edit/apply.py` (other milestones).
