# Split porting/edit/apply.py god-file

## Outcome

Decompose `vibecomfy/porting/edit/apply.py` (3009 LOC) into focused submodules (~<=600 LOC each) within `vibecomfy/porting/edit/` -- the edit-application pipeline. The public surface (`vibecomfy.porting.edit.apply` + any re-exports) stays byte-for-byte identical. Edit-application behavior stays byte-identical.

## Scope

**IN:** split `porting/edit/apply.py` by concern -- the apply execution path, the `_resolve_*` helpers (15 of them per the prior epic's notes), conflict/reconciliation, the atomic apply/undo/rebaseline logic, and the gate-proof hooks. Land submodules in `vibecomfy/porting/edit/` (an `apply/` subpackage or focused sibling modules); `apply.py` becomes a thin facade re-exporting the public surface. If the prior epic's m4-resolution unification of the `_resolve_*` helpers (edit_lint vs edit_apply) is still open and tractable within scope, fold it in; otherwise leave the resolve path as-is and just decompose.

**OUT:** edit-application behavior (apply/reject/undo/rebaseline outcomes, conflict detection) must not change. Don't touch `edit_lint.py`/`edit_session.py`/`ledger.py` behavior, or non-edit files.

## Constraints

- **Behavior-preserving, proven by the gate:** `tests/test_porting_edit_apply.py` + `tests/test_porting_edit_session.py` + `tests/test_porting_edit_ledger.py` must stay green; `make fast` no new failures; `make check` green.
- **Topological risk:** `porting/edit/apply.py` is imported by the edit pipeline (session, ledger, lint). A circular import or broken re-export can stop the suite collecting. The `_resolve_*` helpers are shared with `edit_lint.py` -- a move must preserve the resolution contract or the prior m4-resolution unification. `python -c "import vibecomfy"` + full `__all__` green; `.importlinter` green.

## Done Criteria

- `porting/edit/apply.py` reduced to a thin facade (<=~200 LOC); no submodule >~600 LOC.
- `test_porting_edit_apply.py` + edit-session/ledger tests green; `make fast` no new failures.
- `python -c "import vibecomfy"; [getattr(vibecomfy,n) for n in vibecomfy.__all__]` succeeds.
- `lint-imports`: 1 kept, 0 broken. `make check` green.

## Touchpoints

`vibecomfy/porting/edit/apply.py` -> new submodules under `vibecomfy/porting/edit/`. Test import lines only if private symbols move (update; don't break).

## Anti-scope

Don't change edit-application behavior. Don't touch `porting/emitter.py` or `comfy_nodes/agent/edit.py` (other milestones). Don't refactor `edit_lint.py`/`edit_session.py` beyond what the apply split requires.
