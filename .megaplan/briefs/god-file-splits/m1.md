# Split porting/emitter.py god-file (finish structural-decomposition m2-emitter)

## Outcome

Decompose `vibecomfy/porting/emitter.py` (5129 LOC) into focused submodules (~<=600 LOC each) under `vibecomfy/porting/emit/` -- **finishing** what the `structural-decomposition` epic's m2-emitter milestone started (it created the `porting/emit/` subpackage + helpers but left `emitter.py` itself intact). The public surface (`vibecomfy.porting.emitter`, its `__all__`, and `porting/__init__.py`'s lazy-exports) stays byte-for-byte identical. Emitted workflow output stays byte-identical.

## Scope

**IN:** split `porting/emitter.py` by seam into cohesive submodules -- at minimum: signature emission (`emit_available_node_signatures`/`InputSignatureField`/`NodeSignatureRow`/`format_signature_rows`), model-constant hoisting (`_model_*`/`_canonical_model_*`/`_hoist_constants`), wrapper-module resolution (`_wrapper_*`), public-input binding (`_PublicInput*`/`_public_input_specs`), subgraph emission (`_Subgraph*`/`_build_section_groups`), the object-info identity plumbing (`_use_object_info_identities`/`_identity_for_*`/`_drain_lookup_warning_diagnostics` -- recently ported in from the fork consolidation), and the `emit_ready_template_python`/`emit_scratchpad_python`/`emit_agent_edit_python`/`format_as_python` entry points. Land submodules in `vibecomfy/porting/emit/` alongside the existing emit helpers; `porting/emitter.py` becomes a thin facade that re-exports the public surface.

**OUT:** the emitter's OUTPUT (generated workflow Python) must not change. Don't touch the `emit_*.py` entry-point modules' behavior, non-emitter files, or any public symbol name/location.

## Constraints

- **Behavior-preserving, proven by the gate** (keystone): characterization emitter snapshots (`PYTHONHASHSEED=0 pytest tests/characterization/test_emitter_snapshots.py`) must show the SAME 7 pre-existing quarantined failures + 15 passed, ZERO new failures; `make snapshots` (`tools.regenerate_snapshots --check`) must report ALL unchanged.
- **Topological risk (the real failure mode):** a circular import or broken re-export DESIGNED INTO THE PLAN can stop the test suite from collecting -- the gate then can't catch it. The planner must reason explicitly about the import graph: new submodules must not import each other circularly, and `porting.emitter`'s public surface (its `__all__` + `porting/__init__.py`'s lazy-export map) must remain importable from the same paths. `python -c "import vibecomfy"` + full `__all__` traversal must stay green.
- `.importlinter` (ir-core-no-porting contract) stays green. `make check` green end-to-end.

## Done Criteria

- `porting/emitter.py` reduced to a thin facade (<=~200 LOC) or gone; no submodule >~600 LOC.
- Characterization emitter snapshots: no new failures vs the 7-failure/15-pass baseline.
- `make snapshots --check`: all unchanged.
- `python -c "import vibecomfy"; [getattr(vibecomfy,n) for n in vibecomfy.__all__]` succeeds.
- `lint-imports`: 1 kept, 0 broken. `make check` green.

## Touchpoints

`vibecomfy/porting/emitter.py` -> new submodules under `vibecomfy/porting/emit/`. `vibecomfy/porting/__init__.py` lazy-export map only if canonical paths move (prefer: keep `vibecomfy.porting.emitter` as the public path via a facade).

## Anti-scope

Don't change emitted output. Don't rename/move public symbols. Don't refactor non-emitter code. Don't touch `comfy_nodes/agent/edit.py` or `porting/edit/apply.py` (other milestones).
