# How Should Credentials Be Synced?

## Direct Answer

Credentials should be synced through an **explicit, declarative SecretSpec manifest** pushed from the laptop to the remote machine over SSH. Each credential is defined by a spec that declares its local source path, persistent remote destination under `/workspace/secrets/`, runtime injection target, file permissions, and a health-check command. The sync command (`agentbox creds push`) copies only declared credentials — never a full environment dump. A companion `test` command validates liveness remotely. Rotation updates both sides atomically. Credentials never transit Discord; they travel exclusively over the already-established SSH control channel.

## Rationale

The laptop is the credential source of truth. The AgentBox machine is a derived consumer that needs specific secrets to operate autonomously (GitHub tokens, Codex/Claude auth, API keys). First principles demand:

1. **Least privilege** — Only push what the box actually needs, not the user's entire keychain. The plan's own rule is correct: "sync only explicit credentials. Do not dump the entire local environment."

2. **Addressability** — Each credential must be individually testable and rotatable. A monolithic env file fails this; a manifest of named specs passes it.

3. **Channel integrity** — Discord is a chat surface with message history, bot logs, and potential LLM summarization. Secrets must never pass through it. SSH is already the bootstrap and control channel; it should also be the credential channel.

4. **Fail-closed** — The Guardian must know *before* launching an operation whether its required credentials are present and valid. This requires a machine-readable credential registry, not ad-hoc env sourcing.

5. **File-over-env** — Credentials should land as permissioned files (`chmod 600`), not raw environment variables. Files avoid process-tree leakage via `/proc`, survive tmux session restarts cleanly, and are easier to audit (`ls -la /workspace/secrets/`).

## Risks And Caveats

1. **Disk-at-rest exposure.** Permissioned files on a remote VM are a meaningful risk. The plan correctly recommends starting with `chmod 600` and designing the interface so `age`, `pass`, or 1Password CLI can replace the backend later. The SecretSpec abstraction should not bake in plaintext as a permanent assumption.

2. **OAuth token expiry.** Codex and Claude OAuth tokens rot. The Guardian must detect token failures (via health-check commands that return non-zero) and notify Peter with the exact `agentbox creds push` command needed — not just "auth failed."

3. **Bootstrap circularity.** The SSH key that secures the push channel is itself a credential. The initial bootstrap must handle this manually. Document it clearly; don't try to automate the first push.

4. **Manifest drift.** If the local manifest declares credentials that don't exist on disk, or the remote machine has stale files not reflected in the manifest, operations fail mysteriously. The `test` command should be the canonical reconciliation mechanism.

5. **Rotation race conditions.** If a credential is rotated locally while an operation is running on the remote box, the operation could fail mid-flight. Rotation should be treated as a coordination point — perhaps requiring no active operations depending on that credential.

## One Concrete Design Implication

**The operation registry must model credential dependencies per operation kind.** When the Guardian prepares to launch or restart an operation, it should cross-reference the operation's declared credential requirements against the remote machine's credential health status. If `agentbox creds test` shows a required credential is missing or expired, Guardian should block the operation and notify Peter with the specific remediation command, rather than launching and letting it fail mid-run.

This means the operation record (or the operation kind adapter) needs a field like:

```yaml
required_credentials:
  - name: github-token
    health_check: "gh auth status"
  - name: codex-oauth
    health_check: "test -f /workspace/secrets/codex-auth.json"
```

This dependency model turns credential sync from a one-time push into an ongoing health property that the Guardian can reason about — which is consistent with the plan's bias toward "figure out what's wrong and fix it" rather than passive reporting.
