# Repository Strategy Roadmap

Establish an authoritative typed Markdown repository strategy with a validated Now/Next/Later roadmap whose actionable entries reference immutable tickets or canonical epics, with rebuildable projections and complete lifecycle, CLI, migration, test, documentation, and repository-adoption support.

## Current truth and canonical index

This README is the initiative front door. The North Star remains the durable
end-state constraint; `STRATEGY.md` is the authoritative repository roadmap;
briefs define the coordinated workstreams. The initiative is committed, but
editing its documents does not launch or approve a chain.

- [NORTHSTAR.md](NORTHSTAR.md) — durable end state, invariants, and drift signals.
- [STRATEGY.md](STRATEGY.md) — authoritative typed Now/Next/Later roadmap.
- [decisions/document-ticket-initiative-lifecycle-and-hot-context.md](decisions/document-ticket-initiative-lifecycle-and-hot-context.md) — canonical lifecycle, placement, promotion, recent-activity, and resident navigation contract.
- [research/current-surfaces-20260713.md](research/current-surfaces-20260713.md) — pre-implementation surface audit.
- `briefs/` — coordinated milestone briefs.
- `research/` — evidence, audits, and synthesized alternatives.
- `decisions/` — accepted durable contracts and their consequences.
- `notes/` — retained working notes awaiting promotion.
- `handoff/` — curated handoffs; raw agent/subagent output stays in its run store and is cited from a canonical synthesis.
- `assets/` — supporting non-prose files.

`NORTHSTAR.md` and `chain.yaml` are lifecycle/readiness artifacts, not required
for an exploratory initiative-shaped workspace. Search this initiative and its
related tickets/documents before adding a new artifact; promote notes or
research into a decision/brief and update this index when they become current
truth.
