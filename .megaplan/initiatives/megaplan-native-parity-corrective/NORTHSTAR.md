# Megaplan Native Parity Corrective North Star

Canonical Megaplan must become source-authoritative native workflow code.

The final semantic authority is:

- `arnold_pipelines/megaplan/workflows/workflow.pypeline`;
- named native subworkflows imported by that source;
- declared policies attached to named source constructs;
- retained pure phase bodies behind typed interfaces.

The final system must not require `components.py`, route bindings, handler refs,
manifest backend route translation, `Pipeline.native_program` compatibility
projections, auto-drive next-step derivation, or CLI command handlers to
understand report-owned product topology.

The intended end state is described by
`docs/arnold/megaplan-native-representation-report.md`; the current-state map is
`docs/arnold/megaplan-native-current-codebase-map.md`; the execution plan is
`docs/arnold/megaplan-native-parity-corrective-plan.md`; the substrate
decision/judgement layer is `docs/arnold/megaplan-native-oracle-synthesis.md`.

## Non-Negotiables

- The current component-call skeleton fails the semantic checker before any
  extraction milestone can close.
- Canonical Megaplan executes through `.pypeline` lowering into the existing
  DSL/manifest runtime for this corrective epic.
- `build_pipeline()` must consume lowered `.pypeline` topology or be replaced /
  quarantined before broad extraction can close.
- Each traceability row is re-proved from row-level source/policy/pure-body
  evidence. No row inherits old `enabled` or `implemented` status.
- Components, handlers, manifests, graph projections, traces, auto-drive, and
  CLI are consumers/adapters. They are not semantic authority.
- Each milestone deletes or quarantines the component/handler carriers it
  replaces. Component collapse is incremental, not a final big-bang cleanup.
- Behavior parity is proven with split-outcome scenarios, not happy-path smoke
  tests.
- Any narrowing of the end-state target must include both a checker rule that
  still blocks the old false-pass pattern and a behavior scenario proving the
  narrowed carrier cannot smuggle routing.

## Done Means

The canonical `.pypeline` source and named subworkflows visibly own prep,
critique, gate, revise, tiebreaker, finalize, execute, review, override, human
gates, retries, model routing, suspension, and resume. A reviewer can follow the
workflow without reading handler-local state transitions or component route
tables, and installed-package execution follows the same source-derived
semantics.
