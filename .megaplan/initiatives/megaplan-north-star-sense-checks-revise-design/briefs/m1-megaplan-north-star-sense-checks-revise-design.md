# Implement Megaplan North Star Sense Checks in Revise

## Objective

Implement the core one-sprint mechanism described in:

- `docs/arnold/megaplan-north-star-sense-checks-revise-design.md`

This is the S0/review-plumbing sprint needed before the native semantic parity
work relies on North Star sense checks. Do not implement semantic-parity
extraction. Implement only the Megaplan runner/planning machinery that lets
explicit North Star questions become enforceable revise/review actions.

## Scope

Build the smallest complete implementation that makes North Star sense checks
actionable:

1. Add `north_star_actions[]` to the gate/gate-carry path.
2. Render `north_star_actions` into revise context when present.
3. Add `north_star_actions_addressed[]` to revise output/meta handling.
4. Add pre-worker revise guard behavior:
   - halt before invoking the worker for `add_human_halt`;
   - halt for unmappable blocking actions;
   - halt for route-authoritative exemptions, baseline rewrites, target
     narrowing, and live-plan topology/resume decisions when they are not
     explicitly mapped to a concrete plan change.
5. Add post-worker revise validation:
   - addressed actions must cite concrete plan refs;
   - prose-only "handled" claims fail;
   - each addressed action must satisfy action-type-specific structural markers
     where practical.
6. Add schema-side dangerous-category severity normalization:
   - route authority;
   - baselines;
   - row/carrier exemptions;
   - target narrowing;
   - generated conformance authority;
   - live-plan topology/resume risk.
   These are always blocking, even if the worker/reviewer labels them advisory.
7. Add `north_star_critical` to chain/milestone config. Reject
   `north_star_critical: true` with robustness modes that skip the required
   plan/gate/revise/review enforcement phases. Do not silently upgrade
   robustness.
8. Add review/finalize closeout blocking for unresolved blocking North Star
   actions, using the repo's existing gate/review/finalize architecture. If the
   repo has no single clean closeout hook, implement the narrowest compatible
   hook and document remaining integration work as a ticket.
9. Update Megaplan skill/docs so agents know:
   - where questions are defined;
   - questions are explicit, not inferred from arbitrary prose in v1;
   - gate/review create blockers;
   - revise must make concrete plan changes or halt;
   - `bare`/`light` cannot be used with `north_star_critical`.

## Non-Goals

- Do not implement native semantic parity extraction.
- Do not build a broad autonomous governance system.
- Do not infer questions from arbitrary North Star prose in v1.
- Do not make full worker/model I/O replay.
- Do not make this depend on live model output for tests.
