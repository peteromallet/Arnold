## Source Brief

# Canonical Run-State Control Plane

## Outcome

Deliver the root fix for the repair/status loop that caused sessions like
`agent-ui-lifecycle-parity`, `progress-auditor-stage-metrics`, and
`megaplan-native-parity-corrective` to cycle through misleading `blocked`,
`manual_review`, `failed`, `repairing`, and `needs human` labels.

The implementation must introduce a read-only canonical run-state resolver and
wire it into the first enforcement point that matters: machine-actionable
implementation blockers must not become human-only stops.

## Why This Exists

The current stack has multiple independent readers of run state:

- live process / active worker
- plan `state.json`
- execute/finalize artifacts
- chain state
- repair-data
- watchdog markers
- status/Discord summaries

Those readers disagree. One layer projects a nuanced state into `manual_review`
or `failed`; another layer treats that derived label as truth; the superfixer
then patches a narrow guard instead of addressing the evidence model.

The specific current example is `megaplan-native-parity-corrective`: the plan is
blocked on real unfinished implementation work (`AWF018_ROUTE_METADATA_MISMATCH`
from missing route bindings and stale topology fixtures), but the system treats
that dependency as a terminal manual-review stop instead of a machine-actionable
repair/replan.

## Locked Decisions

- Build a read-only resolver. It classifies evidence; it does not mutate state.
- Keep mutation policy in existing consumers: watchdog, repair-loop, chain,
  auto, status/Discord, progress auditor.
- Split canonical state from action policy.
- `manual_review`, `blocked`, and `failed` are raw labels, not final decisions.
- `HUMAN_ACTION_REQUIRED` must require an explicit human gate: unresolved user
  action, approval, credential/account/quota, human verification, or policy
  block.
- Machine-actionable implementation blockers, including route-binding gaps,
  fixture refreshes, stale assertions, deterministic test failures, and task
  budget exhaustion, are not human gates.
- Do not touch profile TOML files, default routing, provider selection, phase
  pins, or model/vendor policy.

## Scope

IN:

- Implement `arnold_pipelines.megaplan.run_state` with frozen dataclasses,
  evidence gathering, and ordered classifiers.
- Add canonical states:
  - `RUNNING`
  - `REPAIRING`
  - `RETRYABLE_EXECUTION_BLOCK`
  - `REAL_IMPLEMENTATION_BLOCK`
  - `HUMAN_ACTION_REQUIRED`
  - `COMPLETED`
  - `STALE_DERIVED_STATE`
  - `BROKEN_STATE_MACHINE`
  - `UNKNOWN`
- Add typed human gates:
  - `approval_required`
  - `credential_missing`
  - `quota_or_account`
  - `human_verification`
  - `policy_block`
  - `explicit_user_action`
- Add fixture tests for the July incident shapes.
- Wire status snapshot/format to expose canonical state in observe-only mode.
- Wire watchdog needs-human preservation/notification to consult canonical
  state before escalating.
- Wire repair-loop prompt generation to include canonical state, stale sources,
  root-cause fingerprint, and next-action hint.
- Add a first enforcement path so `REAL_IMPLEMENTATION_BLOCK` with
  `human_required=false` triggers machine repair/replan rather than a human-only
  stop.
- Include a fixture for `megaplan-native-parity-corrective` where
  `AWF018_ROUTE_METADATA_MISMATCH` is classified as a machine-actionable real
  implementation block.

OUT:

- Do not fully rewrite chain progression.
- Do not change model/profile routing.
- Do not directly fix the `megaplan-native-parity-corrective` route bindings in
  this sprint unless needed only as a fixture or smoke validation.
- Do not make the resolver a mutating control plane.
- Do not make status optimistic; real implementation blocks should remain
  visible as real blocks.

## Done Criteria

- Resolver tests cover:
  - live worker beats stale manual-review marker;
  - budget exhausted/no files modified is retryable;
  - repeated same-fingerprint retry escalates to broken state machine;
  - `AWF018_ROUTE_METADATA_MISMATCH` is `REAL_IMPLEMENTATION_BLOCK` and not
    human-required;
  - deferred baseline plus completed real tasks is progressable/completed;
  - stale needs-human marker after recovery is stale derived state;
  - explicit unresolved approval remains human-required;
  - missing credential/account remains human-required;
  - profile/model config is unchanged after resolution.
- Status/Discord-facing data can show canonical state without changing legacy
  fields in observe-only mode.
- Watchdog does not preserve or emit needs-human for canonical
  machine-actionable implementation blocks.
- Repair-loop prompts receive the canonical state and cannot count generic
  engine patches as success when target implementation remains unattempted.
- There is an environment-gated rollout path:
  - `ARNOLD_RESOLVER_OBSERVE=1`
  - `ARNOLD_RESOLVER_ENFORCEMENT=0`
- Focused tests pass.
