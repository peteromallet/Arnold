# C4: Authoring-API Enforcement + Hard Acceptance Gate (+ Migration-Delta)

Condenses old m7 (authoring-API enforcement) + m8 (hard acceptance gate), and ABSORBS the genuine
remainder of old m5/m6 (load-bearing + long-tail migration) as a migration-delta task. Bases off
`arnold-generalized-pipeline @ 944d10cc`. This sprint passes or fails the epic.

## Outcome

Close seam #4 — Author ⇄ Runtime — by making the authoring API's contract declarations ENFORCED, and
then prove the whole epic against its own failure class and the platform-scale claim. Framed
HONESTLY as **correctness + composability at scale + a SHAPE-not-MEANING limit**, NOT a "50 pipelines
overnight" velocity claim: the contract guarantees that a pipeline author declares contracts and the
runtime handles every crossing correctly; it does NOT guarantee that authoring is fast or that a
well-typed payload is semantically true.

1. **Enforce the authoring API's `reads`/`writes`.** `ReadRef`/`WriteRef` and `reads`/`writes`
   already exist on stages (`arnold/pipeline/types.py:217-218/248-249`) and `derive_binding_map`
   exists (`arnold/pipeline/builder.py:15/141`), but the `binding_map` is today "a Megaplan concern"
   (`types.py:271/278`) and the generic executor enforces NOTHING. C4 makes a stage's declared
   input/output contract bind to the typed ports and validate through C1's executor seam + chokepoint
   + the structural audit at runtime. The author declares `{input contracts, invocation spec (kind +
   adapter_config), output contract}` — "instruction" DEMOTED to the model adapter's `adapter_config`
   — and the runtime handles every crossing.
2. **`arnold pipeline check` (a contract-aware static command).** Today only a module-validation
   `check` verb exists (`arnold/pipelines/megaplan/cli/arnold.py:40/153`) — NOT contract-aware. C4
   adds a static authoring check that verifies declared contracts are well-formed, producer/consumer
   contracts line up, AND declared capabilities (requires-vision-model / requires-image-decoder) are
   satisfiable before a run.
3. **Hard acceptance gate**: regression of the four motivating failures + a validation-overhead/
   concurrency benchmark + a seam-coverage matrix + a SECOND toy pipeline (the Arnold-native
   `evidence-pack` verifier) riding the contract via the authoring API.
4. **Migration-delta (absorbs old m5/m6):** the migration's phase relocation moved finalize/critique/
   review/gate and the long tail into `arnold/pipelines/megaplan/` already on the model-seam path, so
   m5/m6 are LARGELY ABSORBED. C4 folds in only the GENUINE remainder surfaced by the outbound
   coverage proof — any residual `_normalize_worker_payload` fuzzy-key path or un-migrated parse site
   — and routes it onto the contract / deletes the old path; the bulk is not re-litigated.

## Scope

IN:

- **Authoring enforcement**: bind a stage's declared `reads`/`writes` to the typed ports (C1) and
  validate the crossing through the chokepoint + structural audit at runtime, with no author-side
  assembly/parsing. The author declares `{input contracts, invocation spec (kind + adapter_config),
  output contract}`; "instruction" is a field of the model adapter's `adapter_config`, so a model
  step and a (future) non-model step are both expressible via the one invocation-spec shape (the
  non-model adapter may stay fail-closed/unbuilt but must be AUTHORABLE).
- **`arnold pipeline check`**: a static command (distinct from the existing module-validation `check`
  verb) verifying contract well-formedness, producer/consumer line-up, schema-version availability,
  AND capability satisfiability (e.g. requires-vision-model, requires-image-decoder) before a run.
- **Migration-delta**: run the OUTBOUND coverage proof (every `validate_payload`/output-parse site
  catalogued + closure-proven, the twin of C2's inbound `render_step_message` closure); fold ONLY the
  genuine remainder (residual `_normalize_worker_payload`, any un-migrated parse site) onto the
  contract and delete its old path. Do NOT re-migrate what the migration's phase relocation already
  moved onto the model-seam path.
- **Acceptance gate (hard)**:
  - Regression of the 4 motivating failures: wrong-typed payload passes (closed by the structural
    audit, C2); char→token overflow silently exceeds budget (closed by the real-tokenizer
    assembly-time budget, C2); first-key-valid parse accepts a malformed output (closed by
    `capture_step_output` + audit, C2); suspended child silently treated as completed (closed by
    suspension-aware composition, C3).
  - A validation-overhead/concurrency benchmark with CONCRETE thresholds: fail the gate if validation
    adds >10% phase wall-clock, OR >500ms short-phase, OR p95/artifact over {2ms metadata, 8ms ≤1MiB,
    25ms 1-4MiB, 150ms 100MiB-with-hash}. Load = linear 10-stage + fan-out 8/32/64, artifacts
    1KiB-100MiB, 20 runs median/p95, gate at width 32. By-ref: full-parse envelope + structured
    ≤1MiB; >1MiB validate the sidecar manifest (content-type/schema-hash/size/sha256); blobs always
    by-ref; hash on WRITE not read. (Resolved decision #5.)
  - A seam-coverage matrix: every architectural-spine seam (Step⇄Step, Step⇄Model incl.
    Engine⇄Worker, Step⇄State, Author⇄Runtime, plus Engine⇄World and control-flow forks) marked
    implemented / delegated (to Evidence-First) / out-of-scope, with each one's location.
  - The SECOND pipeline: a deterministic Arnold-native `evidence-pack` verifier
    (`ingest → parallel_validators → reduce → human_review? → emit_attestation`) authored purely via
    the C4 authoring API and riding the contract end-to-end — external `ReadRef`/`WriteRef`, multiple
    content-types, fan-out-reduce, a typed verdict (NOT megaplan labels), human suspend/resume
    (driven by a simulated fixture), route-bypass-prevention, large-by-ref. NOT a renamed megaplan
    clone (which proves nothing).
  - A go/no-go acceptance verdict gating the merge.

OUT:

- New feature work or new milestones beyond enforcing the existing declarations and proving the
  built foundation.
- The `human_review` verb / resume UX (a feature on top of C3's suspension primitive).
- Running Evidence-First (a separate epic); C4 only confirms the seam is delegated, not implemented.
- Re-migrating the load-bearing-5/long-tail bulk the migration's phase relocation already moved — C4
  folds ONLY the genuine residual delta.
- Any new validator/registry/chokepoint/type; all consumed.

## Constraints

- Authoring enforcement must be additive + gradual-typing-governed; an existing un-migrated pipeline
  still runs unchanged and a running chain is not disturbed.
- An author must not hand-roll serialize-in or validate-out for a declared crossing.
- `arnold pipeline check` must catch declaration AND capability mismatches before a run, not at first
  crossing.
- The gate must be objective: each motivating failure has a concrete reproduction that fails
  pre-contract and passes post-contract.
- The benchmark must exercise the always-on audit hot path at a realistic concurrency level.
- The 2nd toy pipeline must use the public authoring API (not internal hand-rolling) or it does not
  prove the seam-4 claim; and must be a genuine non-megaplan shape, or it proves nothing.
- The seam-coverage matrix must be exhaustive over the spine — an unaccounted seam is a gate failure.

   validates-out, runs the structural audit, and propagates suspension for a declared crossing with
   no author-side assembly/parsing; a pipeline authored purely via the `{input contracts, invocation
