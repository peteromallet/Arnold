# North Star: Extension Reality Convergence

## End State

The Reigh video-editor extension system is truthful and converged across trust, composition authority, and export readiness. Extensions are presented as trusted local code unless a later untrusted-runtime epic adds real isolation. Composition facts used by planner/export/preview flow through a canonical authority spine instead of scattered legacy timeline reads. Export readiness is decided by planner-owned blockers, with guard scans and provider route checks demoted to structured planner inputs.

## Success In One Sentence

From a clean checkout on top of the completed extension-foundation work, Reigh can prove that extension trust claims are honest, the first composition fact family has moved to graph authority, and every user-facing export block originates from canonical planner readiness.

## Immutable Constraints

- Do not implement fake or partial permission enforcement and call it sandboxing.
- Do not claim marketplace, third-party, or untrusted extension safety.
- Keep extension code trusted and unsandboxed unless a separate future epic introduces isolation and brokered APIs.
- Do not delete legacy timeline storage fields before compatibility projection and regression tests exist.
- Do not let migrated fact families remain authoritative through direct legacy timeline reads.
- Do not add a second export readiness authority beside the planner.
- Do not weaken quality gates, docs checks, or release checks to make the epic pass.
- Preserve existing editor behavior for timelines and packages that do not use migrated composition features.
- Keep generated-module, provider, and export-guard scans where useful, but make them feed planner readiness instead of bypassing it.

## Required Proof

- SDK, JSON schema, examples, docs, and manager UI agree on the trusted-code/access-disclosure model.
- A regression gate fails if docs or UI imply sandboxing, runtime permission enforcement, marketplace safety, or third-party safety.
- The first composition authority ratchet proves shader/ref readiness comes from graph-projected facts, not direct legacy shader fields.
- A fixture proves legacy-only shader/ref fields do not count as authoritative after projection is disabled.
- User-facing export/render blocked states are derived from planner blockers.
- Generated-module artifact state, provider availability, output-format availability, contribution conflicts, shader/material blockers, and live-binding blockers are represented in planner readiness.
- Quality checks pass from a clean checkout with the convergence gates enabled.

## Explicitly Deferred

- True permission enforcement through iframe, Worker, process, SES-like isolation, or a brokered host API.
- Marketplace install, remote extension discovery, update/delete flows, code signing, and dependency resolution.
- Full composition-spine completion beyond the fact families named by this epic.
- Full process/sidecar runtime hardening unless required as an input to export-readiness diagnostics.
