# L3 reconciler-auditor + audit-the-auditor + external coverage + retire duplicates
**Profile: partnered-5/full/high (+with_prep)**

## Outcome
L3 is a reconciler-first auditor that audits the resolver + L2 themselves; red-main CI + engine-tree consistency are detected; the 3 duplicate initiatives are retired.

## Scope
- L3 (`six_hour_auditor.py` + `arnold-progress-auditor`) becomes reconciler-first: compares resolver vs ledger vs live process vs repair attempts vs install freshness vs GitHub sync; surfaces drift as a deterministic gather reason.
- **GAP #3 (audit-the-auditor):** L3 recursion guard; `auditor_escalate_to_human` (no self-referential "more meta-repair when meta-repair is broken"); resolver-correctness checks beyond fixtures (semantically-wrong output detectable; `confidence=low` gates).
- **GAP #6 (external coverage):** detect red-main CI as base-health (`gh pr checks`/`gh run list` for the base branch — resolver claims PR/CI but never reads CI today); verify engine-tree consistency across sibling mirrors (import gate for changed CONSUMERS, not just pipeline registrations — `/workspace/arnold`'s ~46 uncommitted shared files).
- Retire incident-control-plane, superfixer-repair-custody, tiered-repair-hardening as separate initiatives (facets now in master); document + clear stale markers.
- Anti-scope: NO fixing main CI (detect only); NO recreating deleted workspaces (detect only); NO merging workflow-boundary-contracts.

## Constraints
- L3 is read-only (audits + hands off; never mutates live chains).
- The resolver + L2 are themselves auditable (no unbackstopped authority).

## Done Criteria
- L3 audits the resolver + L2 (test: a lying resolver / false-FIXED L2 caught within one cycle).
- Red-main CI detected as base-health (test).
- Engine-tree inconsistency across mirrors caught pre-`verified_recovered` (test).
- The 3 duplicate initiatives' sessions retired/marked superseded.

## Execution contract

- **Sizing:** roughly two weeks of human work.
- **Locked decisions:** L3 is read-only, recursion fails to human escalation,
  and duplicate control-plane initiatives remain retired.
- **Open questions / human gates:** approve external CI/provider credentials,
  escalation owner and retirement evidence; no automatic external mutation.
- **Touchpoints:** six-hour auditor/progress wrapper, incident ledger, GitHub/CI
  evidence, install/source consistency and initiative retirement markers.
- **Anti-scope:** detect but do not fix red main or deleted workspaces; do not
  merge or alter WBC.
- **Evidence handoff:** auditor self-test, external coverage fixtures, retirement
  ledger and M5 lineage baseline.
- **Profile rationale:** difficulty 5/5, `partnered-5/full/high +prep`; the
  backstop itself can lie or recurse while local tests remain green.
- **Dependencies:** M3 canonical watchdog/guard/verifier behavior.
- **Rollback / compatibility:** disable L3 scheduling while preserving audit
  events; retired initiatives remain launch-disabled and are not revived.
