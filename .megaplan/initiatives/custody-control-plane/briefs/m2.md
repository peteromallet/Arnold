# Wire resolver into status + repair dispatch + enforcement + observability
**Profile: partnered-5/full/high (+with_prep)**

## Outcome
status_snapshot + repair-dispatch consume the resolver; dispatch is a HARD gate (rejects non-canonical inputs); drift is emitted to the ledger.

## Scope
- Wire resolver into `status_snapshot._classify_session` — defers to canonical before the needs-human/watchdog/chain-complete/liveness branches. PR fields come from the resolver (live `gh`) not chain-health storage (`status_snapshot.py:786`).
- Wire into `classify_repair_dispatch` (`repair_contract.py:540-702`) — consumes `CanonicalRunState`; `manual_review` never a dispatch policy; unknown → `broken_superfixer`; machine-actionable → L1/replan.
- **GAP #5 (enforcement-as-gate):** dispatch REJECTS inputs not from `resolve_run_state()` (authoritative, not advisory — or it's bypassed like the phase_model pin was).
- **GAP #4 (observability):** mandatory `DRIFT_DETECTED` emissions to `events.ndjson`/ledger when the resolver's verdict disagrees with the prior derived label (wire the dead `DRIFT_DETECTED` event, `observability/events.py:155`).
- Anti-scope: NO watchdog/chain-guards (m3) or L3 (m4); do NOT change the resolver (m1).

## Constraints
- Resolver is the single classification truth at these two call sites.
- Legacy fields kept (backward-compat); `stale_sources` surfaces disagreement.

## Done Criteria
- status + dispatch consume the resolver (test: canonical-vs-derived disagreement is surfaced, not silently relabeled).
- Dispatch rejects a non-canonical input (test).
- `DRIFT_DETECTED` emitted on disagreement (test).

## Execution contract

- **Sizing:** roughly two weeks of human work.
- **Locked decisions:** canonical evidence is mandatory for dispatch; drift is
  durable; legacy fields are projections only.
- **Open questions / human gates:** approve observe-to-enforce promotion,
  freshness thresholds and rollback owner; default remains observe-only.
- **Touchpoints:** status snapshot/format, repair contract/dispatch, feature
  flags and drift events.
- **Anti-scope:** no watchdog/chain/L2/L3 expansion or resolver redesign.
- **Evidence handoff:** shadow comparison, enforcement-negative tests, drift
  receipts and rollout/rollback record for M3.
- **Profile rationale:** difficulty 5/5, `partnered-5/full/high +prep`; a bypass
  can preserve dual authority despite green resolver tests.
- **Dependencies:** M1 schema, fixtures and deploy identity.
- **Rollback / compatibility:** disable enforcement while retaining observation
  and drift append; never restore raw labels as action authority.
