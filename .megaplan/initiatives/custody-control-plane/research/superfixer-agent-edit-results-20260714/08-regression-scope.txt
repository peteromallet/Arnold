## Root Cause

**The observed argparse error** occurs because the dirty `/workspace/arnold/managed_agent.py` adds four new **required** CLI flags — `--origin-kind`, `--origin-id`, `--origin-component`, `--trigger-id` (lines 1006–1009) — that did not exist in the clean supervisor source (rev 4847a74c6). If a **deployed** wrapper at `/usr/local/bin/` (installed from the clean source) invokes `python3 -m arnold_pipelines.megaplan.managed_agent run` against the dirty PYTHONPATH, argparse fails hard.

## Smallest Source Patch (managed_agent.py argparse)

Make the four provenance flags **optional with safe defaults** instead of `required=True`. This preserves the new guard infrastructure while letting wrappers upgrade asynchronously:

```python
# Lines 1006–1009: change required=True → default=""
run.add_argument("--origin-kind", default="")
run.add_argument("--origin-id", default="")
run.add_argument("--origin-component", default="")
run.add_argument("--trigger-id", default="")
```

In `main()` (lines 1055–1060), gate provenance construction: skip `machine_origin_provenance()` when `args.origin_kind` is empty; pass an empty/sentinel provenance dict instead. `reserve_managed_command` already validates via `normalize_machine_origin_provenance`, which will raise if provenance is missing — the guard remains closed.

## Missed Backstop Patch

`confirm_managed_agent_dispatch()` in `arnold-watchdog` (lines 338–385) already validates manifests via `validate_automatic_managed_manifest`. But the watchdog's **initial dispatch** at line 4389 only calls `confirm_managed_agent_dispatch` *after* the subprocess fork. There is no pre-flight validation that the wrapper's own `managed_agent run` invocation includes the provenance flags. **Add a dry-run validation** before line 4372 that shell-escapes and checks the managed_cmd array for `--origin-kind` presence, refusing to dispatch if absent. This is the backstop that ensures no wrapper shipping from the clean source can silently regress.

## Concurrent Dirty Changes — Do Not Overwrite

The entire provenance infrastructure is one atomic feature spanning `managed_agent.py` (+410), all 6 wrappers (+54 to +413), `launch_hermes_agent.py` (+122), `subagent.py` (+1523), `repair_contract.py` (+141), and `repair_requests.py`. Cherry-picking only the argparse soft-fail would leave the wrappers in an inconsistent state. The `--require-output`, `--child-difficulty-ceiling`, sealed stdin, `validate_automatic_managed_manifest`, and `MACHINE_ORIGIN_ENV` inheritance chain are all coupled. The smallest deployable unit is the full set. The **argparse optional-default patch** above is the narrowest surgical fix that preserves all guards.
