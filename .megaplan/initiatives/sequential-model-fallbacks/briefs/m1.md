# Profile Fallback Chains V1

## Outcome

Implement v1 ordered fallback model chains for Megaplan profiles. A profile, tier model, or prep model entry can use a TOML array of model specs; Megaplan preserves the chain through profile expansion, state/resume, preflight, fanout, and observability, then attempts fallback for non-execute availability failures only.

## Scope

In:

- Add a shared fallback-chain module with `FallbackSpecChain`, normalization, canonical compact JSON encode/decode, `provider_family(spec)`, and retryability classification.
- Accept and validate `str | list[str]` profile entries for phase routes, `tier_models`, and `prep_models`.
- Preserve all existing single-string profile behavior.
- Map vendor/depth/critic/available-model/DeepSeek rewrites over every chain element.
- Encode only unavoidable string-only `phase_model` persistence values as `phase=__fallback_json__:[...]`; decode before `parse_agent_spec`.
- Thread chains through tier resolution, prep stage resolution, adaptive critique, tiebreakers, and `WorkerUnit` fanout without changing `AgentMode.__iter__`.
- Add ordered attempts for non-execute worker dispatch and per-unit fanout.
- Suppress ambient runtime fallback when an explicit configured chain is present.
- Preserve and preflight execute chains, but block automatic `execute`/`loop_execute` fallback in v1 with an explicit `execute_fallback_unsafe` error.
- Extend routing ledger, receipts, history, active step, status, and audit/reporting with additive fallback-attempt fields.
- Update local/runtime preflight, cloud preflight, cloud CLI phase-model materialization, chain handling, and resume validation so chains are not flattened.
- Add focused tests and docs.

Out:

- No comma-delimited syntax.
- No CLI fallback-list syntax.
- No fallback for quality, malformed output, schema, test, evidence, blocked, gate, review, or semantic failures.
- No v1 automatic execute fallback.
- No topology-changing parallel review fallback.

## Constraints

- Use the design in `research/model-fallback-design.md` as the source of truth.
- Use `decisions/deepseek-wave-synthesis.md` and `decisions/final-codex-synthesis.md` for risk notes and judgment calls.
- Add golden characterization tests before changing routing behavior.
- Keep `AgentMode.__iter__` as `(agent, mode, refreshed, model)`.
- Legacy scalar `model`, `selected_spec`, cache identity, and session identity must refer to the selected/actual attempt, not the configured chain.
- `phase_model` remains `list[str]`; chain YAML/cloud upload must preserve encoded strings exactly.
- `provider_family(spec)` decides whether auth/quota/rate-limit fallback is independent.

## Done Criteria

- Existing single-string profiles, overrides, persisted state, prep routing, execute tier routing, local preflight, and cloud preflight pass characterization tests unchanged.
- TOML array phase, tier, and prep entries validate; empty arrays and non-string members fail clearly.
- Policy rewrites apply to every chain element.
- Encoded `phase_model` chain values round-trip through state validation, resume, chain/cloud paths, and `resolve_agent_mode`.
- Non-execute retryable infrastructure failure attempts the next spec; semantic/permanent failure does not.
- `execute` and `loop_execute` fallback chains are preserved and reported but never automatically attempt spec index `>0` in v1.
- Fanout fallback is per unit and survives `WorkerUnit` cloning and process pack/unpack.
- Observability records configured specs, attempted specs, selected spec/index, fallback trigger, and failed-attempt reasons without removing existing scalar fields.
- Cloud preflight reports dependencies for every spec in every chain.
- Tests cover the minimum matrix in `research/model-fallback-design.md`.
