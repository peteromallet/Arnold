# Extension Reality Chain Recovery Continuation North Star

Make `extension-reality-convergence-epic` safe to run again directly on the cloud chain runner.

The desired end state is not merely that product code landed. The chain system must be able to verify publication truth, avoid the M3 false-completion class, surface custody mismatch instead of reporting complete, and restart or resume from a corrected target without trusting stale `.megaplan` state.

Non-negotiable invariants:

- Do not restart the old corrupted chain as-is.
- Do not delete or rewrite user or agent work without preserving it first.
- Keep recovery artifacts and branch evidence durable.
- Treat GitHub branch/PR state and fresh commands as authoritative over old summaries.
- Prefer focused patches and tests over bundling unrelated dirty cloud work.
- Before declaring success, prove the corrected target contains M1+M2+M3+M4 and the chain engine rejects the original false-positive pattern.
