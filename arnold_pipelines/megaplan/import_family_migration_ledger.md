### Migration guidance (derived from compatibility audit)

| Strategy | Symbols | Rationale |
|----------|---------|-----------|
| **Repoint to `arnold.pipeline.types`** | Edge, PipelineVerdict, Port, PortRef | Identical or Arnold-superset shapes. Consumer constructor calls and equality checks work without change. |
| **Retain in Megaplan-owned module** | StepContext, Step, StepResult, Overlay | Fundamentally different runtime contracts or Megaplan-specific opinions. Repointing would break consumers. |
| **Use Arnold as base; Megaplan extends** | Stage, ParallelStage, Pipeline | Arnold provides the neutral core shape with all legacy fields (or supersets). Megaplan consumers can use Arnold types directly; Megaplan-specific extensions (overlays, builder) belong in Megaplan-owned modules. |
